## Case A/B 및 이연 경계

- **Case A = 장소 우선**: 장소 저장 → 무숙소 초안 일정 → 완성 동선 기반 숙소 권역 추천 → 숙소 등록 → 숙소 앵커 재정렬. 무숙소 생성은 이 전용 온램프에서만 허용한다.
- **Case B = 숙소 우선**: 숙소 등록 → 등록 숙소 위치·체크인/아웃을 앵커로 일정 생성. 숙소 우선 기본 생성에서 숙소가 없으면 버튼을 비활성화한다.
- `US-E3-06`은 **1차 수동 등록 경로가 범위**다. OTA 포스트백 기반 1탭 자동 등록만 후속 이연이다.
- 원문 `units.md`의 '숙소 없이 여행 먼저 생성(Case B)' 표기는 오기이며 본 문서는 제품 개요·시나리오 정본에 따라 Case A로 통일한다.

**요지**

**원문 수용 기준**

- [ ] 로그인/회원가입 화면에 소셜 4개 버튼과 '회원가입'(수단 비종속 라벨) 진입을 노출하고, '이미 계정이 있으신가요? 로그인' 링크로 전환할 수 있다. 앱은 로그인 필수이며 비로그인 진입은 초대·공유 딥링크 수신에 한정한다.
- [ ] 소셜 인증 성공 시 제공자 고유 ID(sub)로 신규 생성/기존 로그인을 분기하고, 동의 거부·취소 시 "로그인이 취소되었습니다" 안내로 로그인 화면 복귀, 제공자 오류 시 재시도 버튼을 표시한다.
- [ ] 이메일 가입은 인증 링크 방식(유효 24시간·재발송 분당 1회/일 5회 제한)이며, 인증 완료 전 '미인증' 상태로 약관 동의 이후 온보딩으로 진행시키지 않는다.
- [ ] 비밀번호는 8자 이상+영문/숫자 포함+유출 목록 검증을 통과해야 하며 해시로 저장한다.
- [ ] 이미 가입된 이메일 재가입 시 중복 계정을 만들지 않고 로그인/비밀번호 재설정을 유도하며 재설정 링크 경로를 제공한다.
- [ ] 동일 이메일이 소셜/이메일 양쪽에 확인되면 기존 수단으로 유도하고, 이메일 대조 불가(Apple 비공개 릴레이 등)에만 별도 계정 생성(1차 수동 연결 미제공, CS 처리).
- [ ] 모든 계정은 '여행자' 단일 유형으로 생성되며 다른 유형 선택지는 없다.

### US-E1-03 — 닉네임 자동 생성과 수정

- **유형**: Story
- **외부 Requirement ID**: `US-E1-03`
- **상위 Epic**: E1 가입·온보딩
- **Priority**: P0

- **초기 SP**: 5
- **의존**: US-E1-01; PR8 금칙어 사전
- **Source**: `docs/planning/user-stories.md:63-69`

**요지**

> 여행자로서 자동 생성 닉네임을 그대로 쓰거나 원할 때만 고치고 싶다 — 가입 직후 닉네임 때문에 막히지 않으려고.

**원문 수용 기준**

- [ ] 계정 생성 시 '형용사+여행명사+2자리 숫자' 패턴으로 중복 없는 닉네임을 자동 생성해 기본값으로 채우고 충돌 시 재추첨한다.
- [ ] 자동 생성값만으로 온보딩을 통과할 수 있으며 "닉네임은 나중에 설정에서 바꿀 수 있어요"를 명시한다.
- [ ] 닉네임은 2~20자로 제한하고 길이 위반·금칙어·중복 시 인라인 오류로 진행을 막는다. 금칙어 기본 사전은 가입 시점 검증부터 적용.
- [ ] 입력 닉네임이 중복이면 "이미 사용 중인 닉네임입니다"와 함께 대체 닉네임을 추천해 원탭 적용한다.

**Subtasks**

- [ ] `ST-E1-03-01` **[BE] 닉네임 자동 생성과 수정 서버·도메인 계약 구현** — 완료조건: Story의 상태·검증·오류 계약을 구현하고, “계정 생성 시 '형용사+여행명사+2자리 숫자' 패턴으로 중복 없는 닉네임을 자동 생성해 기본값으로 채우고 충돌 시 재추첨한다.”를 포함한 서버 단위·통합 테스트가 통과한다.

### US-E1-04 — 위치 권한 just-in-time 고지·요청

- **유형**: Story
- **외부 Requirement ID**: `US-E1-04`
- **상위 Epic**: E1 가입·온보딩
- **Priority**: P0
- **목표 Sprint**: S1
- **초기 SP**: 5
- **의존**: US-E1-17; 실제 OS 권한 발화는 U3 '내 주변'·U6 여행 중 진입
- **Source**: `docs/planning/user-stories.md:71-77`

**요지**

> 여행자로서 위치 권한을 실제 필요한 순간에 이유와 함께 요청받고 싶다 — 온보딩에서 불필요한 권한 요구 없이 진행하려고.

**원문 수용 기준**

- [ ] 온보딩 단계에서는 OS 위치 다이얼로그를 호출하지 않고, 위치가 실제 쓰이는 첫 맥락('내 주변 숙소 탐색'·'여행 중 실행' 첫 진입) 직전에 목적 프리프롬프트를 1회 보인 뒤 OS 다이얼로그를 호출한다.
- [ ] 허용 시 현재 위치를 숙소 탐색·동선 기준점에 활용한다.
- [ ] 거부 시 등록 숙소 위치 기준으로 동작하고 "현재 위치 기능은 설정에서 켤 수 있습니다"를 안내하며 거부만으로 온보딩을 중단하지 않는다.
