### US-E5-01 — 숙소 기준 날짜별 일정 생성

- **유형**: Story
- **상위 Epic**: `E5` — AI 일정 생성·확정
- **Priority**: `P0`
- **목표 Sprint**: `S5`
- **초기 SP**: `13`
- **의존**: U3 등록 숙소·M7, U4 CP2 여행 컨텍스트
- **Source**: [user-stories.md](../user-stories.md) · `US-E5-01`

**요지**: 여행자로서 등록 숙소 위치·체크인/아웃 날짜를 기준으로 여행 기간 전체의 날짜별 AI 일정을 생성하고 싶다 — 매일 어디서 출발·복귀하는지가 맞는 현실적 일정을 받으려고.

**원문 수용 기준**:
- [ ] 등록 숙소 위경도·체크인/아웃 날짜를 일정 생성 고정 입력값(출발점)으로 사용하고 체크인 당일~체크아웃 당일 각 날짜에 일자 일정을 1개씩 생성한다. 생성·재계획 연산은 서버에서 수행한다.
- [ ] 숙소 우선 생성은 등록 숙소가 없으면 버튼을 비활성화하고 "숙소를 먼저 등록하면 일정을 만들 수 있어요"를 안내한다. 단 **장소 우선(스토리 11 '숙소 나중 등록') 온램프는 1급 경로로 여행지 중심 무숙소 생성을 허용**한다.
- [ ] 같은 기간에 위치 다른 숙소가 여러 개면 각 숙소 체크인/아웃 날짜에 맞춰 날짜별 기준점을 자동 전환하고, 숙소 전환일은 출발=A숙소·복귀=B숙소 편도 동선으로 모델링한다.
- [ ] 숙소 지오코딩 실패 시 지도에서 위치를 직접 지정하도록 요청하고 지정 전까지 해당 숙소 기준 생성을 보류한다.
- [ ] 체크인/아웃 날짜 누락·역전 시 일정을 생성하지 않고 날짜 수정 화면으로 이동시킨다.
- [ ] 위치 권한 거부여도 등록 숙소 위치를 출발점으로 생성하며 현재 위치 필요 기능은 등록 숙소 위치로 대체하고 그 가정을 화면에 표기한다.

**Subtasks**:
- [ ] `ST-E5-01-01` `[BE]` 숙소·여행 컨텍스트를 읽어 일자·다중 거점·전환일을 생성하는 API를 구현하고 CP2 계약 테스트를 통과시킨다.
- [ ] `ST-E5-01-02` `[APP]` 등록 숙소 유무·지오코딩·날짜 오류·위치 권한 상태별 생성 CTA와 안내 화면을 구현한다.
- [ ] `ST-E5-01-03` `[DATA]` 일자별 거점 스냅샷과 생성 세션 저장 구조를 마이그레이션하고 날짜·좌표 무결성 제약을 검증한다.
- [ ] `ST-E5-01-04` `[QA]` 단일/다중 숙소, 전환일, 날짜 역전, 위치 권한 거부 시나리오의 API·UI 통합 테스트를 통과시킨다.

### US-E5-02 — 취향 기반 POI 선별·추천

- **유형**: Story
- **상위 Epic**: `E5` — AI 일정 생성·확정
- **Priority**: `P0`
- **목표 Sprint**: `S5`
- **초기 SP**: `8`
- **의존**: U1 취향, U3 M7 후보 풀, C1 LLM Gateway
- **Source**: [user-stories.md](../user-stories.md) · `US-E5-02`

**요지**: P2 민준으로서 여행 스타일·예산·동행·선호 활동·이동 방식·음식 취향을 입력하면 그에 맞는 장소를 추천받고 싶다 — 내 취향·형편에 맞는 곳들로 일정이 채워지도록.

**원문 수용 기준**:
- [ ] LLM이 사용자 자유 입력(취향 문장 포함)을 해석해 각 후보 POI에 선호 점수를 매기고 그 점수를 일정 최적화의 보상값으로 사용한다.
- [ ] LLM은 그라운딩된 후보 POI ID 목록에서만 선택(closed-set)해 실재 장소만 노출하고, 후보 목록 밖 POI는 구조적으로 포함될 수 없다.
- [ ] 예산은 솔버 하드 제약이 아니라 LLM 추천 소프트 가중치로 반영한다 — 카테고리별 무료/저/중/고 비용 추정으로 예산대에 맞는 POI를 우선 추천.
- [ ] 이동 방식(도보/자차 등)에 따라 장소 간 거리 기반 이동시간을 다르게 계산해 추천에 반영한다.
- [ ] 예산으로 채울 POI가 없으면 "예산을 높이거나 활동 수를 줄여보세요"를 안내하고 빈 일정 대신 무료/저비용 대안 POI를 우선 제시한다.
- [ ] 선호 조건이 너무 좁아 후보 0건이면 결과를 0으로 만든 조건을 표시하고 완화를 제안한다.

**Subtasks**:
- [ ] `ST-E5-02-01` `[AI]` 후보 ID만 반환하는 LLM 점수 스키마를 구현하고 목록 밖 ID가 거부되는 closed-set 테스트를 통과시킨다.
- [ ] `ST-E5-02-02` `[BE]` 취향·예산·이동 방식을 점수 입력으로 조합하고 후보 0건·저비용 대안 응답 계약을 구현한다.
- [ ] `ST-E5-02-03` `[APP]` 반영 취향, 저비용 대안, 조건 완화 제안을 빈 화면 없이 표시한다.
- [ ] `ST-E5-02-04` `[QA]` 프롬프트 인젝션·후보 0건·LLM 실패에서 실재 후보만 노출되는 회귀 테스트를 통과시킨다.

### US-E5-04 — 숙소·필수 방문지 고정 반영

- **유형**: Story
- **상위 Epic**: `E5` — AI 일정 생성·확정
- **Priority**: `P0`
- **목표 Sprint**: `S5`
- **초기 SP**: `8`
- **의존**: US-E5-03, U4 거점·필수 방문지
- **Source**: [user-stories.md](../user-stories.md) · `US-E5-04`

**요지**: P1 지유로서 꼭 가야 하는 장소를 미리 지정하면 일정이 그것을 중심으로 짜이길 원한다 — AI 추천 속에서도 내가 정한 우선순위가 절대 밀리지 않도록.

**원문 수용 기준**:
- [ ] 등록 숙소 체크인/아웃 시각과 시각 고정형 필수 방문지를 변경 불가 고정 블록으로 놓고 나머지 추천 POI를 충돌 없이 배치한다.
- [ ] 필수 방문지는 생성 전·후 어느 시점에든 지정할 수 있고 기본은 포함 고정형(반드시 포함, 시각·순서는 솔버 위임)이며 시각 지정은 선택이다. 화면 라벨은 사용자 언어('시간 정해두기'/'아무 때나 꼭 가기').
- [ ] 시각 고정형 지정 시 방문 날짜·시작 시각·예상 체류 시간을 입력받고 시각 미지정 시 "영업시간·동선에 맞춰 자동 배치됨"을 명시한다.

- **유형**: Story
- **상위 Epic**: `E8` — 여행 기록·회고
- **Priority**: `P1`
- **목표 Sprint**: `S8`

- **의존**: US-E8-08, US-E8-11
- **Source**: [user-stories.md](../user-stories.md) · `US-E8-14`

**요지**: 여행자로서 지난 여행을 월 캘린더에서 한눈에 보고 특정 여행으로 들어가고 싶다 — 시간 축으로 여행 기록을 쉽게 되짚으려고.

**원문 수용 기준**:
- [ ] 월 캘린더에 여행 기간(체크인~체크아웃 날짜)을 마킹하고 같은 달에 여러 여행이 있으면 모두 표시한다(여행 날짜 구간은 생성 단계에서 겹침이 차단되므로 서로 중첩되지 않는다).
- [ ] 캘린더 여행 마킹 또는 하단 '지난 여행' 목록에서 특정 여행을 선택해 그 여행의 날짜별 기록·회고(US-E8-08·US-E8-11)로 진입할 수 있다.
- [ ] 월을 이전/다음으로 이동해 과거 여행을 탐색할 수 있다.
- [ ] 캘린더는 '기록' 탭과 마이페이지(지난 여행) 양쪽에서 진입할 수 있다.
- [ ] 저장 여행 기록이 하나도 없으면 빈 캘린더 대신 '아직 기록된 여행이 없습니다' 안내와 새 여행 생성 진입점을 노출한다.

**Subtasks**:

- [ ] `ST-E8-14-02` `[APP]` 이전/다음 월, 여행 마킹·목록 선택, 기록/마이 양쪽 진입과 빈 상태를 구현한다.
