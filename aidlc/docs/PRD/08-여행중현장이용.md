# 여행 중 현장 이용

> 상위 문서: [PRD 인덱스](./README.md)

> 📍 본 섹션은 여행 중 사용자가 현장에서 일정을 **정상 실행**할 때의 화면을 정의한다 — '여행 중 일정 재계획(Plan-B)'와 구분된다(재계획이 아닌 실행 흐름). (방문 체크·기록의 저장 규칙은 '여행 기록/회고'를, 이동·길찾기 산출 기준은 'AI 일정 생성'을 정본으로 따른다.)

1. 사용자는 일정상의 장소에 도착하면 도착을 확인하고 방문을 시작·완료 처리할 수 있어야 한다.
   - 수용 기준:
     - 앱은 현재 위치가 다음 예정지 인근임을 감지하거나 사용자가 수동으로 '도착'을 표시하면, 방문 시작/완료 체크·사진·메모 진입점을 활성 일정 위에 노출해야 한다.
     - 방문 완료 표시는 활성 일정의 진행 상태(완료/진행 중/다음)에 반영되고, 실제 체류 시간 측정의 기준이 되어 'Plan-B' 스토리 2의 자동 트리거(체류 초과·이동 지연) 입력으로 사용되어야 한다.
     - 방문 완료 체크·사진·메모의 저장·기록 규칙은 '여행 기록/회고' 스토리 1·2를 정본으로 따른다(본 화면은 현장 진입점만 제공).
   - 예외:
     - 위치 권한이 없거나 GPS 정확도가 낮으면 자동 도착 감지를 보류하고 수동 체크인으로 대체해야 한다. ('여행 기록/회고' 스토리 3과 일관)

2. 사용자는 현재 방문 중(또는 예정)인 장소의 상세 정보를 현장에서 확인할 수 있어야 한다.
   - 수용 기준:
     - 앱은 현재 장소의 영업시간, 예상 체류 시간, (가용 시) 혼잡도, 주변 추천을 보여주고, 다음 일정까지의 여유 시간을 함께 표기해야 한다.
     - 예상 체류·혼잡도 등은 추정치임을 명시하고 산출·데이터 출처를 표기해야 한다. (영업시간 데이터 한계는 'AI 일정 생성' 스토리 3의 '영업시간 미확인' 처리와 일관)
   - 예외:
     - 장소 상세 데이터(영업시간·혼잡도)가 누락되면 빈칸 대신 "미확인"으로 표기해야 한다.

3. 사용자는 현재 위치에서 다음 예정지까지의 거리를 활성 일정 화면에서 바로 확인하고 외부 지도 앱으로 길찾기를 이어갈 수 있어야 한다. (앱은 자체 길찾기를 구현하지 않으므로 별도 길찾기 전용 화면을 두지 않는다.)
   - 수용 기준:
     - 앱은 별도 길찾기 전용 화면 없이, 활성 일정 화면의 '다음' 행/맥락 액션에 현재 위치 → 다음 예정지까지의 거리(거리 기반 추정)를 인라인으로 보여주되 차량/도보 예상 소요시간은 표시하지 않으며, 실제 단계별 길안내·소요시간(대중교통 노선·환승 포함)은 '외부 지도 앱으로 열기'로 위임해야 한다. 외부 지도앱으로 이탈했다가 앱으로 돌아오면, 앱은 현재 활성 일정·다음 행동(도착 체크인 등)으로 매끄럽게 복귀시켜 맥락을 잃지 않게 해야 한다(외부 이탈→복귀 핸드오프, '숙소 탐색' 스토리 5와 동일 패턴). (외부 위임 원칙 ADR-0012)
     - 이동 거리는 추정치임을 명시하고 'AI 일정 생성' 스토리 3·6의 산출 기준(장소 간 거리 기반 추정 — ADR-0009, 실시간 대중교통 경로 미제공)과 일관되게 표기해야 한다(소요시간은 표시하지 않음).
   - 예외:
     - 연결 가능한 외부 지도 앱이 없으면 웹 지도 길찾기로 폴백하고, 그것도 불가하면 앱이 산출한 거리 요약(직선/도로 거리·추정)을 최종 안내로 표시해야 한다(소요시간·수단별 경로는 표시하지 않음). ('AI 일정 생성' 스토리 6 폴백·ADR-0011과 일관)
