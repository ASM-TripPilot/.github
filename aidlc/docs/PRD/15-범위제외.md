# Out of Scope (범위 제외)

> 상위 문서: [PRD 인덱스](./README.md)

- **자체 숙소 예약 시스템 / 인앱 예약 확정**: 앱은 숙소를 탐색·저장만 하며, 실제 예약·확정은 외부 OTA 링크(딥링크)로 이동해 처리한다. 앱 내에서 객실을 점유·확정하지 않는다.
- **인앱 결제 / 자체 PG 정산**: 숙소 결제는 모두 외부 OTA에서 이루어진다. 앱은 어떤 형태의 인앱 결제 흐름·결제 수단 등록·정산도 제공하지 않는다.
- **내부 객실 재고 관리 / 중복 예약 방지 로직**: 앱은 객실 재고를 보유·동기화하지 않으며, 잔여 객실 수·가격은 외부 OTA의 표기를 따른다. (재고 정합성은 OTA 책임.)
- **숙박업주(호스트) 입점 / 숙소·객실 등록 / AI 리스팅 빌더**: 호스트 온보딩, 숙소 등록, 객실 타입·요금 설정, AI 리스팅 생성 기능을 제공하지 않는다.
- **호스트 예약 관리 / 호스트 재무 대시보드 / AI 경영(재무) 에이전트**: 호스트용 예약 인박스, 매출·정산·수익 대시보드, AI 경영·재무 분석 에이전트를 제공하지 않는다.
- **항공권 검색·예약·발권**: 항공 상품의 검색·예약·발권·요금 비교를 제공하지 않는다.
- **액티비티(투어·체험) 탐색·예약 기능 일체**: 액티비티 탐색·상세·외부 예약 연동·일정 고정 입력 등 액티비티 전용 기능을 제공하지 않는다. 여행 일정의 고정 블록은 등록 숙소와 사용자 지정 필수 방문지로 구성한다. (ADR-0005)
- **OTA 실시간 재고 동기화 / OTA 정산 자동화**: OTA와의 실시간 객실 재고 동기화, 정산·수수료 자동 정산 파이프라인을 구축하지 않는다.
- **PMS 완전 연동 / 채널매니저**: 숙박 관리 시스템(PMS) 연동, 채널매니저 기능을 제공하지 않는다.
- **환불·취소·분쟁 자동화**: 예약 환불·취소·분쟁 처리는 외부 OTA에서 처리하며, 앱은 이를 중개·대행하지 않는다.
- **숙소 평가 에이전트 심사 자동화**: 숙소 품질을 AI가 심사·승인하는 자동화 프로세스를 제공하지 않는다.
- **소셜 그래프 (팔로우 · 구독 · 스크랩 컬렉션 · 평판 점수) / 독립 게시판 · 포럼형 토론**: 팔로우·구독·스크랩(컬렉션 저장)·평판 점수 등 소셜 그래프 기능과, 일정과 무관한 스레드형 글쓰기·답글 게시(독립 게시판/포럼)는 제공하지 않는다. 단 **공개 일정에 대한 좋아요·댓글은 커뮤니티에 포함**한다(좋아요=토글·집계, 댓글=공개 일정에 달리는 평면 반응(대댓글·수정 없음)으로 한정, 신고·숨김·운영 검토 인프라 공유). 댓글은 일정에 종속된 반응이며 일정과 무관한 독립 게시판 글이 아니다. ('여행자 커뮤니티' 섹션 소셜 인터랙션 스토리 8·9 참조)
- **타인 공개 일정의 원본 직접 편집**: 커뮤니티에서 다른 사용자가 공개한 일정을 원본 그대로 직접 편집하는 것은 제공하지 않는다 — '내 여행으로 가져오기'(원본과 분리된 사본 복제)로만 활용한다('여행자 커뮤니티' 섹션). (내 여행에 동행자를 초대해 함께 실시간 편집하는 기능은 '동행 공동 편집' 섹션에서 지원하며 범위에 포함된다.)
- **다국어 지원**: 초기에는 한국어 단일 언어로 제공하며, 다국어 UI·콘텐츠를 제공하지 않는다.
- **해외(국외) 커버리지**: 초기 출시는 국내(대한민국)에 한정하며, 해외 도시는 커버하지 않는다(이후 단계적 확장).
- **항공 지연 자동 반영 / AI 자동 예약 변경·취소**: 항공 지연을 일정에 자동 반영하거나, AI가 사용자 대신 예약을 자동 변경·취소하지 않는다.
- **여행 영상 자동 생성**: 여행 기록·회고를 영상으로 자동 편집·생성하는 기능을 제공하지 않는다.

> **참고(Problem Statement과의 정합성)**: Problem Statement에서 "기존 OTA들은 숙소·항공·액티비티를 예약한다"는 서술은 *기존 OTA 일반*의 범위를 설명하는 맥락이며, TripPilot 자체의 범위가 아니다. TripPilot은 위 Out of Scope에 따라 항공 상품을 다루지 않는다.
