# Testing Decisions (테스트 결정)

> 상위 문서: [PRD 인덱스](./README.md)

### 좋은 테스트의 기준

- 테스트는 LLM이 만든 문장의 표현 품질이 아니라 **사용자에게 도달한 결과가 필수 조건을 충족했는지**를 검증한다. (예: 생성된 일정이 등록된 숙소 위치·체크인/아웃 날짜를 벗어나지 않았는지)
- 테스트는 단일 기능이 아니라 **'숙소 저장 → 외부 예약/저장 숙소 등록 → AI 일정 생성 → 여행 중 재계획'이 하나의 흐름으로 연결되는지**를 검증한다. 각 단계의 출력이 다음 단계의 입력으로 정상 전달되어야 한다.
- 테스트는 **등록된 숙소의 위치·체크인/아웃 날짜가 AI 일정 생성의 기준점(출발점)으로 반영되는지**를 검증한다. 등록 숙소가 바뀌면 일정의 지리적·시간적 기준점도 그에 맞게 갱신되어야 한다.
- 테스트는 **외부 OTA 제휴 링크 연결이 핵심 예약 경로로 정상 동작하는지**를 검증한다. 탐색·저장한 숙소에서 사용자가 외부 OTA 링크로 정확히 연결되어야 한다(올바른 숙소·날짜 파라미터 전달 포함).
- 테스트는 **재계획 결과가 남은 일정·이동 시간·고정 항목(등록 숙소 체크인/아웃, 외부 예약한 항목)과 충돌하지 않는지**를 검증한다.
- 테스트는 **추천 POI가 외부 장소 데이터에 그라운딩(RAG)되어, LLM이 지어낸 환각 POI나 폐업한 장소가 일정에서 제외되는지**를 검증한다.
- 테스트는 **사용자가 소셜 또는 이메일로 가입·로그인할 때 동일 이메일의 계정이 중복 생성되지 않는지(계정 충돌 시 단일 계정으로 통합), 이메일 미인증 계정이 활성 계정으로 취급되지 않는지**를 검증한다.
- 테스트는 **외부 API(OTA 링크/숙소 데이터/지도·경로/LLM)가 실패하거나 빈 결과를 반환할 때 앱이 fallback 흐름으로 복구하는지**를 검증한다. 사용자에게 빈 화면이나 오류만 남지 않아야 한다.

### 테스트 대상 모듈

1. **Saved Accommodation (외부 예약/저장 숙소 등록)** — 사용자가 외부에서 예약했거나 앱에서 저장한 숙소를 등록할 때, 숙소 위치와 체크인/아웃 날짜가 정상 저장되고 이 정보가 AI 일정 생성의 기준점(출발점)으로 정확히 반영되는지 검증한다. 등록 정보 수정 시 기준점이 갱신되는지도 검증한다.
2. **Accommodation Search (숙소 탐색·저장)** — 지역·필터 조건으로 숙소를 탐색할 수 있고, 결과 항목마다 외부 OTA 제휴 링크가 정상 노출·연결되는지 검증한다. 빈 결과·필터 무결과 시 안내가 노출되는지 검증한다.
3. **Itinerary Generation (AI 일정 생성)** — 등록 숙소 위치·날짜를 기준점으로 일정이 생성되고, 일정이 체크인/아웃 범위와 지리적 기준점을 벗어나지 않는지 검증한다.
4. **Itinerary Recalculation (일정 재계획)** — 여행 중 변경 발생 시 남은 일정·이동 시간·고정 항목(등록 숙소, 외부 예약 항목)과 충돌하지 않게 재계획되는지 검증한다.
5. **Plan-B Detection (Plan-B 재계획 트리거 감지)** — 재계획이 필요한 상황(지연·휴무·악천후 등)을 감지하고 재계획 흐름으로 연결하는지 검증한다.
6. **Affiliate Link (외부 OTA 제휴 링크 — 핵심 예약 경로)** — 숙소·날짜 등 파라미터가 올바르게 담긴 외부 OTA 링크로 연결되는지, 링크 생성 실패 시 fallback이 동작하는지를 핵심 예약 경로로서 검증한다.
7. **Travel Archive (여행 기록)** — 여행한 일정·등록 숙소·방문 항목이 기록으로 정상 보존·조회되는지 검증한다.
8. **AI Reflection/Summary (AI 회고·요약)** — 여행 기록을 기반으로 회고·요약이 생성되고, 실제 기록 데이터에 근거하는지(없는 내용을 지어내지 않는지) 검증한다.
9. **Place Data (POI 데이터 그라운딩/RAG)** — 추천 POI가 외부 장소 데이터에 그라운딩(RAG)되는지, LLM이 지어낸 환각 POI·폐업/존재하지 않는 장소가 일정에서 제외되는지 검증한다. 외부 장소 데이터가 없거나 조회 실패 시 해당 POI를 일정에 포함하지 않는지도 검증한다.
10. **Notification (Plan-B·일정·회고 알림)** — Plan-B/일정/회고 알림이 채널별 토글 설정대로 발송되고, 중복 억제 규칙(동일 트리거의 반복 알림 억제)대로 동작하는지 검증한다. 알림 권한 거부 시 인앱 표시로 대체되는지도 검증한다.
11. **Trip Creation / User Profile (여행 생성·사용자 취향)** — 숙소를 아직 등록하지 않은 상태에서도 여행 생성·일정 생성이 실패하지 않는지(us_tripcreate 스토리 2), 다도시 거점 여행에서 거점별 기준점이 처리되는지, 취향이 미설정인 사용자에게 중립 기본값으로 일정이 생성되는지(us_account 스토리 14)를 검증한다.

### 테스트 불포함

- 항공권의 검색·예약·발권.
- 액티비티(투어·체험) 탐색·예약 일체 (액티비티 기능은 범위에서 제외 — ADR-0005).
- 숙소의 자체(인앱) 예약·인앱 결제·내부 객실 재고 관리.
- 숙박업주(호스트) 사이드 기능 일체 — 호스트 입점, 숙소 등록, AI 리스팅 빌더, 호스트 예약 관리, 호스트 재무 대시보드, AI 경영(재무) 에이전트.
- 외부 OTA의 실시간 객실 재고 동기화, OTA 정산 자동화, 환불·취소·분쟁 자동화.
- 외부 OTA에서 발생하는 결제·예약 확정·취소 결과의 정확성 (앱은 링크 연결까지만 책임지며 OTA 내부 처리는 검증 범위 밖).
- 전 세계 모든 국가·도시에 대한 일정 정확도 보장.
- LLM 생성 문장의 표현 품질 절대 평가.
- AI에 의한 자동 예약 변경·취소.
