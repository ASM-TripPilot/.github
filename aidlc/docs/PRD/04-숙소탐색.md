# 숙소 탐색 / 저장 / 외부 예약 연동

> 상위 문서: [PRD 인덱스](./README.md)

> 📐 **데이터 모델 註**: 본 섹션의 정본 모델은 **"탐색·저장은 앱, 상세·예약·결제는 외부 OTA 딥링크"(teaser + 딥링크)** 다(업계 표준·합법). 데이터 종류별 취급: ① **정적 콘텐츠**(이름·위치·설명·시설·사진 URL·체크인아웃) = 글로벌 OTA 콘텐츠 API로 캐싱 적재 가능(갱신주기 준수) ② **정확 가격·재고** = 캐싱 금지, 표시 시점 라이브 호출(결과 목록 일괄 호출 금지 — 사용자가 '가격 보기'/딥링크로 정확 가격을 볼 때만). 단 **대표 가격대(가격 등급·구간 같은 정적 분류)** 는 콘텐츠 메타로 정적 표시·캐싱 가능(정확 1박 요금이 아니므로 가격 캐싱 금지 대상 아님) ③ **식별자** = place_id·OTA 숙소 ID는 영구, 좌표는 지도 API 약관 허용 범위 내 캐싱(국내 지도 API는 영구 캐싱 금지·실시간 호출 약관 준수) ④ **리뷰·평점** = 앱 내부에 표시하지 않고 "리뷰 보기 → 외부 OTA" 딥링크로 위임한다(원문·점수 모두). 국내 OTA(야놀자·여기어때 등)는 공개 데이터 API가 없어 **딥링크 연결만** 하고 데이터를 크롤링하지 않는다(부정경쟁 민사 리스크). **숙소 유형·편의시설**: 유형·시설은 가격과 분리된 정적 콘텐츠라 글로벌 OTA Content API·TourAPI로 확보·캐싱 가능하다 — 유형 필터는 전면 제공, 편의시설은 한국=TourAPI 즉시 확대·글로벌=계약 시 확대(스토리 2).

1. 사용자는 여행지를 입력해 숙소를 탐색할 수 있어야 하며, 체크인/체크아웃 날짜·인원은 탐색 단계에서 입력받지 않는다.
   - 수용 기준:
     - 여행지(또는 '내 주변')만 입력받으며, 탐색 단계에서는 체크인/체크아웃 날짜·인원을 입력받지 않는다. 여행지가 비어 있으면 탐색 버튼이 비활성 상태로 유지된다. 앱은 대표 가격대·딥링크로 안내하고 정확 가격·재고는 외부 OTA에서 확정하며, 일정 생성에 필요한 체크인/체크아웃 날짜는 숙소 등록(스토리 6)에서 받는다. (ADR-0002 '등록된 숙소=출발점')
     - 숙소 데이터 소스는 글로벌 OTA(Booking·Agoda·Expedia 등)의 인벤토리를 기본으로 하며, 국내 숙소는 글로벌 OTA의 한국 인벤토리 + 지도/장소 데이터(Google Places·공공 TourAPI)로 보강한다. 국내 OTA(야놀자·여기어때 등)는 공개 데이터 API가 없으므로 딥링크 연결 대상으로만 다루고 데이터를 크롤링하지 않는다.
     - 검색을 실행하면 앱은 외부 OTA 조회를 시작하고, 결과 카드에 숙소명·대표 가격대(가격 구간·등급)·위치를 표기한다. **목록 카드는 정적 대표 가격대를 기본으로 하고, 정확한 1박 가격은 사용자가 '가격 보기'를 누르거나 딥링크로 나갈 때 라이브 호출한다 — 결과 목록 전체에 대해 정확 가격을 일괄 라이브 호출하지 않는다(비용·지연·OTA 약관). 정확 가격 값 자체는 캐싱하지 않는다**(가격 캐시 금지 — OTA 약관).
     - 결과가 N건 이상이면 페이지네이션 또는 무한스크롤로 추가 결과를 노출한다.
   - 예외: 위치 권한이 거부된 상태에서 '내 주변' 검색을 시도하면, 앱은 등록된 숙소 또는 여행지 중심 좌표로 대체해 조회한다.
2. 사용자는 검색 결과를 가격대, 숙소 유형(호텔·게스트하우스·펜션·모텔·호스텔·아파트·B&B·한옥 등), 편의시설(주차·조식·와이파이 등), 거점 후보 지역까지의 거리 기준으로 필터링·정렬할 수 있어야 한다.
   - 수용 기준:
     - **숙소 유형 필터는 전면 제공한다** — 유형은 가격과 분리된 정적 콘텐츠라 글로벌 OTA Content API(Expedia 유형 코드·Booking·Google)와 한국관광공사 TourAPI(cat3; '펜션' 등 한국형 유형은 TourAPI를 정본으로)로 다중 분류를 확보·캐싱할 수 있다. 소스별 코드 차이는 Place Data 모듈에서 표준화한다(ADR-0009).
     - **편의시설(amenities) 필터는 조건부 확대한다** — 한국 숙소는 TourAPI로 핵심 3종(주차·조식·와이파이) 너머 확장하되 필드 채움률을 검증해 채워지는 항목만 노출하고, 글로벌 숙소는 풀 Content API(계약 파트너) 확보 시 OTA 표준 카테고리 단위로 확대하며 미확보 시 핵심 3종으로 폴백한다. 편의시설 표시는 가격 비교 화면이 아닌 곳에 배치한다(Booking 약관). 가격대 필터는 대표 가격대(구간) 기준으로 적용하고, 정확 가격에 의존하는 **가격순 정렬은 라이브 전수 호출을 전제하므로 제공하지 않고 '대표 가격대순'으로 대체한다**. 라이브 가격 호출은 사용자가 명시적으로 정확 가격을 볼 때만 발생시킨다.
     - 필터를 적용한 뒤 결과가 0건이면, 앱은 어떤 필터가 결과를 0으로 만들었는지 표기하고 해당 필터의 완화를 제안한다.
3. 사용자는 숙소 상세 화면에서 위치(지도 핀), 사진, 가격대, 편의시설, 체크인/체크아웃 시각을 확인할 수 있어야 한다.
   - 수용 기준:
     - 리뷰·평점은 앱 내부에 표시하지 않고, "리뷰·평점 보기 → 외부 OTA" 딥링크로 위임한다(원문·점수 모두). 외부 OTA의 리뷰·평점을 앱 내부 화면에 표시하지 않는다.
     - 숙소 상세는 위치·사진·가격대·편의시설·체크인/체크아웃 시각 같은 정적 콘텐츠 위주로 구성하고, 평가 정보가 필요하면 "외부 OTA에서 확인" 진입을 노출한다.
   - 예외: 영업시간 등 데이터가 누락되면, 앱은 해당 필드를 빈칸으로 두지 않고 "미확인"으로 표기한다.
4. 사용자는 마음에 드는 숙소를 위시리스트(저장 목록)에 저장하고, 저장 목록에서 다시 꺼내 등록·예약으로 이어갈 수 있어야 한다.
   - 수용 기준:
     - 저장은 로그인 사용자 계정에 귀속되어 기기를 바꿔도 유지된다.
     - 저장 목록의 각 항목은 숙소명·대표 사진·가격대·위치를 카드로 표시한다.
     - 사용자는 저장 항목을 삭제하거나, 메모(예: "역에서 가까움")를 덧붙일 수 있다.
   - 예외: 저장 시점과 조회 시점의 가격·재고가 다를 수 있음을 안내하고, 가격은 "외부 OTA 기준, 변동 가능"으로 표기한다.
5. 사용자는 숙소 상세·저장 화면에서 [외부 OTA에서 예약하기]를 눌러 해당 OTA 딥링크(외부 브라우저 또는 인앱 웹뷰)로 이동해 실제 예약·결제를 진행할 수 있어야 한다.
   - 수용 기준:
     - 외부 이동 직전, 앱은 "예약·결제는 외부 OTA에서 진행되며 TripPilot은 제휴(어필리에이트) 수수료를 받을 수 있다"는 제휴 링크 안내·수수료 고지를 표시한다.
     - 앱은 자체 예약·객실 선택·예약 확정·인앱 결제를 제공하지 않으며, 모든 결제 흐름은 외부 OTA로 위임한다.
     - 동일 숙소에 연결된 OTA가 여러 개면, 앱은 OTA명과 표시 가격을 함께 보여주고 사용자가 어느 링크로 나갈지 선택하게 한다.
     - 사용자가 외부 OTA로 이탈했다가 앱으로 돌아오면(포그라운드 복귀), 앱은 "방금 본 [숙소명], 예약하셨나요? → 거점으로 등록하기" 복귀 핸드오프 카드를 노출해 곧바로 등록(스토리 6 빠른 등록)으로 연결해야 한다. 사용자는 이를 무시하고 계속 탐색할 수 있어야 한다(강제 아님).
   - 예외: OTA 링크가 깨졌거나(404·만료) 응답하지 않으면 "지금 이 OTA로 연결할 수 없어요"를 표시하고, 가능하면 같은 숙소의 다른 OTA 링크 또는 숙소 장소 검색으로 우회 경로를 제시한다.
6. 사용자는 외부 OTA(또는 다른 경로)에서 이미 예약한 숙소를 앱에 '등록'할 수 있어야 한다. 등록 정보는 숙소(장소), 체크인/체크아웃 날짜, 인원이며, OTA 예약번호와 결제 금액은 선택 입력이다.
   - 수용 기준:
     - 등록된 숙소(장소 좌표)와 체크인/체크아웃 날짜가 AI 일정 생성의 기준점(거점·기간)이 된다. (ADR-0002의 '예약=출발점'을 '등록된 숙소=출발점'으로 계승)
     - 등록 경로는 두 가지다 — (a) **앱의 외부 OTA 제휴 링크를 통한 예약이 확인되면**(외부 OTA 복귀 핸드오프 스토리 5) 앱이 해당 숙소를 자동으로 등록 후보로 제시해 1탭 등록하게 하고, (b) 앱을 거치지 않은 예약·기존 예약은 위치·체크인/아웃 날짜를 **수동 입력**해 등록한다. **예약번호·메일 연동을 통한 자동 인식은 제공하지 않는다.**
     - 숙소 검색 결과·저장 목록에서 곧바로 [예약했어요 / 등록] 동작으로 등록을 시작할 수 있고, 이때 숙소명·위치·가격대는 자동 채워진다.
     - 등록을 마치면 앱은 해당 숙소를 거점으로 한 [AI 일정 생성하기] 진입점을 즉시 노출한다.
     - 사용자는 등록한 숙소의 날짜·인원·예약번호·금액을 나중에 수정하거나 등록을 취소할 수 있다.
   - 예외: 체크아웃이 체크인보다 빠르거나 날짜가 비어 있는 등 등록 정보가 불완전하면 저장을 막고 어떤 항목이 빠졌는지 인라인으로 표시한다. 위치(좌표)가 확정되지 않은 등록은 "지도에서 위치 확인" 단계를 요구한다.
7. 사용자는 일자별로 거점 숙소를 다르게 등록할 수 있어야 한다(예: 다도시 이동 여행에서 도시별 숙소).
   - 수용 기준:
     - 등록된 여러 숙소는 날짜 구간이 겹치지 않게 검증되며, 각 구간의 일정은 해당 구간 거점을 기준으로 생성된다.
   - 예외: 두 숙소의 날짜 구간이 겹치면 "같은 날짜에 거점이 둘이에요"를 표시하고 구간 조정을 요구한다.
8. 사용자는 OTA 탐색 결과에 의존하지 않고, 지도 장소 검색·예약 링크 붙여넣기·지도 핀 지정으로 숙소를 직접 등록할 수 있어야 한다. (이미 외부에서 예약을 마친 숙소를 빠르게 거점으로 등록하는 주 경로)
   - 수용 기준:
     - 사용자는 지도/장소 검색(숙소명·주소; Place Data 모듈이 연동한 지도 API 기반, 예: Google Places 등 — 벤더는 ADR-0009에 따라 단일 API에 종속되지 않는다)으로 실제 숙소 장소를 찾아 위치·좌표를 자동 확보하고 거점으로 지정할 수 있다. 이 경로는 앱 내 OTA 숙소 검색 결과(스토리 1)에 해당 숙소가 없어도 독립적으로 동작해야 한다.
     - 검색 결과에서 숙소를 선택하면 숙소명·주소·좌표가 등록 폼에 자동 채워지고, 사용자는 체크인/체크아웃 날짜·인원만 입력하면 등록이 완료되어야 한다. (이미 외부 OTA에서 예약을 마친 숙소를 이 경로로 빠르게 등록한다.)
     - 사용자가 OTA 예약 링크/URL을 붙여넣으면, 앱은 가능한 범위에서 숙소명·위치를 추출해 등록 폼에 미리 채운다.
     - 추출이 안 되거나 장소가 검색되지 않으면, 사용자가 숙소명과 지도 핀 위치를 직접 지정해 등록을 완료할 수 있다.
   - 예외:
     - 지도 검색에서 동명·유사 숙소가 여러 건 반환되면, 앱은 후보 목록과 각 후보의 지도 핀을 함께 보여주고 사용자가 정확한 한 곳을 선택해 좌표를 확정하게 해야 한다.
     - 붙여넣은 링크가 숙소 페이지가 아니거나 위치를 특정할 수 없으면 "위치를 찾지 못했어요. 지도에서 직접 지정해 주세요"로 안내하고 수동 핀 지정 흐름으로 넘긴다.
     - 지도/장소 검색 API가 응답하지 않으면(타임아웃·오류) 앱은 침묵 실패하지 않고 지도 핀 직접 지정 흐름으로 폴백해야 한다. (ADR-0011)
9. 사용자는 등록·저장된 숙소 목록에서 숙소명, 위치, 체크인/체크아웃 일정, 인원, (등록 시 입력한 경우) OTA명·예약번호·금액, 그리고 거점 사용 여부를 확인할 수 있어야 한다.
    - 수용 기준:
      - 이 목록은 앱은 예약 진행/확정 상태를 추적하지 않으며(예약·결제는 외부 OTA에서 처리), 어디까지나 사용자가 입력한 외부 예약 기록과 거점 지정 상태만 보여준다.
    - 예외: 금액·예약번호 등 선택 항목이 비어 있으면 빈칸 대신 "미입력"으로 표시하고 입력을 강요하지 않는다.
10. 앱은 검색 조건에 맞는 숙소가 없으면 빈 화면 대신 "조건에 맞는 숙소가 없어요"와 함께 날짜·지역·필터 완화 같은 다음 행동을 제안해야 한다.
    - 예외: 특정 지역이 OTA 제휴 미지원이면 "이 지역은 아직 제휴 숙소가 없어요. 숙소를 직접 등록하면 일정을 만들어 드려요"로 안내하고 수동 등록(8번) 경로로 연결한다.
11. 앱은 외부 OTA 가격·목록을 모으는 동안 "숙소를 모으는 중" 로딩 상태를 표시하고, 일부 OTA가 응답하지 않아도 나머지 결과로 목록을 구성하며 침묵 실패하지 않아야 한다.
    - 수용 기준:
      - 일부 공급자 장애 시 "일부 가격이 빠졌을 수 있어요"를 표기하고, 가능한 결과는 즉시 보여준다.
    - 예외: 모든 OTA 조회가 실패하면 "지금 숙소 정보를 불러올 수 없어요"와 재시도 버튼을 제공하고, 사용자가 원하면 숙소 수동 등록(8번)으로 우회할 수 있게 한다.
