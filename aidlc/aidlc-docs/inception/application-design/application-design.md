## 핵심 설계 결정 요약

1. **컴포넌트 경계** (AD-1): PRD 모듈 17 + M18 Trip Execution(Δ7) + 공통 C1 LLM Gateway·C2 Solver Engine·C3 Moderation — requirements.md 결정과 1:1 추적
2. **모듈 내부** (AD-2): api/application/domain/infra 레이어드 통일, 외부 API 6종(카카오·TMap·기상청·TourAPI·LLM·FCM)은 포트 인터페이스 뒤 격리 — "외부 API 1종 = 소유 모듈 1곳 = 어댑터 포트 1벌", 벤더 교체가 매트릭스에 무영향
3. **모듈 간 통신** (AD-3): 공개 퍼사드 동기 호출 + 도메인 이벤트(트랜잭셔널 아웃박스, at-least-once, 구독자 멱등) — 동기 상호 참조 0건(순환 부재 위상 정렬 증명), 호출자 TX 참여 예외는 2건만(changelog 원자성 G132·확정 스냅샷 D13)
4. **상태 머신 정본 3종**: 여행(계획중→확정→진행중→종료, D19·D21), 일정(초안→편집중→확정 스냅샷 동결→current 분리, D14·D20), 방문(다음→도착확인대기→진행중→완료/스킵, D23) — components.md §3이 정본
5. **후속 게이트 성립**: M15~M17은 1차 모듈 퍼사드만 소비(역의존 S·E 0건) — ADR-0016 분리 출시 구조 보장
6. **유닛 시퀀스** (UW-1~4): U1 기반·계정 → U2 앱 셸 → U3 숙소·장소 → U4 여행 → U5 일정 생성 → U6 실행·Plan-B → U7 기록·회고 → U8 알림·설정 (+후속 U9~U11), 단일 배포·모노레포

## 확장 규칙 반영 지점

- **SECURITY**: 전 퍼사드 인증·소유권 검증 공통 규약(08), 보안 로직 M1·C1 격리(11), LLM 서버 재조회 경계(D31), 법정 로그 append-only 분리 저장(14·N2)
- **RESILIENCY**: 외부 의존 전부 타임아웃+폴백+사용자 고지 3요소(10 — services.md §0.1 표), 워크로드 중요도는 requirements.md §6.3, RTO/RPO·백업은 Infrastructure Design에서 구체화(12)
- **PBT**: 솔버 하드 제약(영업시간·이동 부등식·고정 블록 불변)·상태 머신 3종·changelog diff 재구성 왕복이 속성 테스트 1급 대상 — 유닛별 Functional Design PBT-01에서 식별
