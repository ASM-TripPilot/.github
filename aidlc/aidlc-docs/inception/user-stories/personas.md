# TripPilot 페르소나 (Personas)

> **생성일**: 2026-07-04 · **구성 방식**: 취향 축 기반 3원형 (story-generation-plan.md Question 4 = A)
> TripPilot의 계정 유형은 '여행자' 단일이다(PRD 03, ADR). 아래 3원형은 계정 유형이 아니라 **온보딩 취향 7종(여행 스타일·페이스·예산·동행·활동·이동·음식)의 대표 조합**으로, 개인화·Plan-B·기록 기능이 사용자에 따라 어떻게 달리 동작해야 하는지의 관점 차이를 드러내기 위한 설계·테스트 기준점이다. 스토리의 "As a 여행자"는 3원형 모두에 적용되며, 특정 원형 관점이 중요한 스토리만 페르소나 이름을 사용한다.
> 보조 액터로 **운영자**(어드민)를 정의한다 (D35).

---

## P1. 지유 — 꼼꼼 계획형

| 항목 | 내용 |
|---|---|
| 프로필 | 29세 직장인, 연 3~4회 국내 여행, 여행 2~3주 전부터 준비 |
| 취향 축 | 페이스: 빡빡하게(5곳+) · 스타일: 관광+미식 · 이동: 대중교통 · 예산: 중간(전체 총액 직접 입력) |
| 목표 | 시간 낭비 없는 검증된 동선 — 영업시간·이동시간이 어긋나지 않는 일정 |
| 불편점 | 블로그·지도·예약 앱을 오가며 직접 검증하는 피로, 계획이 어긋났을 때의 연쇄 붕괴 |
| TripPilot 사용 방식 | 숙소를 먼저 등록하고 필수 방문지를 여러 곳 지정, '같이 고르기'로 슬롯을 직접 통제, 시각 고정형 필수 방문지 활용, 일정 확정 후 D-day 알림 의존 |
| 대표 검증 관점 | 하드 제약(영업시간·이동시간·고정 블록) 무위반, 확정 해제→재확정 흐름(D20), 필수 방문지 한도(G40), 시각 고정 충돌 처리 |

## P2. 민준 — 균형 실속형

| 항목 | 내용 |
|---|---|
| 프로필 | 35세, 가족(아동 동반) 여행 중심, 연 1~2회, 준비 시간 부족 |
| 취향 축 | 페이스: 균형있게(3~4곳) · 스타일: 휴양+자연 · 이동: 렌터카 · 동행: 가족 · 예산: 구간 선택(중간) |
| 목표 | 최소 입력으로 무난하고 실패 없는 일정 — AI에게 맡기되 결과는 신뢰 가능해야 |
| 불편점 | 가족 조건(아이 동반·이동 최소화)을 반영한 추천 부재, 예산 초과 |
| TripPilot 사용 방식 | 온보딩 취향 일부 건너뛰기, '완전 AI 자동'으로 초안 생성 후 마음에 안 드는 슬롯만 교체(LOCK), 예산은 소프트 가중치로 반영(G47), 숙소 미등록 상태에서 일정 먼저 생성하는 온램프 사용 |
| 대표 검증 관점 | 중립 기본값 동작(취향 미설정 시 일정 생성 무실패), 재생성 시 LOCK 보존(G46), 전체 총액 예산 파생 계산(D26), 다중 숙소·가족 동행 필터 |

## P3. 하람 — 즉흥 유연형

| 항목 | 내용 |
|---|---|
| 프로필 | 24세 대학원생, 혼자 또는 친구와 즉흥 주말 여행, 당일 변경 잦음 |
| 취향 축 | 페이스: 느긋하게(1~2곳) · 스타일: 카페+야경 · 이동: 도보+대중교통 · 예산: 저가 |
| 목표 | 현장에서의 유연함 — 계획은 뼈대만, 상황(날씨·기분·체력)에 따라 바로 재구성 |
| 불편점 | 갑작스런 비·휴무에 대안을 급히 검색하는 스트레스, 기록을 남기고 싶지만 정리가 귀찮음 |
| TripPilot 사용 방식 | 여행 중 Plan-B 수동/자동 트리거의 주 사용자, 휴식 모드·즉석 방문 추가, GPS 기록 옵트인으로 자동 발자취, AI 회고 초안 의존, 자정 초과 야경 활동(D29) |
| 대표 검증 관점 | Plan-B 10초 응답(D38)·당일 잔여 재정렬(C10), 트리거 알림 피로 억제(G58), 오프라인 기록 입력 큐(Δ6), 방문 확인 프롬프트(D23) |
