## 2. 프로젝트 방향·범위

### 2.1 프로젝트 목표 (D01)

**실서비스 출시**가 1차 목표다. 따라서:
- 위치정보법 신고, OTA 제휴 계약 등 **비개발 선결 과제도 요구사항으로 추적**한다 (§8).
- 코드·아키텍처·보안·복원력은 프로덕션 품질을 기준으로 한다 (§3 확장 규칙 전체 강제와 정합).

- 여행자 커뮤니티 (10-여행자커뮤니티.md, M15) — 단, 공개 스냅샷 모델(D16)·기본 비공개 원칙·모더레이션 인프라 요건은 데이터 모델에 선반영
- 동행 공동 편집 (11-동행공동편집.md, M17, ADR-0016) — 단, 동기화·잠금 아키텍처(D30)는 1차 설계에서 확장 가능하게 결정

**범위 제외** (15-범위제외.md 정본 그대로): 자체 예약·인앱 결제·객실 재고·호스트(B2B)·항공·액티비티·PMS/채널매니저·환불 분쟁·소셜 그래프(팔로우 등)·독립 게시판·항공 지연 자동 반영·AI 자동 예약 변경·여행 영상 생성. 초기 출시는 **국내(대한민국) 한정·한국어 단일**.

## 3. 확장 규칙 적용 (Extension Configuration)

| 확장 | 적용 | 근거 답변 |
|---|---|---|
| Security Baseline (SECURITY-01~15) | **전체 강제 (blocking)** | CQ1=A |
| Resiliency Baseline (RESILIENCY-01~15) | **적용** — 설계 시점 방향성 모범 사례 | Q6=A |
| Property-Based Testing (PBT-01~10) | **전체 강제 (blocking)** | Q7=A |

**RTO/RPO·DR 전략 (CQ4=A, RESILIENCY-02)**: RTO/RPO 목표 **시간 단위**, DR 전략 **Backup & Restore**. 데이터는 자동 백업으로 보호하고, 재해 시 IaC 재배포 + 백업 복원으로 복구한다. 리전 토폴로지는 **단일 리전 + 다중 가용영역(Multi-AZ)** (RESILIENCY-08 — RTO/RPO A안과 정합, Infrastructure Design에서 최종 확인). 이 목표는 Application Design·NFR·Infrastructure Design 전 단계로 전파한다.

이후 모든 단계는 적용 확장 규칙의 검증 기준을 차단 제약으로 평가하고, 단계 완료 메시지에 컴플라이언스 요약을 포함한다.

### 5.3 신규 기능 요구사항 (PRD 미기재)

**N1. 연령 확인 (D33)** — 회원가입 흐름에 생년월일 입력 또는 '만 14세 이상입니다' 확인을 필수 단계로 추가한다. 만 14세 미만은 가입을 차단하고 사유를 안내한다. 소셜 가입 경로에도 동일 적용한다.

**N2. 위치정보법 법정 의무 (D34)** —
- 약관 동의 화면에 '위치기반서비스 이용약관' **별도 필수 동의** 항목 추가 (이용약관·개인정보 처리방침과 분리 체크)
- GPS 여행 기록 보관(09 스토리 3)은 **별도 옵트인 동의** — 철회·탈퇴 시 해당 위치 데이터 즉시 파기
- **위치정보 수집·이용·제공 사실 확인자료(로그) 테이블**을 데이터 모델에 포함 — append-only, 최소 6개월 보존, 애플리케이션 역할은 자기 로그 삭제 권한 없음 (SECURITY-14와 정합)
- 위치 동의 상태는 3층(OS 권한 × 앱 내 법정 동의 × GPS 기록 옵트인)으로 관리하고, 조합별 기능 동작 매트릭스를 Functional Design에서 명문화한다 `[G182]`

**N3. 약관 개정 재동의 `[G21,G183]`** — 약관 버전에 '재동의 필요' 플래그를 두고, 중대 변경(불리한 변경·수집 항목 확대)은 스플래시 분기에서 재동의 화면을 강제하며, 경미 변경은 인앱 공지로 처리한다.

**N4. 강제 업데이트 게이트 `[C6]`** — 스플래시에서 서버 최소 지원 버전을 확인하고, 미달 시 강제 업데이트 화면으로 분기한다. 서버 응답 계약에 최소 버전 필드를 포함한다.

**N5. 고객 지원·정책 문서 재열람 `[C7]`** — 설정에 이용약관·개인정보처리방침·위치기반서비스 약관·오픈소스 라이선스 재열람과 이메일 문의 링크를 제공한다 (앱스토어 심사·위치정보법 상시 열람 요건).

**N6. 여행 제목 `[C2]`** — 여행 생성 시 제목을 선택 입력받고, 미입력 시 '{여행지} N박M일' 형식으로 자동 생성한다. 언제든 수정 가능하며 금칙어 검증을 적용한다.

**N7. Trip Execution 모듈 (M18) `[G173]`** — 08-여행중현장이용의 기능(활성 일정 허브 상태, 도착 확인 프롬프트, 방문 시작/완료 상태 전이, 실제 체류 시간 측정)을 소유하는 모듈을 신설한다. M9 Plan-B Detection에 체류·이동 이벤트를, M12 Travel Archive에 actual 기록을 공급한다.

**N8. 마케팅 수신 동의 관리 `[G184]`** — 온보딩에서 마케팅 수신 동의(선택)를 수집하고 설정에 철회 토글을 제공한다. 1차 출시에서 마케팅 알림 발송은 하지 않는다(발송 기능은 후속).

### 6.2 가용성·복원력 (Resiliency Baseline)

- **백업**: 관리형 DB 자동 백업(+PITR 활성화 권장 — 목표 상회 충족), 사진 오브젝트 스토리지 버저닝, 백업 암호화, 보존 기간·복원 검증 절차는 Infrastructure Design에서 확정 (RESILIENCY-12) `[G147]`
- **의존성 격리**: 모든 외부 호출(지도·OTA·날씨·LLM·FCM) 명시적 타임아웃 + 서킷 브레이커 + 우아한 성능 저하 — ADR-0011 '침묵 실패 금지'의 인프라적 구현 (RESILIENCY-10)
- **오토스케일링·쿼터**: 컴퓨트 오토스케일 min/max 한도, 외부 API rate-limit·쿼터 문서화 및 80% 알람 (RESILIENCY-09)

### 6.3 워크로드 중요도 분류 (RESILIENCY-01)

| 컴포넌트 | 중요도 | 불가용 시 영향 |

### 6.4 보안 (Security Baseline 전체 강제)

- **인증** (SECURITY-12, D36): 액세스 토큰 1시간 + 리프레시 90일 회전, 다기기 허용. 토큰은 OS 보안 저장소(Keychain/Keystore) 보관. 비밀번호 8자 이상 + 유출 비밀번호 목록 검증, 적응형 해시(bcrypt/argon2) 저장 `[G22]`. 로그인 브루트포스 방어(점진 지연 또는 잠금). 어드민 계정 MFA 지원.
- **접근 제어** (SECURITY-08): 전 엔드포인트 deny-by-default 인증 필수(D22 로그인 필수와 정합), 리소스 ID 참조 시 소유권 검증(IDOR 방지), 공동 편집 권한(소유자/편집자/뷰어)은 서버 측 강제.
- **LLM 경계** (SECURITY-11, D31): LLM 컨텍스트 주입은 서버 재조회 방식 — 타 사용자 비공개 데이터·내부 지표(제휴 수수료 등)가 구조적으로 LLM 입력에 미포함. LLM 호출 rate-limit(사용자별 상한) 적용.
- **입력 검증** (SECURITY-05): 전 API 스키마 검증, 파라미터라이즈드 쿼리, 문자열 최대 길이, 요청 크기 제한. UGC(닉네임·제목·메모)는 금칙어+새니타이즈.
- **로깅·알림** (SECURITY-03·14): 구조화 로깅(타임스탬프·상관 ID·레벨), 중앙 집중 로그, PII·토큰 로그 금지, 인증 실패·권한 위반 알림, 로그 90일+ 보존, 위치정보 법정 로그는 append-only 분리(N2).
- **공급망** (SECURITY-10): 잠금 파일 커밋, 의존성 취약점 스캔 CI 포함, 프로덕션 이미지 버전 고정, SBOM 생성.
- 암호화(SECURITY-01)·네트워크(SECURITY-07)·헤더(SECURITY-04)·하드닝(SECURITY-09)·무결성(SECURITY-13)·예외 처리(SECURITY-15)는 NFR Design·Infrastructure Design·Code Generation 단계에서 검증 기준으로 적용한다.

### 6.5 법률·개인정보

- 위치정보법: §5.3-N2 전면 반영 + **위치기반서비스사업 신고**는 출시 선결 과제(§8)
- 개인정보보호법: 만 14세 미만 차단(N1), 계정 삭제 30일 유예+연쇄 삭제·익명화(D18), 데이터 내보내기(§7 기본값), 약관 동의 증적 보존
- 국외 이전: 국외 LLM 사용 시 개인정보처리방침에 국외 이전·처리위탁 고지, LLM 전송 필드는 목적 최소화 `[G181]`
- 지도 API 약관: 영구 캐싱 금지·실시간 호출·출처 표기를 데이터 모델 차원 반영(D13) — 사용자 확정 데이터 스냅샷의 법무 확인은 선결 과제(§8)
- OTA: 국내 OTA 크롤링 금지, 리뷰·평점 미표시, 제휴 수수료 고지(광고·제휴 표시 규제)

### 6.6 테스트·품질 (D37 + PBT 전체 강제)

- **계층 분리**: PR CI에서 LLM·외부 API는 모킹(어댑터 계약 인터페이스 + fake), 솔버·하드 제약 검증은 실코드 테스트. 실 LLM 회귀 평가는 릴리스 파이프라인에서만.
- **CI 게이트** `[G114]`: 하드 제약 검증(숙소 기준점·충돌 무배치·POI 그라운딩·계정 무결성) 테스트는 100% 통과 필수(머지 차단), 그 외는 비차단 리포트로 시작.
- **POI 그라운딩 판정** `[G115]`: LLM은 후보 ID 목록에서만 선택하는 closed-set 설계 — 그라운딩 실패가 구조적으로 불가능하게 한다.
- **PBT**: 솔버 하드 제약(영업시간 내 배치·이동시간 부등식·고정 블록 불변)은 속성 기반 테스트의 1급 대상. 직렬화 왕복·상태 머신(여행·일정·방문 상태)에 PBT 적용. 프레임워크: Kotest(백엔드)·fast-check(클라이언트). 시드 로깅·수축(shrinking) 필수 (PBT-08). 각 유닛의 속성 식별은 Functional Design에서 수행 (PBT-01).
- **트리거 재현** `[G116]`: Plan-B 판정 로직은 clock·외부 데이터 주입 가능한 순수 함수로 분리해 결정적 테스트.
- **E2E** `[G118]`: API 레벨 시나리오(숙소 저장→등록→일정 생성→재계획)를 CI 필수로, UI E2E는 핵심 해피패스 1~2개.

### 6.7 관측성 `[G144]` (RESILIENCY-05·SECURITY-14)

구조화 로그 + APM/크래시 리포팅 + 외부 API 실패율 대시보드. 침묵 실패 금지(ADR-0011)의 서버 관측: 외부 API 실패·LLM 오류·솔버 검증 실패는 전부 계측 대상. 헬스체크 엔드포인트(shallow+deep)는 설계 단계 요건.

### 6.8 규모 가정 `[G142]`

초기 출시(국내 한정): **DAU 1천 / 동시 일정 생성 10건 / 지역당 POI 후보 5천** (MVP 검증 규모). 오토스케일 한도·비용 상한 알람과 함께 Infrastructure Design의 입력으로 사용한다.

## 9. 후속 단계로 이관된 결정

아래는 요구사항이 아닌 설계 단계 결정으로, 해당 단계에서 확정 질문을 제시한다.

- **NFR Design**: 변경 관리 프로세스(RESILIENCY-03), CI/CD 도구·롤백 메커니즘·배포 스타일(RESILIENCY-04), 인시던트 대응 프로세스(RESILIENCY-15), 복원력 테스트 접근(RESILIENCY-14)
- **Infrastructure Design**: 클라우드 벤더·관리형 서비스 선정, 백업 보존 기간·복원 검증 절차, 리전 토폴로지 최종 확인
- **Functional Design**: 위치 동의 3층 조합별 동작 매트릭스(G182), 각 유닛 PBT 속성 식별(PBT-01), 일정·여행·방문 상태 머신 상세 전이도

## 10. 요구사항 요약

TripPilot은 **'등록된 숙소를 출발점으로 AI가 실행 가능한 일정을 만들고, 여행 중 변수에 Plan-B로 대응하며, 기록·회고로 이어지는' B2C 여행자 슈퍼앱**이다. 1차 출시는 핵심 여정(온보딩→숙소→여행 생성→AI 일정→여행 중 실행→Plan-B→기록/회고→알림/마이) + 국내 한정 + 실서비스 품질이며, 커뮤니티·공동편집·AI 어시스턴트는 데이터 모델 여지만 확보한 채 후속 게이트로 분리한다. 아키텍처는 RN(Expo)+Spring Boot(Kotlin)+PostgreSQL 모듈러 모놀리스이고, 보안·복원력·속성 기반 테스트 기준선이 전 단계 차단 제약으로 적용된다.
