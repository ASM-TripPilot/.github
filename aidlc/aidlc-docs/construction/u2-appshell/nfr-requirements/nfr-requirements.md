## 0. U2 NFR 프로필 — "클라이언트 셸"의 NFR 본질

U2에는 신규 인프라·신규 서버 워크로드가 사실상 없다. U2 NFR의 본질은 **클라이언트 진입 경험의 품질**이다:

1. **첫 인상 지연** — 스플래시 진입 분기가 얼마나 빨리·확실하게 목적지로 도달하는가(US-E2-01, G5).
2. **내비게이션 연속성** — 5탭 셸의 상태 보존·전환 지연(US-E2-03, G6·D38).
3. **부분 실패 내성** — 홈 대시보드가 후행 유닛 데이터 부재·부분 장애에도 "공사 중" 인상 없이 동작(US-E2-02, 침묵 실패 금지).
4. **게이트 안전성** — 강제 업데이트 게이트 오구성이 전 사용자 차단으로 번지지 않는가(US-E2-06, N4).

서버측은 **부트스트랩 API 1종 + 홈 대시보드 집약 조회 1종**(server/app 소유, BFF성)뿐이며, 인증·무상태·관측성·복원력의 서버 규약은 **U1 전역 정본을 그대로 상속**한다.

## 1. 성능 (D38, §6.1)

### 1.1 스플래시 부트스트랩 3초 예산 — 단일 왕복 집계 (US-E2-01·06, G5, N4)

§6.1이 U2에 직접 부과하는 목표는 **일반 화면 전환 300ms**와 **스플래시 세션 검증 3초(G5)** 둘이다. G5의 3초는 서버 성능 목표가 아니라 **클라이언트의 최종 대기 한계(fail-policy 경계)**다(U1 nfr §1.3 상속). U2는 스플래시가 확인해야 할 3가지 상태 — 세션 유효성·최소 지원 버전·재동의 필요 — 를 **단일 부트스트랩 왕복 1회로 병합 집계**한다(순차 3왕복 금지 — 3초 예산을 왕복 수로 잠식하지 않기 위함).

- **NFR-U2-PERF-01 (부트스트랩 집계 계약)**: 부트스트랩 응답은 세 필드를 한 번에 반환한다 — `minSupportedVersion`(항상), `sessionState`(토큰 제시 시), `reconsentRequired`(토큰 제시 시). 서버 처리 목표는 U1 정본(p50 100ms·p95 500ms, U1 nfr §1.2)을 상속하며 U2는 이를 재산정하지 않는다. 무상태 검증(PAT-PERF-01) + 단일 질의로 구성되어 p95 500ms 내 완결.
- **NFR-U2-PERF-02 (3초 클라이언트 예산 배분)**: 스플래시 시작 → 부트스트랩 왕복(TLS+RTT+서버 p95 ≈ 통상 1초 미만) → 분기 결정 함수 실행(순수 함수·µs 수준) → 목적지 전환. **3초 경과 시 응답 미도착이면 G5 폴백 발동**(§2.1) — 3초는 콜드 스타트·네트워크 열화·부분 장애에서만 도달하는 예외 경로여야 한다.
- **NFR-U2-PERF-03 (분기 우선순위 결정성)**: 분기 결정은 **버전 게이트 > 재동의 > 세션 > 온보딩 잔여 > 홈** 고정 우선순위의 순수 함수다(US-E2-06 하드 제약·PBT 대상, §5). 어떤 입력 조합에도 정확히 1개 목적지를 산출하며, 이 함수의 실행 비용은 성능 예산에서 무시 가능하다(네트워크가 지배).

### 1.2 탭 전환 300ms · 상태 보존 (US-E2-03, G6, D38)

- **NFR-U2-PERF-04 (탭 전환 지연)**: 하단 5탭 간 전환은 **300ms 화면 전환 예산(D38) 내** 완료한다. 탭 루트는 클라이언트 로컬 상태이므로 전환 자체는 네트워크 비의존 — 예산 초과 요인은 탭 최초 진입 시의 데이터 로딩뿐이며, 이는 스켈레톤·부분 표시(§1.3)로 전환 체감과 분리한다.
- **NFR-U2-PERF-05 (탭 상태 메모리 보존)**: 각 탭의 이전 화면 상태(스크롤·하위 스택)는 **세션 내 메모리 보존**(앱 재시작 시 초기화 — G6), 현재 탭 재탭 시 스크롤 탑. 보존은 각 탭 내비게이션 스택 인스턴스를 언마운트하지 않고 유지하는 것으로 달성(§5 PAT-U2-03) — 왕복 재조회로 상태를 복원하지 않는다(전환 지연·불필요 트래픽 방지).

### 1.3 홈 카드 병렬 로딩 · 부분 실패 허용 (US-E2-02)

- **NFR-U2-PERF-06 (병렬 로딩)**: 홈 대시보드의 독립 카드(여행 카드=U4·인기 장소=U3·추억=U7 공급, 활성 일정=U6, 최종 통합=U8)는 **상호 독립 병렬 로딩**한다. 한 카드의 지연·실패가 다른 카드의 표시를 차단하지 않는다.
- **NFR-U2-PERF-07 (부분 실패 허용 — 침묵 실패 금지)**: 홈 집약 조회는 **가용 카드만 담은 부분 응답**을 허용한다(US-E2-02: "일부 실패해도 가용한 카드만 우선 표시"). 실패 카드는 빈칸이 아니라 재시도 가능한 오류 상태 또는 유도 문구로 표시한다(스켈레톤 → 성공/오류/빈 상태). 후행 유닛 미구현 카드는 **미노출 원칙**(unit-of-work.md U2 리스크 완화 — "공사 중" 인상 차단).

## 2. 가용성·복원력 (§6.2·§6.3, RESILIENCY-10)

### 2.1 부트스트랩 API 경로 — U1 Auth 의존 + 오프라인 폴백 (G5)

부트스트랩 API의 세션 검증 부분은 **M1 Auth(Critical, §6.3)에 의존**한다 — 세션 상태 판정이 곧 Auth 판정이다. 따라서 부트스트랩 경로의 가용성 바닥은 U1이 결정하며(U1 nfr §3.1 상속), U2는 **클라이언트측 폴백 절반**을 소유한다(U1 nfr §1.3: "API 계약·타임아웃 정책의 서버측 절반은 U1, 실제 스플래시 분기 UI는 U2").

- **NFR-U2-AV-01 (오프라인/열화 스플래시 폴백 — G5 정본)**: 부트스트랩 3초 타임아웃 또는 네트워크 불가 시 —
  - **로컬 토큰 미만료** → **홈 진입 + 백그라운드 재검증**(온라인 복구 시). 재검증에서 세션 무효(401+사유 코드, U1 계약)가 판명되면 로그인으로 유도.
  - **로컬 토큰 만료** → 로그인 분기.
  - **최소 지원 버전 확인 불가** → **페일오픈**(G5 타임아웃 정책 준용, US-E2-06) — 확인 불가만으로 강제 업데이트 차단을 걸지 않는다(오구성·확인 실패가 전 사용자 차단으로 번지는 것을 방지, unit-of-work.md U2 리스크 완화).
- **NFR-U2-AV-02 (침묵 실패 금지)**: 검증 지연 중 스플래시는 진행 표시를 유지하고(US-E2-01), 폴백 발동·백그라운드 재검증 실패는 **계측 이벤트로 보고**한다(§4 관측성 — 3초 타임아웃 발동률 메트릭, U1 PAT-OBS-03 클라이언트 행 상속).
- **NFR-U2-AV-03 (홈 집약 조회 복원력)**: 홈 조회는 **RESILIENCY-10 상속** — 타임아웃·부분 응답(NFR-U2-PERF-07)·재시도 안내. 홈은 여행·일정 조회(Critical)를 포함하나, U2 시점에는 카드 슬롯이 대부분 빈 상태이므로 실질 부하 낮음. 외부 의존 0(unit-of-work.md U2 — 외부 연동 없음)이라 서킷 브레이커 대상 신규 없음.

### 2.2 오프라인 조회 미보장 경계 (D24/Δ6)

- **NFR-U2-AV-04 (오프라인 미보장 명시)**: 여행 중 일정 조회를 포함한 U2 내비게이션은 **온라인 전제**이며 오프라인 조회를 보장하지 않는다(D24/Δ6, US-E2-03). 오프라인 시 빈 화면·침묵 실패가 아니라 **오류·재시도 안내**를 표시한다. 캐시 정책 경계는 §3.2.

## 3. 클라이언트 NFR (D37 클라이언트 계층 + G6·D24)

### 3.1 탭 상태 보존 수명 (G6)

- **NFR-U2-CL-01 (보존 범위·수명)**: 탭 상태 보존은 **세션(앱 프로세스 생존) 범위**다 — 앱 재시작 시 초기화(G6). 보존 대상은 각 탭의 내비게이션 스택·스크롤 위치이며, 서버 상태(TanStack Query 캐시)는 별도 수명(§3.2). 메모리 보존이므로 영속 저장소(SecureStore·AsyncStorage) 미사용 — 재시작 초기화가 요구 동작 그 자체(과보존 금지).

### 3.2 캐시 정책 경계 (D24)

- **NFR-U2-CL-02 (캐시 = 최적화, 가용성 아님)**: U2의 서버 상태 캐시(홈 카드·부트스트랩 결과)는 **지연 최적화일 뿐 오프라인 가용성 요소가 아니다**(U1 NFR-U1-AV-03의 클라이언트 대칭). 캐시 미스·만료는 온라인 재조회로 충족하며, 오프라인에서 캐시로 "조회 보장"을 흉내 내지 않는다(D24 미보장 확정과 정합 — 낡은 데이터로 인한 오판 방지). 부트스트랩 결과는 스플래시 세션 한정(재기동 시 재조회), 홈 카드는 짧은 stale 허용 + 포그라운드 복귀 시 재검증.
- **NFR-U2-CL-03 (딥링크 라우팅 무결성)**: 알림·딥링크 진입은 대상 탭 활성화 + 그 탭 스택 푸시(G7, §5 PAT-U2-06). 비로그인 딥링크 진입은 **초대·공유 딥링크 수신에 한정**(D22, US-E2-01) — 그 외 딥링크는 인증 게이트를 통과해야 대상 화면에 도달(라우팅 가드, §5 참조). 이는 클라이언트 계약이며 서버 소유권 검증(§4)과 이중.

## 4. 보안 (Security Baseline 상속 — §6.4)

U2는 SECURITY 규칙의 **신규 1차 구현 지점이 아니다** — 전역 보안 설정(deny-by-default·헤더·에러 핸들러·구조화 로깅·PII 마스킹)은 U1 스캐폴드가 소유하고 U2는 상속한다. U2 증분은 부트스트랩·홈 두 신규 엔드포인트의 규약 적용뿐이다.

- **NFR-U2-SEC-01 (부트스트랩·홈도 deny-by-default 상속)**: 홈 대시보드 집약 조회는 **인증 필수 + 토큰 주체(accountId) 소유권 스코핑**(SECURITY-08, NFR-U1-SEC-16·18 상속) — 타 사용자 여행·일정 데이터 미혼입. 부트스트랩의 세션·재동의 부분도 토큰 검증 후에만 응답(무상태 검증, PAT-PERF-01 상속).
- **NFR-U2-SEC-02 (최소 버전 체크 = 비인증 허용 경계)**: 부트스트랩 응답 중 **`minSupportedVersion` 필드만 비인증 공개**다 — 로그아웃·만료·구버전 사용자도 강제 업데이트 게이트를 받아야 하므로 토큰 없이 접근 가능해야 한다(NFR-U1-SEC-16이 이미 예약한 공개 화이트리스트: "부트스트랩 확인 중 비세션 부분(최소 지원 버전)"). **경계 명시**: 동일 엔드포인트가 (a) 비인증 부분(최소 버전)과 (b) 인증 부분(세션·재동의)을 함께 반환하되, (b) 필드는 **유효 토큰 제시 시에만 채워지고** 무토큰/무효 토큰에는 미노출 — 비인증 경로로 세션 정보가 누출되지 않는다.
- **NFR-U2-SEC-03 (딥링크 진입 검증)**: 딥링크로 진입한 화면도 대상 리소스 접근 시 서버 소유권 검증을 거친다(SECURITY-08 — IDOR 원천 차단, NFR-U1-SEC-18 상속). 비로그인 딥링크 진입 한정(초대·공유, D22)은 클라이언트 라우팅 가드(§3.3)와 서버 검증의 이중 방어이며, 클라이언트 가드만으로 접근 통제를 신뢰하지 않는다.
- **NFR-U2-SEC-04 (강제 업데이트 게이트 하드 제약)**: 최소 버전 미달 앱은 **로그인 포함 전 서비스 진입 차단**(US-E2-06 DoD 하드 제약). 이 차단은 클라이언트 게이트가 1차이나, 미달 버전이 부트스트랩 외 API를 호출할 경우의 서버측 거부 정책은 U2 증분 없이 표준 401/앱 버전 헤더 관측으로 충분(과설계 금지 — DAU 1천·게이트 목적은 UX 안내이지 침입 차단이 아님).

## 5. 테스트·품질 (D37 + PBT — §6.6)

- **NFR-U2-QA-01 (스플래시 분기 PBT — 1급 대상)**: 스플래시 분기 결정 함수(세션 상태 × 약관 버전 × 앱 버전 → 목적지)를 **순수 함수로 분리**해 fast-check 속성 테스트(US-E2-01·06 DoD): (a) 어떤 입력 조합에도 정확히 1개 목적지, (b) 우선순위 불변(**버전 게이트 > 재동의 > 세션 > 온보딩 잔여 > 홈**), (c) 타임아웃 폴백 결정성(로컬 토큰 만료 여부 → 홈/로그인). 시드 로깅·수축 필수(PBT-08).
- **NFR-U2-QA-02 (탭 상태 보존 왕복 PBT)**: 탭 상태 보존·복원 왕복 속성(G6 — 임의 탭 방문 순서·스크롤 위치에 대해 재탭 시 스크롤 탑·타 탭 상태 불변).
- **NFR-U2-QA-03 (하드 제약 테스트)**: 강제 업데이트 미달 버전의 서비스 진입 차단(로그인 포함), 비로그인 진입이 딥링크 수신 한정임을 검증하는 라우팅 가드 테스트 100%(US-E2-03·06 DoD).
- **NFR-U2-QA-04 (계약 포인트)**: U1 세션·재동의 API **소비자 측 계약 테스트** + 홈 카드 슬롯별 공급 계약(후행 유닛이 채울 응답 필드)을 스키마로 고정(unit-of-work.md U2 DoD).

## 6. 규모 (§6.8 G142 — U2 신규 산정 없음)

- **NFR-U2-SC-01**: 부트스트랩 호출량은 U1이 이미 산정했다 — "앱 실행마다 1회 × DAU 1천 × 일 5회 = 5,000건/일 ≈ 피크 0.28 RPS"(U1 nfr §2.1). U2는 이 부트스트랩 API의 **클라이언트 소비자**이며 서버 산정을 재수행하지 않는다. 홈 집약 조회는 부트스트랩과 유사 빈도(앱 진입당 1회)로, SI §1.2의 태스크 2개가 흡수(추가 산정 불요, G142).
- **NFR-U2-SC-02**: U2 신규 컴퓨트·DB·오토스케일 결정 **없음** — 부트스트랩·홈 조회는 U1 app 모놀리스에 흡수, 유일한 신규 저장은 앱 구성(최소 지원 버전) 1건 내외(unit-of-work.md U2 산출물, infrastructure-design.md §2).

## 7. 미결 항목 — 없음 (전역 결정 상속)

**U2는 신규 인프라·벤더 종속 미결 항목이 없다.** U1이 PD-1~9를 전부 종결(U1 infrastructure-design.md §0)했고, U2는 그 종결 결과와 전역 결정 GD-1~7을 상속하기 때문이다.

| 잠재 결정 항목 | U2 처리 | 근거 |
|---|---|---|
| 클라우드·CI/CD·배포·롤백 | GD-1~7 상속(재질문 없음) | u1-foundation-nfr-design-plan.md §2 |
| 컴퓨트·DB·시크릿·로그·알림 스택 | SI 정본 상속(재정의 금지) | SI §1·3·6·8, U1 PD-1~9 종결 |
| 부트스트랩 API 서버 스택 | U1 app 모듈 재사용(신규 0) | tech-stack-decisions.md, unit-of-work.md U2 |
| 최소 지원 버전 저장 방식 | 원격 구성 — 앱 config vs 서버 파라미터(권고는 infrastructure-design.md §2) | 유닛 증분(경량) |
| 내비게이션 라이브러리 | Expo Router(U1 tech-stack §3.8 기확정) — U2는 탭·딥링크 구체화 | tech-stack-decisions.md |

**대기 항목 0 — 차단 소견 없음.** U2의 유일한 유닛 로컬 결정(최소 버전 저장 방식)은 신규 인프라가 아니라 기존 메커니즘 선택이며, infrastructure-design.md §2에서 권고로 종결한다.
