# 1. U2 신규 패턴

## PAT-U2-01 스플래시 부트스트랩 단일 왕복 집계 `[U2 — 서버 절반 U1 상속]`

**문제**: 스플래시는 목적지 결정 전 3가지 상태(세션 유효성·최소 지원 버전·재동의 필요)를 알아야 한다(US-E2-01·06, US-E1-18). 이를 순차 3왕복으로 확인하면 3초 클라이언트 예산(G5)을 왕복 수만큼 잠식하고, 부분 실패 시 어느 왕복까지 성공했는지에 따라 분기 상태가 파편화된다.

**적용(설계)**:

- **1왕복 집계 계약**: `server/app` 부트스트랩 엔드포인트가 세 필드를 한 응답으로 반환(NFR-U2-PERF-01) — `minSupportedVersion`(항상), `sessionState`(토큰 제시 시), `reconsentRequired`(토큰 제시 시). 세션·재동의는 U1 M1·동의 도메인 재조회, 최소 버전은 앱 구성 조회. 서버 목표 p95 500ms는 U1 정본 상속(재산정 없음).
- **분기 결정은 순수 함수**: 응답(또는 폴백 신호) → 목적지를 **고정 우선순위 순수 함수**로 계산 — `버전 게이트 > 재동의 > 세션 무효 > 온보딩 잔여 > 홈`(NFR-U2-PERF-03). 부수효과(네비게이션) 없이 목적지만 산출해 PBT 가능(§NFR-U2-QA-01).
- **게이트 유지·해제**: expo-splash-screen을 부트스트랩 완료 또는 3초 타임아웃까지 유지, 결정 후 목적지 전환과 함께 hide(무한 스플래시 금지, U2-C5).

```text
[콜드 스타트]
  → 네이티브 스플래시 유지(preventAutoHide)
  → 로컬 토큰 읽기(expo-secure-store, 1회) → 세션 스토어 로드
  → 부트스트랩 1왕복 (minVersion + session + reconsent)  ──3초 타임아웃──┐
  → 분기 결정 순수 함수(우선순위 고정)                                     │
  → 목적지 전환 + splash hide                             (타임아웃 시 PAT-U2-02 폴백)
```

**U2 적용 지점**: `features/home` 진입 전 스플래시 컨트롤러, 분기 결정 함수(`shared/` 순수 모듈), 부트스트랩 쿼리(TanStack Query, U2-C4). 서버 계약은 `server/app`(U2-S1).

**검증 기준**: NFR-U2-PERF-01·03 — 부트스트랩 응답이 3필드 집계임을 계약 테스트로 확인(U1 소비자 측 계약, NFR-U2-QA-04), 분기 함수가 어떤 입력에도 정확히 1개 목적지·우선순위 불변(PBT NFR-U2-QA-01). 서버 p95 500ms는 U1 정본 검증 상속.

## PAT-U2-02 클라이언트 오프라인 우선 세션 폴백 `[U2]`

**문제**: 부트스트랩이 3초 안에 오지 않는 상황(콜드 스타트·네트워크 열화·부분 장애)에서 스플래시가 무한 대기하거나 침묵 실패하면, 앱은 "켜지지 않는 앱"이 된다. 반대로 무조건 로그인으로 보내면 정상 세션 사용자를 매번 재로그인시키는 마찰이 생긴다.

**적용(설계)**: G5를 설계 정본으로 승격 — **로컬 토큰 상태 기준 폴백**(NFR-U2-AV-01):

| 3초 타임아웃/오프라인 시 상태 | 폴백 동작 | 고지·계측 |
|---|---|---|
| 로컬 토큰 미만료 | **홈 진입 + 백그라운드 재검증** — 온라인 복구 시 부트스트랩 재시도 | 진행 표시 유지(침묵 실패 금지), 재검증 실패(401+사유) 시 로그인 유도 |
| 로컬 토큰 만료 | 로그인 분기 | — |
| 최소 버전 확인 불가 | **페일오픈**(강제 업데이트 미발동) | 확인 실패 계측 — 오구성·확인 실패가 전 사용자 차단으로 번지지 않게(U2 리스크 완화) |

- **설계 원칙**: 게이트(강제 업데이트)는 **확인된 미달일 때만** 닫힌다 — 확인 불가는 페일오픈(가용성 우선). 이는 U1 PAT-SEC-04의 "보안 게이트 fail-closed"와 반대 방향인데, 근거는 **강제 업데이트가 보안 게이트가 아니라 UX 안내 게이트**이기 때문(NFR-U2-SEC-04 — 침입 차단 아님). 방향을 항목별로 명시해 애매함을 남기지 않는다(U1 PAT-RES-03 원칙 상속).
- **백그라운드 재검증**: 홈 진입 후 부트스트랩을 백그라운드 재시도해 세션 무효가 판명되면 로그인으로 유도(U1 nfr §1.3 서버 계약: 401+사유 코드).

**U2 적용 지점**: 스플래시 폴백 컨트롤러, 백그라운드 재검증 훅(`shared/api`), 최소 버전 페일오픈 로직(PAT-U2-05와 결합).

**검증 기준**: NFR-U2-AV-01·02 — 3초 타임아웃 주입 시 토큰 미만료→홈·만료→로그인·버전확인불가→페일오픈 3분기 테스트(PBT NFR-U2-QA-01의 타임아웃 결정성 항), 폴백·재검증 실패의 계측 이벤트 발행 확인(PAT-OBS-03 상속). RESILIENCY-10(우아한 저하) — 침묵 실패 없음 통합 테스트.

## PAT-U2-03 탭별 독립 내비 스택 + 상태 보존 `[U2]`

**문제**: 5탭 내비게이션에서 탭 전환마다 화면을 재마운트·재조회하면 전환 지연(300ms 예산 초과)과 불필요 트래픽이 발생하고, 사용자의 스크롤·하위 화면 컨텍스트가 유실된다(G6 위반). 동시에 규칙을 화면마다 코드로 관리하면 후행 유닛에서 파편화된다(unit-of-work.md U2 리스크).

**적용(설계)**: Expo Router 레이아웃 규약(U2-C2)으로 상태 보존을 **구조로 강제**:

- **탭별 독립 스택**: `app/(tabs)/_layout.tsx`의 각 탭이 자체 `Stack` — 탭 전환 시 비활성 탭을 **언마운트하지 않고 유지**(세션 내 메모리 보존, G6). 재탭 시 스크롤 탑 훅(G6 후단).
- **보존 수명 = 세션**: 메모리 보존이며 앱 재시작 시 초기화(NFR-U2-CL-01) — 영속 저장 미사용(과보존 금지, D24 오프라인 미보장과 정합).
- **몰입 화면 탭바 숨김**: `(tabs)` 밖 스택 그룹으로 온보딩·입력 폼·생성·실행 화면을 배치해 탭바가 파일 구조로 숨겨짐(US-E2-04 — `screenOptions` 산발 제어 아님). 탭바는 `shared/ui` 단일 공용 컴포넌트 + 린트 규칙으로 일관성 강제(unit-of-work.md U2 리스크 완화).

**U2 적용 지점**: `app/(tabs)/_layout.tsx`(5탭), 각 탭 스택, 몰입 화면 그룹, `shared/ui` 탭바.

**검증 기준**: G6/NFR-U2-PERF-04·05 — 탭 전환 300ms 내·상태 보존/복원 왕복 PBT(NFR-U2-QA-02: 임의 방문 순서·스크롤 위치에 재탭 시 스크롤 탑·타 탭 상태 불변), 탭바 숨김 규칙 스냅샷 테스트(US-E2-04 화면군).

## PAT-U2-04 홈 카드 병렬 로딩·부분 실패 격리 `[U2]`

**문제**: 홈 대시보드는 여러 후행 유닛(U3·U4·U6·U7·U8)이 공급하는 카드의 집합이다. 한 카드의 지연·실패가 전체 홈을 막거나, 실패를 빈칸으로 방치하면 "공사 중"·"고장 난 앱" 인상을 준다(US-E2-02, unit-of-work.md U2 리스크).

**적용(설계)**:

- **카드별 독립 병렬 쿼리**: 홈 카드는 카드별 TanStack Query(U2-C4) — 상호 독립 병렬 로딩(NFR-U2-PERF-06). 서버 홈 집약 조회는 **가용 카드만 담은 부분 응답**(BFF성, 침묵 실패 금지)을 허용(NFR-U2-PERF-07).
- **상태 3분기 표시**: 각 카드는 스켈레톤(로딩) → 성공/오류(재시도 가능)/빈 상태(유도 문구). **후행 유닛 미구현 카드는 미노출**(unit-of-work.md U2 "미구현 카드는 미노출 원칙") — 빈 카드로 존재감을 남기지 않음.
- **첫 사용자 빈 홈**: 여행·숙소 0건이면 빈 대시보드 대신 첫 행동 유도(여행 만들기·예약 없이 일정·장소 저장 온램프, US-E2-02·05).

**U2 적용 지점**: `features/home` 카드 컨테이너·카드별 쿼리 훅, `server/app` 홈 집약 조회(부분 응답 스키마 = 슬롯 계약, NFR-U2-QA-04).

**검증 기준**: NFR-U2-PERF-06·07/RESILIENCY-10 — 한 카드 실패 주입 시 타 카드 정상 표시·실패 카드 오류 상태(빈칸 아님) 테스트, 데이터 0건 시 빈 홈 대신 첫 행동 유도 노출 테스트(US-E2-02 DoD), 홈 슬롯 스키마 공급 계약 고정(NFR-U2-QA-04).

## PAT-U2-05 강제 업데이트 게이트 `[U2]`

**문제**: 지원 종료 구버전 앱은 깨진 화면·오류를 낳는다(N4). 그러나 게이트를 무조건 닫는 방향으로 설계하면 **오구성·확인 실패 1건이 전 사용자 차단(서비스 전면 장애급)**으로 번진다(unit-of-work.md U2 리스크).

**적용(설계)**:

- **비교 순수 함수**: 현재 앱 버전(expo-constants, U2-C6) vs 서버 `minSupportedVersion`(부트스트랩) semver 비교 — 클라이언트 순수 함수(PBT 대상). 미달이면 강제 업데이트 화면 분기, **로그인 포함 전 서비스 진입 차단**(US-E2-06 하드 제약, NFR-U2-SEC-04) + 스토어 이동 액션(아웃바운드 링크).
- **페일오픈 안전판**: 최소 버전 **확인 불가 시 게이트 미발동**(PAT-U2-02와 결합, G5 타임아웃 정책 준용) — 게이트는 UX 안내이지 침입 차단이 아니므로 fail-open이 정합(NFR-U2-SEC-04).
- **운영 안전**: `minSupportedVersion`은 **원격 구성 + 변경 이력**(infrastructure-design.md §2) — 오구성 시 재배포 없이 되돌릴 수 있게(unit-of-work.md U2 리스크 완화). 게이트 결정 함수는 PBT로 회귀 방지.

**U2 적용 지점**: 스플래시 분기 함수 내 버전 게이트(최우선순위, PAT-U2-01), 강제 업데이트 화면(`features/home` 또는 전용 라우트), 원격 구성 조회(`server/app`).

**검증 기준**: N4/NFR-U2-SEC-04 — 미달 버전의 서비스 진입(로그인 포함) 차단 하드 제약 테스트 100%(NFR-U2-QA-03), 확인 불가 시 페일오픈 테스트, 버전 게이트가 분기 우선순위 최상위임을 PBT로 확인(NFR-U2-QA-01).

## PAT-U2-06 딥링크 라우팅 — 탭 활성화 + 스택 푸시 `[U2]`

**문제**: 알림·딥링크 진입이 대상 화면으로 가되 내비게이션 컨텍스트(어느 탭·스택)를 무시하면 뒤로가기·탭 상태가 깨진다(G7). 동시에 딥링크는 인증 우회 진입 경로가 될 수 있다(비로그인 진입 통제, D22).

**적용(설계)**:

- **경로=URL 규약**: Expo Router linking(U2-C3)으로 딥링크 → **대상 화면이 속한 탭 활성화 + 그 탭 스택에 푸시**(G7)를 선언적으로 처리. 경로 추가 = 라우트 추가라 후행 유닛 화면이 자동 커버.
- **인증 가드 라우트**: 비로그인 딥링크 진입은 **초대·공유 딥링크에 한정**(D22, US-E2-01) — 그 외는 레이아웃 레벨 인증 가드가 로그인으로 리다이렉트 후 원 목적지 복귀. 클라이언트 가드는 1차이며, 대상 리소스 접근은 **서버 소유권 검증**(SECURITY-08, PAT-SEC-01 상속)이 2차(NFR-U2-SEC-03 — 클라이언트 가드만 신뢰 금지).

**U2 적용 지점**: `app` linking config, 레이아웃 인증 가드, 알림 탭 매핑(U8 알림이 소비할 딥링크 규약을 U2가 고정).

**검증 기준**: G7/NFR-U2-SEC-03 — 딥링크 진입 시 대상 탭 활성화+스택 푸시 테스트, 비로그인 진입이 초대·공유 딥링크 한정임을 검증하는 라우팅 가드 테스트 100%(NFR-U2-QA-03), 서버 소유권 검증 이중 방어 통합 테스트(PAT-SEC-01 상속).
