## 0. U6 NFR 프로필 — "여행 중에 쓰는 앱"의 NFR 본질

U6의 NFR 본질은 **현장 변수(날씨·휴무·지연·체류초과)에 10초 내 유효한 대안으로 대응하되, 허위 알림을 만들지 않고, 포그라운드·저전력·법정 동의 안에서 위치를 다루는 것**이다. 세 축이 핵심이고 서로 긴장한다:

1. **실시간성/성능(핵심)** — D38의 **Plan-B 대안 제시 10초**를 services.md §S2.4 단계 예산으로 분해하고, 예산 초과가 예상되는 LLM(사유 해석)은 **타임아웃을 예산보다 짧게(1.5초)** 잡아 폴백 전환 시간을 확보한다(§0.3). 도착 감지 반응·거리 갱신 등 현장 상호작용 체감을 함께 지킨다.
2. **복원력(핵심)** — 외부·비결정 의존(기상청·지도)은 전부 폴백 필수(§6.3). **기상청 무응답 → 트리거 침묵**(허위 알림 금지 D27 — 없는 위험을 알리지 않는다), **재계획 외부 API 실패 → 수동 수정 폴백**, **위치 불가 → 수동 입력**. 어떤 경로도 여행 중 사용자를 막다른 길에 두지 않는다(ADR-0011). 재계획 대안 산출은 **C1·C2(U5 자산) 재사용** — LLM/외부에 인질 잡히지 않는다.
3. **위치·배터리·프라이버시(핵심)** — 위치 감지는 **포그라운드 한정**(D27·G62 — 백그라운드 위치 권한 미요청), GPS 발자취는 **별도 옵트인**(D34·N2) 전제 저빈도(1~5분) 수집·원시 좌표 파기, 서버 폴링 주기는 **원격 구성 상한**(G195), 위치 처리는 **3층 동의 매트릭스**(G182)를 벗어나지 않는다.

보안·무상태·관측성·배포의 서버 규약은 **U1 전역 정본을 그대로 상속**하고, 재계획 연산(C1·C2)은 **U5 정본을 그대로 재사용**하며, U6는 **서버 폴링·클라 포그라운드 감지·위치 3층 폴백·GPS 법정 로그** 증분만 소유한다.

### 0.1 워크로드 중요도 (requirements.md §6.3)

| 컴포넌트 | 중요도 | 불가용 시 영향·U6 대응 |
|---|---|---|
| 실행 허브 상태 저장·조회(M18 저장부·DB) | **Critical** | 여행 중 활성 일정·방문 상태 접근 불가 — 현장 피해 최대. RDS Multi-AZ·Auth 상속(폴백 없음, 오류·재시도) |
| Plan-B 감지(M9) | **High** | 여행 중 선제 대응 불가 — **수동 재계획 경로 항상 유지**(services.md §S2.1). 자동 감지 축소를 화면 표기 |
| Plan-B 재계획(M10) | **High** | 대안 산출 불가 — **수동 일정 수정 폴백 존재**(US-E7-11·12). C1·C2는 U5 재사용(폴백 체인 상속) |
| 날씨·컨텍스트(M11 기상청 소비) | **Medium 이하** | 트리거 (a) 계열(강수·특보) 미발화 — **트리거 침묵**(허위 알림 금지). 나머지 트리거·수동 경로 정상 |

**핵심 원칙**: 외부·비결정 의존(기상청·지도·LLM)은 전부 **폴백 필수 경로**이며(§6.3 "외부 의존성은 전부 High 이하, 폴백 필수"), M9·M10이 High인 근거는 **수동 폴백이 존재**한다는 점이다 — 자동 대응이 죽어도 여행 중 사용자는 수동 재계획·수동 수정으로 일정을 이어갈 수 있다.

## 1. 성능 · 실시간성 (핵심 · D38, §6.1, services.md §S2.4)

§6.1이 U6에 부과하는 목표는 **Plan-B 대안 제시 10초(초과 시 수동 수정 폴백 제안)** 다. services.md §S2.4가 이 예산을 단계별로 배분한 **정본**이며, 본 절은 그 배분을 타임아웃·폴백 전환 시간과 함께 NFR로 승격한다. 현장 상호작용(도착 감지·거리 갱신)의 체감 목표도 함께 확정한다.

> **경계 명시 — AI 일정 생성(S1) 5초/20초는 U6 아님**: requirements.md §6.1의 "AI 일정 생성 첫 1일 5초/전체 20초"는 **M8 소유로 U5 범위**(services.md §S1.4). U6는 **재계획(S2)만** 소유하며 C1·C2를 **재사용**하되, U5의 5초/20초 예산·생성 트리거는 U5 NFR이 소유한다. 본 절은 S2(재계획 10초)·현장 실행 반응에 국한한다.

### 1.1 재계획 10초 예산 — 단계 예산 (services.md §S2.4 상속·승격)

| 단계 | 예산 | 타임아웃·폴백(§0.3 타임아웃<예산 원칙) | 추적 |
|---|---|---|---|
| 5~6. 세션 생성 + 영향 분석 입력 수집(위치·현재 시각·남은 일정·고정 제약) | **1.0초** | 위치 수동 입력 시간은 예산 외(사용자 상호작용). 위치 폴백 체인은 §2.3 | S2.2, US-E7-03·10 |

| 8. 후보 소싱(M7 — **사용자 저장 장소 우선** G53) | **1.5초** | 저장 장소 우선이라 대부분 내부 조회. 부족 시 주변 신규 POI 확장 | G53, US-E7-04 |

| **합계** | **9.0초** (여유 1.0초) | 초과 시 산출된 후보만 제시, 0건이면 3중 폴백 | **D38** |

- **NFR-U6-PERF-01 (재계획 10초 게이트)**: 사용자가 '대안 보기'를 누른 시점부터 **대안 후보 2~3개 + 전/후 비교를 10초 내 제시**한다(services.md §S2.4). 10초는 **하드 게이트가 아니라 체감 목표**이며, 초과 시에도 산출된 후보만이라도 제시하거나 수동 수정 폴백을 제안한다(빈 화면 금지, US-E7-04·11).
- **NFR-U6-PERF-02 (LLM 타임아웃 1.5초 < 예산)**: 재계획 사유 해석 단계는 **타임아웃 1.5초**로, 10초 예산 안에 규칙 매핑 폴백·후보 소싱·솔버 검증까지 완주할 여유를 남긴다(§0.3). 타임아웃은 예산이 아니라 **폴백 전환 트리거**다. C1은 U5 게이트웨이 재사용(경량 티어).
- **NFR-U6-PERF-03 (솔버 검증 시간 예산)**: 후보별 C2 검증·재정렬은 **후보당 1.5초**(2~3개 = 4.5초). 재정렬 범위가 **당일 잔여만**(C10 — 이월은 미배치 목록)이라 문제 크기가 작아(생성 S1보다 소) 예산 내 완료. C2는 U5 솔버 재사용(warm-start·오라클 검증 상속).
- **NFR-U6-PERF-04 (확정 재검증은 예산 외)**: 확정 버튼 시점 재검증(G56)은 **10초 예산 외 별도 상호작용**이다 — 그 사이 상황 변화로 무효화되면 확정을 차단하고 후보 재산출을 안내한다(services.md §S2.2 11단계).

### 1.2 현장 실행 반응 (Epic 6 — 도착 감지·거리 갱신)

- **NFR-U6-PERF-05 (도착 감지 반응)**: 포그라운드 지오펜스 진입 감지 시 **'도착하셨나요?' 확인 프롬프트를 지체 없이 노출**한다 — 단 도착 확정·방문 완료는 **항상 사용자 탭**으로만 처리(자동 확정 없음, D23). 지오펜스 평가는 클라이언트 로컬 연산(무네트워크·체감 즉시, §2.2). 외부 지도 앱 복귀 시 다음 예정지 근접이면 도착 프롬프트 재노출(US-E6-03).
- **NFR-U6-PERF-06 (거리 갱신 정책)**: 다음 예정지 직선거리는 **화면 진입/포커스 시 1회 산출 + 수동 새로고침**(G65 — 상시 폴링 아님, 배터리·비용 절감). 거리 기반 추정임을 표기하고 소요시간은 미표시(D25). 여유 시간은 다음 계획 시작 시각까지 단순 차이(G67).
- **NFR-U6-PERF-07 (일반 화면 전환 300ms)**: §6.1 상속 — 활성 허브·방문 전이·비교 카드 전환은 로컬 상태 전환. 지도 렌더는 U3 `shared/map` 상속(지연 로드·실패 시 목록 폴백).

## 2. 복원력 (핵심 · §6.2·§6.3, RESILIENCY-10, ADR-0011 — 침묵 실패 금지)

U6 복원력의 정본은 **services.md §0.1 실패 처리 표**와 **U1 PAT-RES-01 4요소 계약**, **U5 재계획 폴백 체인 재사용**이다. 본 절은 U6 폴백 체인을 설계 정본으로 승격한다. **핵심: 트리거는 허위 경보를 만들지 않고, 재계획은 외부에 인질 잡히지 않는다.**

### 2.1 트리거 침묵 — 기상청·휴무 무응답 (services.md §S2.1, D27)

| 의존 단계 | 실패 시 폴백 | 사용자 고지 | 추적 |
|---|---|---|---|
| **기상청(M11 — 단기예보·특보)** | **트리거 침묵** — 트리거 (a) 계열(강수·특보)만 자동 발화 중지, 마지막 성공 데이터 TTL 내 사용 후 침묵 | **없음**(허위 알림 금지) — 수동 재계획 경로는 항상 유지 | D27, US-E7-02, services.md §0.1 |
| **휴무·영업시간(M7 — J2 재조회)** | 해당 POI 스킵 + 로깅, 트리거 침묵(정기 변경만 자동 G192) | 없음 | G192, services.md §4 J2 |

- **NFR-U6-RES-01 (기상청 무응답 = 트리거 침묵, 허위 알림 금지)**: 기상청·영업시간 소스 무응답 시 **자동 알림을 발화하지 않는다**(services.md §0.1 — "허위 알림 금지"). 트리거 (a) 계열만 일시 중지하고 **나머지 트리거(위치 의존 (c)(d))·수동 재계획은 정상 동작**한다. 폴링 실패는 실패율로 계측(§6.7 침묵 실패 금지 관측 — "알림 침묵"은 관측되지만 사용자에게는 조용). 특보는 보수적으로 심각 분류(오탐보다 미탐 방지 — 단, 무응답 시엔 침묵이 우선).
- **NFR-U6-RES-02 (마지막 성공 데이터 TTL 사용)**: 폴링 실패 시 **마지막 성공 날씨 데이터를 TTL 내 사용** 후 만료되면 침묵으로 전환한다(unit-of-work.md U6 리스크 완화) — 오래된 데이터로 발화하지 않는다.

### 2.2 재계획 외부 실패 · 위치 불가 폴백 (services.md §S2.2)

| 의존 단계 | 실패 시 폴백 | 사용자 고지 | 추적 |
|---|---|---|---|
| **LLM 사유 해석(C1 — U5 재사용)** | 규칙 기반 사유-카테고리 매핑, 추천 이유 템플릿/생략 | "추천 이유를 불러오지 못했어요" | US-E7-01·04, NFR-U6-PERF-02 |
| **재계획 외부 API(길찾기·POI 검색·영업시간)** | **수동 일정 수정 화면 전환** — 순서 재배열·삭제·시각 직접 입력 | 누락된 외부 데이터 화면 표기, 숙소 고정 제약 위반 입력 차단 | US-E7-11, services.md §S2.2 9단계 |
| **솔버 유효 후보 0건(C2 — U5 재사용)** | **3중 폴백** — 남은 방문지 1개 건너뛰기 / 휴식 모드 전환 / 수동 일정 수정 | 대안 없는 이유 한 줄 설명 | US-E7-04·12 |
| **위치(GPS 불가·권한 거부)** | 위치 폴백 체인(§2.3) — 수동 입력→마지막 완료 방문지→등록 숙소 | 가정을 화면 표기(추정 출발지) | US-E7-10, G182 |

- **NFR-U6-RES-03 (재계획 외부 API 실패 = 수동 수정 폴백)**: 재계획 중 외부 API 오류(날씨·영업시간·길찾기·POI 검색 실패/시간 초과) 시 **수동 일정 수정 화면으로 전환**한다(US-E7-11). 수동 수정 중에도 **등록 숙소 체크인/체크아웃 고정 제약은 위반 불가**(입력 차단), 어떤 외부 데이터가 누락됐는지 화면 표기, **API 복구 시 자동 검증·재계획 재활성화**.
- **NFR-U6-RES-04 (유효 후보 0건 = 3중 폴백)**: 솔버가 대안 0개를 산출하면 빈 화면 대신 **3중 폴백(건너뛰기/휴식/수동 수정) + 사유 한 줄**을 제시한다(US-E7-04). C2 솔버·오라클 검증은 U5 재사용(NFR-U5-RES-02·03 상속).
- **NFR-U6-RES-05 (기존 유지 = 무변경)**: 대안 대신 "기존 유지" 선택 시 **어떤 변경도 저장하지 않고** 활성 일정으로 복귀한다(US-E7-06). current 갱신은 확정 TX(11단계)에서만 발생(services.md §S2.3).

### 2.3 위치 3층 동의 폴백 체인 (G182 — 어떤 조합에도 폴백)

- **NFR-U6-RES-06 (위치 불가 = 수동 입력 폴백)**: 위치 권한 거부·GPS 불가 감지 시 **재계획을 차단하지 않고** 즉시 수동 위치 입력(지도 핀·검색)으로 안내한다(US-E7-10). 폴백 체인: **GPS → 수동 입력 → 마지막 완료 방문지 → 등록 숙소**, 어느 폴백이든 **가정을 화면 표기**(추정 출발지). 도착 감지도 위치 불가 시 **수동 체크인으로 대체**(US-E6-01).
- **NFR-U6-RES-07 (3층 조합 전부 폴백 존재)**: OS 권한 × 법정 동의 × GPS 옵트인 3층(G182)의 **전 조합에 폴백이 존재**한다 — 권한 거부→수동 도착 확인·위치 수동 입력, 옵트인 거부→발자취 미수집(비교 화면은 계획 동선만·사유 표기, US-E7-13). 조합별 동작 매트릭스는 Functional Design에서 명문화(§9 이관 결정)하고 **테이블 주도 테스트**로 구현(unit-of-work.md U6 리스크 완화). §5 보안과 정합.

### 2.4 어댑터 4요소 계약 (RESILIENCY-10 — U1·U5 상속)

| 의존 (Port · 소유) | 타임아웃 | 재시도 | 서킷 | 폴백 |
|---|---|---|---|---|
| **기상청**(`WeatherPort` · M11) | 어댑터 타임아웃 | 즉시 1회(services.md J1) | 독립 서킷 | 마지막 성공 TTL 후 **트리거 침묵**(NFR-U6-RES-01) |
| **LLM 사유 해석**(`LlmPort` · C1 — **U5 재사용**) | 1.5초 | 스키마 위반 1회 | 벤더 독립 | 규칙 기반 사유 매핑(NFR-U6-PERF-02) |
| **이동시간**(`RoadDistancePort` · C2 — **U5 재사용**) | 어댑터 | 없음 | 독립 | 직선거리×우회계수(G106) |

- **NFR-U6-RES-08 (서킷·계측 의무)**: 기상청 어댑터는 명시적 타임아웃 + 독립 서킷(벌크헤드) + 폴백을 가진다(Resilience4j — U5·U3 확정분 재사용). 서킷 open/half-open·기상청 오류·트리거 침묵 전환은 전부 계측(SI §8.2 A12·A11, §6.7). C1·C2 어댑터는 **U5 정본(NFR-U5-RES-06) 재사용** — U6은 재계획 소비자.

## 3. 위치·배터리·프라이버시 (핵심 · D27·D34·N2·G182·G195)

여행 중 위치 처리는 **법정 동의·포그라운드·저전력** 세 제약을 동시에 지켜야 한다. 본 절은 위치·배터리·프라이버시 NFR을 확정한다.

### 3.1 포그라운드 한정 감지 · 배터리 (D27·G62)

- **NFR-U6-PRIV-01 (포그라운드 한정 — 백그라운드 위치 미요청)**: 위치 감지(지오펜스 도착·이동 지연·체류 초과)는 **포그라운드 한정**이며 **백그라운드 위치 권한을 요청하지 않는다**(D27·G62). 커버리지 한계(백그라운드 미발화)는 **제품 동작으로 수용**(D27 확정) — 앱 재진입 시 일괄 재평가하고, 서버 폴링 계열(날씨·휴무)은 포그라운드와 독립 동작(unit-of-work.md U6 리스크 완화).
- **NFR-U6-PRIV-02 (배터리 예산 — 저빈도 수집)**: GPS 발자취는 **포그라운드 한정 저빈도 수집(1~5분 간격)**(G55/G73)이며 상시 고빈도 추적을 하지 않는다. 지오펜스·거리 산출도 상시 폴링이 아니라 이벤트/포커스 기반(NFR-U6-PERF-06) — 배터리 소모를 구조적으로 억제한다. 걸음 수는 GPS 이동 거리 환산 추정만(추가 권한 없음 G59).
- **NFR-U6-PRIV-03 (위치 just-in-time 발화)**: 위치 권한 OS 다이얼로그는 **여행 중 첫 진입 시점에 발화**한다(US-E1-04 just-in-time 2번째 지점) — 프리프롬프트(사유 고지) 후 요청. 백그라운드 위치는 발화하지 않는다.

### 3.2 서버 폴링 주기 상한 (G195 — 원격 구성)

- **NFR-U6-PRIV-04 (폴링 주기 상한 원격 구성)**: 서버 배치 폴링은 **날씨 1시간·휴무 당일 아침 1회**를 초기값으로 하되 **전부 원격 구성(G195)** — 운영 중 주기·상한을 코드 재배포 없이 조정한다(비용·쿼터 보호). 트리거 빈도 상한(시간당 2회/하루 8회)·민감도 3단계·무시 억제도 원격 구성(G58/G195 — §4 알림 상한과 통합).

### 3.3 GPS 법정 동의·수집 최소화 (D34·N2)

- **NFR-U6-PRIV-05 (GPS 옵트인 전제 수집)**: GPS 발자취는 **GPS 기록 옵트인(3층 동의의 3층, N2) 동의를 전제**로만 수집한다(D34) — 옵트인 없이는 수집하지 않는다. 옵트인 철회·탈퇴 시 **즉시 파기**(D34 — 유예 없음, services.md §S6 2단계).
- **NFR-U6-PRIV-06 (원시 좌표 파기 · 단순화 폴리라인만 보존)**: 원시 GPS 좌표는 **단순화 폴리라인으로 가공 후 파기**하고 원시 좌표는 보존하지 않는다(G55/G73). 서버에는 단순화 폴리라인만 저장 — 위상은 보존하되 원시 재구성은 불가(§6 PBT 대상).

## 4. 트리거 상한·억제 (§7.3 G58/G195 · 순수 함수 G116)

- **NFR-U6-QOS-01 (트리거 빈도 상한)**: 자동 트리거는 **전역 상한 시간당 2회/하루 8회**(초과·동시 발화는 묶어 1회)를 지킨다(G58/G195). 동일 사유·동일 방문지는 1회만 노출하고, 닫으면 재노출하지 않는다(US-E7-02). 상한은 원격 구성(G195).
- **NFR-U6-QOS-02 (민감도·무시 억제 학습)**: 민감도 3단계(적게/보통/많이)가 상한을 **±50% 조정**하고, 사용자가 **2회 연속 무시하면 당일 동일 사유를 억제**(학습, G58). 휴식 모드 중에는 경미 사유 억제, **심각 사유(기상특보·고정 일정 위협)만 유지**(G54 — 심각은 억제 불가).
- **NFR-U6-QOS-03 (판정 순수 함수 분리 — G116)**: 트리거 판정 로직은 **clock·외부 데이터(날씨·체류)를 주입 가능한 순수 함수로 분리**해 결정적 테스트가 가능하게 한다(G116) — 상한·억제·민감도 시뮬레이션을 재현 가능한 PBT로 검증(§6).
- **NFR-U6-QOS-04 (심각/경미 2단 · 푸시 경계)**: 트리거는 severity 2단(severe·minor)으로 구분 — **심각만 푸시(M14 소비), 경미는 인앱 칩**(services.md §S2.1 4단계). 알림에 출처·감지 시각 표기(US-E7-02). **U6은 TriggerFired 발행까지**이며 실제 푸시 발송은 M14(U8) 소관(§5 경계).

## 5. 보안 (Security Baseline 상속 — §6.4 · 위치정보법 증분)

U6는 SECURITY 규칙의 신규 1차 구현 지점이 아니다 — 전역 보안 설정(deny-by-default·헤더·에러 핸들러·구조화 로깅·시크릿)은 U1 스캐폴드가 소유하고 U6는 상속한다. U6 증분은 **위치정보 법정 로그·GPS 데이터 처리·소유권** 세 지점이다.

- **NFR-U6-SEC-01 (위치정보 법정 로그 = U1 append-only 재사용)**: 위치정보 수집·이용·제공 사실확인자료(로그)는 **U1이 생성한 append-only 법정 로그 테이블에 기록**한다(N2·SECURITY-14) — U6은 이 테이블의 **첫 기록 개시 유닛**이다(SI §0 U6: "위치 법정 로그 기록 개시"). GPS 이벤트와 **동일 트랜잭션 기록**(유실 0, SI §5.3). 앱 역할은 **자기 로그 삭제 권한 없음**(DB REVOKE + IAM Deny 2중 강제, SI §3·§5.3·§7.1). 최소 6개월 보존, 계정 삭제 시에도 **법정 로그는 분리 보관 유지**(services.md §S6 2단계).
- **NFR-U6-SEC-02 (위치정보법 준수 — P1 법정 전제)**: GPS 발자취·위치 기반 트리거는 **위치기반서비스사업 신고(P1) 완료를 법정 전제**로 한다 — 신고 완료 전 **해당 기능 비활성 플래그 운영**(unit-of-work.md U6). 위치기반서비스 약관 별도 필수 동의·GPS 기록 별도 옵트인(3층 동의 G182)은 U1 동의 체계 상속(§3.3).
- **NFR-U6-SEC-03 (GPS 데이터 로그 마스킹 · 경계)**: 위치 데이터를 다루는 어댑터·서비스 로그는 **PII·정밀 좌표를 마스킹**(U1 PAT-OBS-01 상속·SECURITY-03) — 구조화 로그·상관 ID에 원시 좌표 미출현. 법정 로그(수집 사실)와 운영 로그(디버깅)를 분리.
- **NFR-U6-SEC-04 (소유권 — IDOR 차단)**: 실행 상태·트리거·재계획 세션·GPS 폴리라인 접근은 **토큰 주체(accountId) 스코핑**(IDOR 차단, NFR-U1-SEC-18 상속·SECURITY-08). 재계획은 확정 시점 소유권 재검증.
- **NFR-U6-SEC-05 (기상청 API 키 = 시크릿)**: 기상청 API 키는 **소스·이미지·설정 파일에 미포함** — Secrets Manager 주입(`external/kma`, SI §6 프레임 내 신규 항목, NFR-U1-SEC-11 상속). 시크릿 스캔 CI 게이트(SECURITY-10).
- **NFR-U6-SEC-06 (LLM 경계 = U5 재사용)**: 재계획 사유 해석의 LLM 컨텍스트 경계(서버 재조회 D31·closed-set 그라운딩 G115·필드 최소화)는 **U5 정본(NFR-U5-SEC-01·02) 재사용** — U6는 소비자이며 신규 LLM 경계를 만들지 않는다.

## 6. 하드 제약·PBT 요건 (D37 · §6.6)

U6는 하드 제약 4계열을 **재계획에서 재사용·재검증**한다(C2 재사용). NFR 관점에서 이는 **품질 게이트(머지 차단)** 요건이다.

- **NFR-U6-QUAL-01 (하드 제약 4계열 100% — 재계획 재검증, D37 머지 차단)**: 재계획 결과도 생성과 **동일한 하드 제약 4계열 검증 100%**(C2 재사용) — 특히 **고정 블록·확정 시각 필수 방문지 불변**, **확정 시점 재검증(G56) 통과 없이 반영 불가**. LLM·기상청은 어댑터 fake 모킹, 솔버·제약 검증은 실코드(D37·§6.6).
- **NFR-U6-QUAL-02 (PBT 대상 — §6.6·G116)**: (a) **트리거 판정 순수 함수**(임의 clock·날씨·체류 입력에 대해 상한 초과 발화 0·무시 2회 후 당일 동일 사유 억제·민감도 단조성), (b) **방문 상태 머신**(예정→도착→시작→완료/스킵) 전이 불변식, (c) **여행 실행 상태 머신**(활성↔휴식→종료 — 자동/수동 종료 경합 안전), (d) **재계획 warm-start 고정 블록 보존**, (e) **휴식 모드 억제 규칙**(심각 사유는 억제 불가), (f) **폴리라인 단순화**(원시 파기 후 재구성 불가·경로 위상 보존). Kotest(서버)·fast-check(클라)·시드 로깅·수축(PBT-08). 속성 식별은 Functional Design(PBT-01).
- **NFR-U6-QUAL-03 (E2E 종단 흐름 완성 — G118)**: **숙소 저장→등록→일정 생성→재계획 E2E를 U6 완료 시점에 CI 필수로 완성**(D37 계층 분리 — 외부는 fake). CP3(소비자)·CP4(공급자 — VisitChecked·TripEnded·DayClosed·changelog diff)·CP5(TriggerFired) 계약 테스트 통과.

## 7. 규모·확장성 (§6.8 G142 — DAU 1천 · 활성 여행 ≤1/사용자)

- **NFR-U6-SC-01 (서버 폴링 대상 = 진행 중 여행 수 — 과설계 금지)**: 서버 배치 폴링(J1 날씨·J2 휴무)의 대상은 **진행 중 여행**이다. 여행 날짜 겹침 차단(D21)으로 **사용자당 활성 여행 ≤1**이며, DAU 1천에서 진행 중 여행 수는 소량(현재 여행 중인 사용자의 부분집합)이다 — **별도 폴링 워커·큐·오토스케일 상향은 과설계**(G142). 기존 ShedLock 스케줄러(SI §1.2) 내에서 흡수한다. **전환 트리거**: 폴링 대상·기상청 호출량이 태스크·쿼터를 상시 압박하면 EventBridge + ECS RunTask 분리 재평가.
- **NFR-U6-SC-02 (기상청 호출 격자 중복 제거)**: 날씨 폴링은 **방문 예정지 좌표를 격자(格子)로 변환 후 중복 제거**해 호출한다(services.md J1) — 인접 방문지는 동일 격자로 묶여 호출량을 억제한다. 트리거 상한(G58)이 중복 발화를 흡수(J1 멱등성). 폴링 주기 상한은 원격 구성(G195, NFR-U6-PRIV-04).
- **NFR-U6-SC-03 (데이터 볼륨)**: U6 신규 테이블 ~8~9개(실행 상태·여행 실행 상태·트리거 이력/억제·민감도·재계획 세션·대안 후보·미배치 목록·날씨 캐시(격자)·GPS 폴리라인). DAU 1천·활성 여행 ≤1 기준 소량 — 단일 PostgreSQL로 충분(SI §3, 샤딩 불요). GPS 폴리라인·법정 로그는 append 성격이나 저빈도(1~5분)·활성 여행 소량이라 일 수만 행 수준(SI §5.3).
- **NFR-U6-SC-04 (폴링/배치 커넥션 벌크헤드)**: 폴링·배치 잡의 DB 커넥션 풀은 실행 허브 API 처리 풀과 분리(U1 PAT-PERF-02 벌크헤드 상속) — 배치가 여행 중 실시간 요청 지연을 유발하지 않게 한다.

## 8. 미결·선결 항목

### 8.1 선결 과제 — 법정·설계 입력 (P1·P3·P7)

U6는 **비개발 선결 3건**이 있으나, 본 NFR 산출물은 **어댑터 fake·비활성 플래그로 비차단**이다.

| 과제 | 차단 대상 | 본 문서의 대응 |
|---|---|---|
| **P1 위치기반서비스사업 신고**(위치정보법 제9조) + 위치 전문 법무 자문 | GPS 발자취·위치 기반 트리거의 **법정 출시 전제** | 위치 3층 모델·법정 로그 스키마는 U1에서 자문 반영 완료(append-only 재사용). **신고 완료 전 기능 비활성 플래그** 운영(unit-of-work.md U6) — 개발·테스트는 비차단 |
| **P3 기상청 공공데이터포털 API 활용 신청** | M11 개발·트리거 (a) 계열(강수·특보)의 전제 | `WeatherPort` 어댑터를 **fake로 개발·테스트 선행**(D37 계층 분리), 실키는 병렬 신청. 무응답 시 트리거 침묵 폴백(NFR-U6-RES-01) — 비차단 |
| P7 약관(위치기반서비스 약관) | 동의 화면·재열람 실문안 | U1 동의 체계(3층 모델·버전) 상속, 문안만 교체 — 비차단 |

### 8.2 벤더·인프라 미결 — 기상청만 (전역 결정 상속)

**U6의 신규 외부 벤더·인프라 미결은 기상청(P3) 1건뿐이다.** U1이 PD-1~9를 전부 종결했고, 재계획 연산(C1·C2)은 U5가 확정했으며, U6는 그 종결·확정 결과와 GD-1~7·SI를 상속한다.

| 잠재 결정 항목 | U6 처리 | 근거 |
|---|---|---|
| 클라우드·CI/CD·배포·롤백 | GD-1~7 상속(재질문 없음) | SI 헤더·U1 nfr-design §2 |
| 컴퓨트·DB·로그·알림·스케줄러 스택 | SI 정본 상속(재정의 금지 — ShedLock·아웃박스·NAT) | SI §1.2·§3·§4·§8 |
| 재계획 연산(C1 LLM·C2 솔버) | **U5 정본 재사용(신규 없음)** | U5 tech-stack·nfr-design |
| 위치 법정 로그·시크릿 프레임 | U1·SI 상속(append-only 테이블·Secrets Manager) | SI §5.3·§6 |
| **기상청 API 활용 신청·쿼터** | **P3 확정 — 어댑터는 벤더 중립·fake 선행** | requirements.md P3·D10 |
| 기상청 클라이언트·폴링·포그라운드 위치·폴리라인 | 기존 스택 내 라이브러리·패턴 구체화(신규 벤더 아님) | tech-stack-decisions.md |
