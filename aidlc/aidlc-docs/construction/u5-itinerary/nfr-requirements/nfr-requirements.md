## 0. U5 NFR 프로필 — "제품의 심장 · 연산 무거움"의 NFR 본질

U5의 NFR 본질은 **비결정적·고비용 LLM과 CPU 집약 솔버 위에서 5초/20초 응답 예산·무실패 보장·운영비 통제를 동시에 지키는 것**이다. 세 축이 핵심이고 서로 긴장한다:

1. **성능(핵심)** — D38의 **첫 1일 5초 / 전체 20초**를 services.md §S1.4 단계 예산으로 분해하고, 예산 초과가 예상되는 LLM은 **타임아웃을 예산보다 짧게(2.5초)** 잡아 폴백 전환 시간을 확보한다(§0.3). 첫 1일 우선 반환 + 잔여 백그라운드로 체감을 앞당긴다.
2. **복원력(핵심)** — LLM(Medium §6.3)은 **결정론 솔버 폴백**으로, 솔버 전 경로 실패는 **최소 일정**으로 지속한다 — 어떤 경로도 빈 화면(침묵 실패)을 만들지 않는다(RESILIENCY-10·ADR-0011). C1·C2는 U6·U7·U9 재사용 자산이므로 폴백을 프로덕션 품질로 만든다.
3. **비용(핵심)** — LLM 호출은 곧 운영비다. **전 일자 공용 1회 호출**(일자별 재호출 없음)·**기능별 티어 분리**(경량/상위, D11)·**사용자별 rate-limit**(SECURITY-11)·후보 풀 상한 프루닝으로 호출량을 구조적으로 억제하고, 국외 이전 고지(G181)·비용 계측(SI §8 A15)을 강제한다.

보안·무상태·관측성·배포의 서버 규약은 **U1 전역 정본을 그대로 상속**하고, U5는 LLM 경계(D31)·closed-set 그라운딩(G115)·솔버 결정성 증분만 소유한다.

### 0.1 워크로드 중요도 (requirements.md §6.3)

| 컴포넌트 | 중요도 | 불가용 시 영향·U5 대응 |
|---|---|---|
| 일정 저장·조회(M8 저장부·DB) | **Critical** | 여행 중 일정 접근 불가 — RDS Multi-AZ·Auth 상속(폴백 없음, 오류·재시도) |
| AI 일정 생성 LLM 경로(C1) | **Medium** | **결정론 솔버 폴백으로 지속**(품질 저하 고지) — LLM에 생성이 인질 잡히지 않음 |
| 솔버 배치(C2) | 기반 — 인프로세스 결정론 | 외부 무의존 순수 연산. 실패 모드는 '해 없음/시간 초과'만 → 최소 일정 폴백 |
| 후보 풀(M7 소비) | 기반(Medium) | 외부 소스 실패 시 정본 캐시 축소 풀(U3 소유 — U5는 소비자) |

**핵심 원칙**: 외부·비결정 의존(LLM·TMap·POI)은 전부 **Medium 이하 경로**이며(§6.3 "외부 의존성은 전부 High 이하, 폴백 필수"), 어떤 경로도 U5의 가용성 바닥이 되어서는 안 된다 — 마지막 폴백은 항상 **결정론 솔버가 만든 유효한 최소 일정**(services.md §0.1).

## 1. 성능 (핵심 · D38, §6.1, services.md §S1.4)

§6.1이 U5에 부과하는 목표는 **AI 일정 생성: 첫 1일 5초 내 노출 · 전체 20초 한계(초과 시 결정론적 솔버 폴백 고지)** 다. services.md §S1.4가 이 예산을 단계별로 배분한 **정본**이며, 본 절은 그 배분을 타임아웃·폴백 전환 시간과 함께 NFR로 승격한다.

> **경계 명시 — Plan-B(S2) 10초는 U5 아님**: requirements.md §6.1의 "Plan-B 대안 제시 10초"는 **M9·M10 소유로 U6 범위**(services.md §S2.4). U5는 **생성(S1)만** 소유하며, C1·C2를 U6이 재사용하되 U6의 10초 예산·트리거는 U6 NFR이 소유한다. 본 절은 S1(첫 1일 5초/전체 20초)에 국한한다.

### 1.1 첫 1일 5초 게이트 — 단계 예산 (services.md §S1.4 상속·승격)

| 단계 | 예산 | 타임아웃·폴백(§0.3 타임아웃<예산 원칙) | 추적 |
|---|---|---|---|
| 1~2. 세션 생성 + 컨텍스트 병렬 로드(M6·M2·M4) | **0.3초** | 단일 DB 왕복 병렬화. M2 실패→중립 기본값 전량 대체(무실패) | S1.2, US-E5-01 |
| 3. 후보 풀 조회(M7 closed-set) | **1.0초** | 정본 캐시 히트 0.2초·외부 보강 상한 1.0초. 외부 실패→정본 캐시 축소 풀 + 기본 모드 플래그 | G115, US-E5-02 |
| 4. **LLM 취향 점수(C1 — 경량 티어)** | **2.5초** | **타임아웃 2.5초 — 초과 즉시 규칙 점수 폴백(+0.1초)**. 전 일자 공용 1회 호출(§3.1) | D11, US-E5-02·09 |
| 5. 1일차 solve(C2) | **0.8초** | 후보 상한(지역당 5천, G142) 기준. 해 없음→완화 제안/충돌 강조 | US-E5-03·04 |
| 6. 저장 + 직렬화 + 응답 | **0.4초** | TX1 day1 초안 저장(세션 `DAY1_READY`) | S1.2·1.3 |
| **첫 1일 합계** | **5.0초** | — | **D38** |

- **NFR-U5-PERF-01 (첫 1일 5초 게이트)**: TX1로 1일차 초안을 저장한 직후 **첫 1일을 5초 내 반환**한다(services.md 6단계). 이후 단계는 백그라운드(§1.2). 5초는 **하드 게이트**가 아니라 체감 목표이며, 초과 시에도 폴백으로 유효 결과를 반환한다(빈 화면 금지, US-E5-09).
- **NFR-U5-PERF-02 (LLM 타임아웃 2.5초 < 예산)**: LLM 점수 단계는 **타임아웃 2.5초**로, 5초 예산 안에 규칙 점수 폴백(+0.1초)·잔여 단계까지 완주할 여유를 남긴다(§0.3). 타임아웃은 예산이 아니라 **폴백 전환 트리거**다.
- **NFR-U5-PERF-03 (솔버 시간 예산)**: 1일차 solve 0.8초, 잔여 일자 solve **일자당 0.5초 목표**(§1.2). 후보 풀은 지역당 5천 상한 프루닝(G142)으로 문제 크기를 억제 — 솔버 시간이 후보 폭발로 발산하지 않게 한다.

### 1.2 전체 20초 한계 — 잔여 일자 백그라운드 (services.md §S1.4)

| 단계 | 예산 | 폴백 | 추적 |
|---|---|---|---|
| 7. 잔여 일자 루프(최대 29일, G42) | **15초 내** | 일자당 solve 0.5초 목표 · **4단계 LLM 점수 재사용(재호출 없음)** | §3.1 |
| **전체 한계** | **20초** | 초과 시 잔여 일자를 **결정론 솔버 단독 모드**로 완성 + 폴백 고지 | D38, US-E5-09 |

- **NFR-U5-PERF-04 (점진 노출)**: 완료된 첫 1일을 먼저 노출하고 나머지 날짜를 백그라운드로 이어 채우며, **생성 중 단계 텍스트·진행률·경과 시간을 표시**한다(US-E5-09). 진행 채널은 점진 응답 스트리밍(tech-stack-decisions.md §3 — SSE 권고).
- **NFR-U5-PERF-05 (20초 하드 타임아웃 후 결정론 단독)**: 전체 20초 도달 시 잔여 일자는 **규칙 점수 + 결정론 솔버 단독 모드**로 완성한다("일부 추천이 기본 모드로 생성되었어요", US-E5-09). 20초는 **비용·UX 상한**이며 무한 대기를 차단한다.
- **NFR-U5-PERF-06 (취소·이어서 생성)**: 생성 중 '취소' 시 부분 일정을 초안 저장 + '이어서 생성' 진입점 제공(G161). '백그라운드로 계속'은 완료 시 화면 복귀 시 반영(US-E5-09).
- **NFR-U5-PERF-07 (편집 재검증 즉시)**: 편집(POI 추가·삭제·재정렬·시각 변경) 시 **클라이언트 경량 검증기(shared/validation)가 즉시 재검증**하고, 저장 시 서버 확정 검증(D28). 클라 검증은 무네트워크 로컬 연산(체감 즉시), 서버 `validateEdit`는 무상태(§0.3 편집 재검증 무상태).

### 1.3 클라이언트 성능

- 일반 화면 전환 300ms(§6.1) 상속 — 시간표/지도 2보기 전환(US-E5-06)은 로컬 상태 전환. 지도 렌더는 U3 `shared/map` 상속(지연 로드·실패 시 시간표 폴백, US-E5-06).
- 생성 진행 화면은 스트리밍 이벤트(단계·진행률·경과)를 수신해 대기 체감을 관리 — 5초 게이트 전에도 진행 상태로 빈 화면을 만들지 않는다(NFR-U5-PERF-04).

## 2. 복원력 (핵심 · §6.2·§6.3, RESILIENCY-10, ADR-0011)

U5 복원력의 정본은 **services.md §0.1 실패 처리 표**와 **component-dependency.md §2.2·2.3(M8→C1·C2 장애 전파)**, **U1 PAT-RES-01 4요소 계약**이다. 본 절은 U5 폴백 체인을 설계 정본으로 승격한다. **핵심: 생성은 절대 LLM에 인질 잡히지 않는다.**

### 2.1 폴백 체인 (services.md §0.1 — 침묵 실패 금지)

| 의존 단계 | 실패 시 폴백 | 사용자 고지 | 추적 |
|---|---|---|---|
| **LLM 점수/설명(C1)** | **규칙 점수 결정론 경로** — 카테고리-취향 축 매핑 + 인기 집계 가중, 설명 문구 생략 | "일부 추천이 기본 모드로 생성되었어요" / "추천 이유를 불러오지 못했어요" | RESILIENCY-10, US-E5-09 |
| **외부 POI/영업시간(M7 어댑터)** | 정본 캐시(J7 동기화분) 축소 풀 → 사용자 저장 POI + 필수 방문지만 최소 풀 | '기본 모드' 플래그·'미확인' 배지 | US-E5-02·03 |
| **이동시간(C2 TMap 어댑터)** | 직선거리 × 우회계수(G106) 추정 → 최후 수동 입력 모드 | 추정치·수동 입력임을 화면 표기 | D25, US-E5-06 |
| **솔버 전 경로 실패(C2 포함)** | **숙소 + 시각 고정형 필수 방문지만 배치한 최소 일정** | "추천을 다시 시도" 동작 노출 | RESILIENCY-10, US-E5-09 |

- **NFR-U5-RES-01 (LLM 폴백 = 결정론 솔버 · Medium 경로)**: LLM 벤더 다운·타임아웃(2.5초)·**출력 스키마 검증 실패**(1회 재시도 후) → C1 서킷 오픈 → M8은 **규칙 점수(취향 가중 휴리스틱) + 결정론 솔버**로 지속(component-dependency §2.2). LLM 경로는 Medium(§6.3)이며 **기본 모드 고지**를 동반한다(침묵 실패 금지). RESILIENCY-01 정합: LLM 경로는 폴백으로 지속되는 Medium 경로다.
- **NFR-U5-RES-02 (솔버 전 경로 실패 = 최소 일정)**: 솔버 실행 불가(제약 과잉·후보 부족)·시간 예산 초과 등 **전 생성 경로 실패** 시 빈 화면 대신 **숙소 + 시각 고정형 필수 방문지만 배치한 최소 일정**을 반환한다(services.md §0.1·S1.9 최후 폴백, US-E5-09). "결정론 폴백은 덜 최적이어도 유효한 해를 보장"(unit-of-work.md U5 리스크 완화).
- **NFR-U5-RES-03 (해 없음 = 완화 제안, 자동 변경 없음)**: 제약 과잉으로 해 없음 → **위반 제약 완화 제안**(필수 방문지 축소·시각 고정 해제·다른 날짜·시간창 확대)을 구조화 응답으로 반환(US-E5-04). 고정 블록 상호 충돌 → **자동 변경 없이** 충돌 강조 + 사용자 질의(US-E5-04). 부분해는 초안 저장 + '이어서 생성'(G161).
- **NFR-U5-RES-04 (결정론 폴백 모드 = 완전 결정성)**: 규칙 점수 + 결정론 솔버 단독 모드는 **동일 입력→동일 출력**(폴백 결정성 — PBT 1급 대상 (d), unit-of-work.md U5 DoD). LLM·외부 데이터 없이도 하드 제약(영업시간·이동 부등식·고정 블록)을 만족하는 유효 일정을 생성한다(US-E5-09).
- **NFR-U5-RES-05 (외부 POI 캐시 축소 — 가용성 아님)**: 후보 풀 조회의 외부 실패는 **정본 캐시 잔존분으로 축소 풀** 구성(U3 소유 캐시 — U5는 소비자). 캐시는 지연 최적화가 아니라 이 경로에서 **폴백 데이터 소스**로 기능한다(component-dependency §2.1).

### 2.2 어댑터 4요소 계약 (RESILIENCY-10 — component-dependency §6 상속)

| 의존 (Port · 소유) | 타임아웃 | 재시도 | 서킷 | 폴백 |
|---|---|---|---|---|
| **LLM 벤더** (`LlmPort` · C1) | 점수 2.5초 / 설명·회고 상위 티어 별도 | 스키마 위반 1회(§0.1) | 벤더 독립 서킷 | 규칙 점수 결정론 폴백(NFR-U5-RES-01) |
| **TMap 도로 거리** (`RoadDistancePort` · C2) | 어댑터 타임아웃 | 없음 | 독립 서킷 | 직선거리×우회계수 1.3+안전계수(G106) → 네이버 2차(D08) |

- **NFR-U5-RES-06 (서킷·계측 의무)**: C1·C2 외부 어댑터는 명시적 타임아웃 + 독립 서킷(벌크헤드) + 폴백을 가진다. 서킷 open/half-open·LLM 오류·솔버 검증 실패는 전부 계측(SI §8.2 A12, §6.7 침묵 실패 금지 관측). TMap은 **C2 소유**(U3는 미호출 — U3 NFR-U3-RES-04 경계).

## 3. 비용 (핵심 · LLM 운영비 통제 · P6·G181)

LLM 호출은 곧 운영비이며, 비용 상한이 후보 풀 크기·호출 전략을 결정한다(unit-of-work.md U5 리스크·선결 P6). 본 절은 호출량을 **구조적으로 억제**하는 NFR을 확정한다.

### 3.1 LLM 호출 통제 (D11·SECURITY-11·G196 정합)

- **NFR-U5-COST-01 (전 일자 공용 1회 호출 — 재호출 없음)**: LLM 취향 점수는 **생성 세션당 전 일자 공용 1회 호출**한다(services.md S1.2 4단계). 잔여 일자 solve는 **동일 점수를 재사용**하고 **LLM을 재호출하지 않는다**(§S1.4 7단계). 이것이 20일 여행에서 LLM 호출을 20회가 아닌 **1회**로 억제하는 1차 비용 방벽이다.
- **NFR-U5-COST-02 (기능별 티어 라우팅 D11)**: LLM 모델은 기능별로 티어를 분리한다 — **취향 점수·재질의 = 경량 티어**, **회고·배치 설명 = 상위 티어**(D11·G194). 경량 작업이 상위 모델 단가를 소모하지 않게 한다. 티어 라우팅은 C1 내부(tech-stack-decisions.md U5-C1).
- **NFR-U5-COST-03 (사용자별 rate-limit — SECURITY-11)**: LLM 호출은 **사용자별 상한 rate-limit**을 적용한다(SECURITY-11, U1 PAT-SEC-03 레이트리미터 재사용 — LC-3 전역 컴포넌트). 반복 생성·어뷰징이 비용 폭증으로 이어지지 않게 하고, 동시 생성 10(G142)과 정합한다.
- **NFR-U5-COST-04 (후보 풀 상한 프루닝)**: LLM 점수 입력 후보는 **지역당 5천 상한 내 프루닝**(§6.8 G142)한다 — 토큰·단가는 후보 수에 비례하므로 후보 폭발을 차단한다(unit-of-work.md U5 리스크: "후보 풀 크기 상한 프루닝"). closed-set 후보 ID만 전송(전체 POI 미전송).
- **NFR-U5-COST-05 (LLM 비용 계측)**: 호출별 티어·토큰·추정 비용을 LLM 호출 로그(엔티티)에 기록하고 **CloudWatch 커스텀 메트릭**으로 발행 → SI §8.2 대시보드 LLM 비용 위젯(U5 확장 슬롯)·A11 LLM 쿼터 80% 알람·**A15 월 비용 예산 알람**(infrastructure-design.md §5). U5가 LLM 비용 계측의 첫 소비자다.

### 3.2 국외 이전·고지 (G181, §6.5)

- **NFR-U5-COST-06 (국외 이전 고지 · 필드 최소화 — G181)**: LLM 벤더가 국외인 경우 **개인정보처리방침에 국외 이전·처리위탁 고지**(G181·§6.5, P7 반영)를 하고, **LLM 전송 필드를 목적 최소화**한다 — closed-set 후보 ID·취향 축·중립 기본값만 전송하고, 타 사용자 데이터·내부 지표(제휴 수수료)는 구조적으로 미포함(D31·SECURITY-11, §5). 벤더 국내/국외 판별·단가는 **P6 확정** 후 remote config·시크릿으로 주입(비차단).

## 4. 확장성 (§6.8 G142 — DAU 1천 · **동시 일정 생성 10**)

### 4.1 동시 생성 규모

- **NFR-U5-SC-01 (동시 생성 10 — 과설계 금지)**: 규모 가정은 **동시 일정 생성 10건**(§6.8 G142)이다. 이는 **CPU 집약 솔버 연산 10개 동시**를 의미하며, 별도 솔버 워커 서비스·큐·오토스케일 상향은 **과설계**다(G142) — 기존 ECS 태스크 2개(1 vCPU/2GB, SI §1.2) 내에서 흡수 가능한지를 태스크 CPU 사이징으로 판단한다(infrastructure-design.md §2). **전환 트리거**: 동시 생성 실측이 태스크 CPU를 상시 포화시키면 솔버 워커 분리·태스크 vCPU 상향 재평가.
- **NFR-U5-SC-02 (백그라운드 잔여 일자 = 요청 단위 비동기)**: 잔여 일자 생성은 **요청 단위 비동기 작업(워커 스레드, 모듈 경계 유지)** 이다(unit-of-work.md U5 스케줄러 잡: 없음). **ShedLock 스케줄 잡·EventBridge·큐 신규 없음** — 생성 요청 수명 내 앱 프로세스 내부 실행(가상 스레드/Executor, tech-stack-decisions.md §4). 백그라운드는 5초 게이트 이후 잔여 일자를 채우고 20초 내 완료 또는 결정론 단독 완성.
- **NFR-U5-SC-03 (솔버 CPU 벌크헤드)**: 솔버(CPU 집약)와 편집 재검증·조회(I/O)의 스레드 풀을 분리(U1 PAT-PERF-02 벌크헤드 상속) — 무거운 solve가 경량 편집 재검증·CRUD 지연을 유발하지 않게 한다. 동시 생성 10 상한을 세마포어/rate-limit으로 보호(NFR-U5-COST-03과 통합).

### 4.2 데이터 볼륨

- **NFR-U5-SC-04 (일정 데이터 볼륨)**: U5 신규 테이블 ~6개(일정 plan/current·일자·슬롯·생성 세션·추천 이유 메타·LLM 호출 로그, unit-of-work.md U5). DAU 1천·여행당 일정 1개(활성 ≤1) 기준 소량 — 단일 PostgreSQL로 충분(SI §3, 샤딩 불요). LLM 호출 로그는 비용 계측·감사용 append 성격(보존 정책은 U1 로그 규약 상속).

## 5. 보안 (Security Baseline 상속 — §6.4 · LLM 경계 증분)

U5는 SECURITY 규칙의 신규 1차 구현 지점이 아니다 — 전역 보안 설정(deny-by-default·헤더·에러 핸들러·구조화 로깅·시크릿)은 U1 스캐폴드가 소유하고 U5는 상속한다. U5 증분은 **LLM 경계·closed-set 그라운딩·소유권** 세 지점이다.

- **NFR-U5-SEC-01 (LLM 컨텍스트 서버 재조회 경계 — D31 · SECURITY-11)**: LLM 컨텍스트 주입은 **서버 재조회 방식** — 클라이언트는 ID만 전달하고 **서버가 요청자 권한으로 재조회**해 주입한다(D31·ADR-0015). 타 사용자 비공개 데이터·내부 지표(제휴 수수료 등)가 **구조적으로 LLM 입력에 미포함**된다(SECURITY-11). **U5는 이 경계의 소비자**이며, C1의 컨텍스트 재조회는 호출자가 전달한 조회 콜백/참조 해석기로 수행해 C1→기능 모듈 역의존을 방지한다(component-dependency §1 C1 원칙).
- **NFR-U5-SEC-02 (closed-set 그라운딩 구조 보안 — G115)**: LLM은 **그라운딩된 후보 POI ID 목록에서만 선택**(closed-set)하며, 후보 목록 밖 POI는 **구조적으로 일정에 포함될 수 없다**(G115·US-E5-02). 출력 스키마 검증 + **ID 화이트리스트 교차 검증을 C1 출구에서 강제** — closed-set 밖 ID 반환은 구조적으로 불가하나 방어적으로 드롭·계측(services.md S1.2 4단계). **프롬프트 인젝션으로 그라운딩을 우회할 수 없는 설계**(LLM 신뢰 불요, unit-of-work.md U5 리스크 완화). POI 그라운딩은 하드 제약 4계열의 하나(D37 머지 차단).
- **NFR-U5-SEC-03 (LLM 호출 rate-limit — SECURITY-11)**: 사용자별 LLM 호출 상한(NFR-U5-COST-03과 동일 메커니즘) — 보안(어뷰징 차단)·비용 통제 이중 목적.
- **NFR-U5-SEC-04 (계정 무결성 — 하드 제약 D37)**: 생성 입력·출력에 **타 계정 데이터 미혼입**(D31 서버 재조회 경계) — 계정 무결성은 하드 제약 4계열의 하나(unit-of-work.md U5 DoD, D37 머지 차단). 일정·세션·슬롯 접근은 토큰 주체(accountId) 스코핑(IDOR 차단, NFR-U1-SEC-18 상속·SECURITY-08).
- **NFR-U5-SEC-05 (LLM API 키 = 시크릿)**: LLM 벤더 API 키·TMap 키는 **소스·이미지·설정 파일에 미포함** — Secrets Manager 주입(`external/llm`·`external/tmap`, SI §6 예약분, NFR-U1-SEC-11 상속). 어댑터별 키 격리, 시크릿 스캔 CI 게이트(SECURITY-10).
- **NFR-U5-SEC-06 (편집 입력 검증)**: 편집·생성 요청은 스키마 검증(SECURITY-05) — 슬롯·시각·POI ID·시간창 범위 검증, 클라 경량 검증기와 서버 확정 검증의 **규칙 명세 단일 원천 공유**(D28 — 통과→거부 UX 불일치 차단, tech-stack-decisions.md U5-C2).

## 6. 하드 제약·PBT 요건 (D37 · §6.6 — U5가 4계열의 본체)

U5는 하드 제약 4계열의 **본체**다(unit-of-work.md U5 DoD). NFR 관점에서 이는 **품질 게이트(머지 차단)** 요건이다.

- **NFR-U5-QUAL-01 (하드 제약 4계열 100% — D37 머지 차단)**: ① 숙소 기준점(모든 일자 동선이 해당 구간 거점 기준), ② 충돌 무배치(시간 중복·영업시간 밖·이동 부등식 위반 배치 0), ③ POI 그라운딩(출력 POI ⊆ 후보 풀 — closed-set), ④ 계정 무결성(타 계정 미혼입) — 100% 통과가 머지 차단 조건. LLM·외부 API는 어댑터 fake 모킹, 솔버·제약 검증은 실코드(D37·§6.6).
- **NFR-U5-QUAL-02 (PBT 1급 대상 — §6.6)**: 임의 후보 풀·시간창·고정 블록 입력에 대해 (a) 영업시간 내 배치, (b) 인접 슬롯 이동시간 부등식, (c) 고정 블록 불변(warm-start 전후 동일, G46), (d) 폴백 모드 결정성(동일 입력→동일 출력), (e) 일정 상태 머신(초안→편집중→확정→해제, D20) 전이 불변식, (f) plan/current 직렬화 왕복. Kotest(서버)·fast-check(클라)·시드 로깅·수축(PBT-08). 속성 식별은 Functional Design(PBT-01).

## 7. 미결·선결 항목

### 7.1 선결 과제 — 설계 입력 (P6)

U5는 **비개발 선결 1건(P6)이 C1 설계의 벤더·티어·비용 파라미터를 확정**한다. 본 NFR 산출물은 **벤더 중립 요건**까지 확정하며 차단되지 않는다.

| 과제 | 차단 대상 | 본 문서의 중립 대응 |
|---|---|---|
| **P6 LLM 벤더 계약·비용 산정 + 국외 이전 고지 문안**(D11·G181) | LLM 벤더/모델/단가, 후보 풀 크기·호출 전략, 국외 이전 여부 | `LlmPort` 추상화·티어 라우팅·**전 일자 1회 호출**(NFR-U5-COST-01)·rate-limit·비용 계측·후보 프루닝 구조를 벤더 중립으로 확정(비차단). 벤더·단가는 P6 후 remote config·시크릿 주입. 국외 이전 고지는 P7 개인정보처리방침(G181) |
| P2 지도 API 약관(TMap 도로 거리) | C2 `RoadDistancePort` 호출 조건 | U3에서 기완료 전제. C2 어댑터 격리 — 실패 시 직선거리×우회계수(G106) 폴백. TMap은 C2 소유(U3 미호출) |

### 7.2 벤더 종속 미결 — LLM 벤더만 (전역 결정 상속)
