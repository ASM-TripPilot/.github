# 1. U5 신규 패턴

## PAT-U5-01 점진 생성·첫 1일 우선 반환 — 5초 게이트·백그라운드 잔여 `[U5]`

**문제**: 전체 여행 일정(최대 29일)을 다 만든 뒤 반환하면 사용자는 수십 초 빈 화면을 본다. D38은 **첫 1일 5초·전체 20초**를 요구하는데, 단일 응답 모델로는 이 체감을 만들 수 없다. 예산 초과가 예상되는 LLM 단계를 예산 그대로 잡으면 폴백 전환 시간이 없어진다.

**적용(설계)**: services.md §S1.4 단계 예산을 **2구간(첫 1일 동기 / 잔여 백그라운드)** 으로 분리하고, 예산 초과 예상 단계는 **타임아웃 < 예산**으로 폴백 여유를 확보한다(§0.3).

```text
[생성 요청] → GenerationSession(COLLECTING) + SSE 진행 채널 개설
  1~2. 컨텍스트 병렬 로드(M6·M2·M4)          0.3초
  3.   후보 풀(M7 closed-set)                1.0초
  4.   LLM 취향 점수(경량, 전 일자 1회)       2.5초 ← 타임아웃 2.5초(초과→규칙 점수 +0.1초)
  5.   1일차 solve(C2)                        0.8초
  6.   TX1 day1 저장 + 응답                    0.4초  ══> 첫 1일 5초 게이트 반환(DAY1_READY)
  ─────────────────────────────────────────────  (여기까지 5.0초)
  7.   백그라운드: 잔여 일자 루프              ≤15초  ← LLM 점수 재사용(재호출 없음), 일자당 solve 0.5초
       │  진행 스트림(단계·진행률·경과·일자 완료) SSE 갱신
       └ 전체 20초 도달 → 잔여 결정론 솔버 단독 모드 + 폴백 고지
  TX-final: 세션 GENERATED + outbox ItineraryGenerated(전체 완료 TX만)
```

**설계 원칙**: (a) 첫 1일은 **동기 반환**(5초 게이트), 잔여는 **요청 단위 비동기 워커**(U5-S2 — 큐 없음), (b) 진행 채널은 **SSE**(U5-S1 — ALB 스티키니스 불요, 유휴 60초 > 20초 예산), (c) **LLM은 전 일자 공용 1회**(4단계)·잔여는 점수 재사용(비용·시간 이중 절감, PAT-U5-05), (d) 20초 하드 타임아웃 후 결정론 단독(무한 대기 차단), (e) 취소→부분 초안 저장 + '이어서 생성'(G161), (f) 부분 일정은 **GenerationSession 상태가 진행 정본**(services.md §1.3).

**U5 적용 지점**: M8 `generateItinerary` 오케스트레이션(2구간 분리·SSE), C1 점수 호출(1회), C2 day1/dayN solve, GenerationSession 상태 머신(COLLECTING→DAY1_READY→GENERATED).

**검증 기준**: D38 — 첫 1일 5초·전체 20초 예산 준수(단계 계측), 20초 초과 시 결정론 단독 전환 테스트(US-E5-09). NFR-U5-PERF — LLM 타임아웃 2.5초 < 예산 확인, 점수 재사용(잔여 일자 LLM 재호출 0) 테스트. G161 — 취소 시 부분 초안 저장·'이어서 생성' 진입.

## PAT-U5-02 LLM 게이트웨이 폴백 체인 — 타임아웃→규칙 점수→결정론 솔버 `[U5]`

**문제**: LLM(C1)은 우리가 제어할 수 없는 지연·오류·스키마 위반을 가진다(Medium §6.3). LLM에 생성이 인질 잡히면 벤더 장애가 곧 제품 마비다. 침묵 실패(빈 화면)는 금지(ADR-0011)다. 솔버(C2)조차 제약 과잉으로 해가 없을 수 있다.

**적용(설계)**: U1 **PAT-RES-01 4요소 계약**을 C1·C2에 의무화하고(component-dependency §6), services.md §0.1 실패 표를 폴백 체인으로 구현한다. **생성은 절대 LLM에 인질 잡히지 않는다.**

| 의존 (Port) | 타임아웃 | 재시도 | 서킷 | 폴백 체인 |
|---|---|---|---|---|
| LLM 점수(`LlmPort` 경량) | 2.5초 | 스키마 위반 1회 | 벤더 독립 | **규칙 점수(카테고리-취향 매핑 + 인기 가중) + '기본 모드' 고지** |
| LLM 설명·회고(`LlmPort` 상위) | 티어별 예산 | 1회 | 벤더 독립 | 설명 문구 생략 / "추천 이유를 불러오지 못했어요" |
| TMap 도로거리(`RoadDistancePort` · C2) | 어댑터 | 없음 | 독립 | 직선거리×우회계수 1.3+안전계수(G106) → 네이버 2차 |

```text
[LLM 점수 호출]
  → 성공(2.5초 내·스키마 통과·closed-set) → POI별 취향 점수
  → 타임아웃 2.5초 / 5xx / 스키마 위반(1회 재시도 후) / 서킷 open
       → 규칙 점수 결정론 경로 + "일부 추천이 기본 모드로 생성되었어요"
[후보 풀(M7)]
  → 외부 소스 실패 → 정본 캐시 축소 풀 → 저장 POI + 필수 방문지 최소 풀
[솔버(C2)]
  → 해 없음(제약 과잉) → 위반 완화 제안(자동 변경 없음) + 부분해 초안(G161)
  → 전 경로 실패 → 숙소 + 시각 고정 필수 방문지만 최소 일정 + "다시 시도"
```

**설계 원칙**: (a) 타임아웃은 예산이 아니라 **폴백 전환 트리거**(2.5초 < 5초 예산), (b) 서킷 open/half-open·LLM 오류·솔버 검증 실패는 **전부 계측**(A12, §6.7 관측), (c) 폴백은 반드시 **사용자 고지·계측과 짝**(침묵 실패 금지), (d) **결정론 폴백 모드는 완전 결정성**(동일 입력→동일 출력, PBT (d)), (e) 최종 폴백은 **결정론 솔버가 만든 유효한 최소 일정**(NFR-U5-RES-02 — 빈 화면 불가). LLM 경로는 Medium(RESILIENCY-01)이며 폴백으로 지속.

**U5 적용 지점**: C1 게이트웨이(타임아웃·서킷·규칙 점수 폴백), M8 오케스트레이션(폴백 분기·기본 모드 플래그·degraded 이벤트 필드), C2 solve(해 없음·최소 일정), M7 후보 풀 소비(캐시 축소). 구현은 Resilience4j(U5-S4), fake 실패 주입(D37).

**검증 기준**: RESILIENCY-10 — C1·C2 명시 타임아웃·독립 서킷·폴백(코드 리뷰+계약 테스트). fake로 LLM 타임아웃·5xx·스키마 위반·서킷 open 주입 시 규칙 점수 폴백 실동작, 솔버 해 없음 시 최소 일정·완화 제안 노출 PR CI 통과(US-E5-09). NFR-U5-RES-04 — 결정론 폴백 결정성 PBT. RESILIENCY-01 — LLM Medium 경로 폴백 지속 정합.

## PAT-U5-03 솔버 warm-start·오라클 검증 — 고정 블록 불변·확정 재검증 `[U5]`

**문제**: 재생성·편집 시 사용자가 고정(LOCK)한 슬롯·시각 고정 필수 방문지·체류 고정값이 흔들리면 신뢰가 깨진다(US-E5-10). 솔버 최적화 로직에 버그가 있으면 하드 제약 위반이 그대로 출력된다. 확정 시점과 편집 시점 사이 상황이 바뀔 수 있다.

**적용(설계)**: **warm-start(고정 블록 초기 해 고정) + 독립 오라클 검증**을 분리한다.

- **warm-start(G46)**: 재생성 시 LOCK 슬롯·체류 고정값·수동 추가 POI·시각 고정 필수 방문지를 **불변 고정 블록**으로 놓고 나머지만 재배치. 방식 전환(같이 고르기→완전 AI) 시에도 확정 슬롯 보존(US-E5-10, 진행분 손실 없음).
- **오라클 검증 분리(D37)**: 솔버 산출 배치를 **독립 검증기**로 하드 제약(영업시간·이동 부등식·고정 블록 불변·closed-set) 재확인 — 최적화 로직과 검증 로직 분리(솔버 버그 ≠ 제약 위반 통과). 검증 실패 배치는 결과에서 제외.
- **확정 시점 재검증(G56 — U6 재사용)**: 편집·재계획 확정 버튼 시점에 `C2.validate` 1회 재검증 — 그 사이 변화로 무효화 시 확정 차단·재산출 안내(services.md S2.11).
- **클라 경량 검증기 규칙 공유(D28)**: 편집 즉시 재검증은 클라 경량 검증기(`shared/validation`), 저장은 서버 확정 검증 — **규칙 단일 원천 파생**(통과→거부 불일치 차단, tech-stack §3).

**U5 적용 지점**: C2 `solve`(warm-start)·`validate`(오라클), M8 `validateEdit`(무상태 클라/서버 이중), 일정 상태 머신(Generated↔Draft 재생성 warm-start).

**검증 기준**: D37 하드 제약 4계열 100%(머지 차단) — 오라클 검증이 충돌 무배치·영업시간·closed-set·계정 무결성 강제. PBT (a)(b)(c) — 영업시간 배치·이동 부등식·**고정 블록 불변(warm-start 전후 동일)**(§6.6 1급 대상). D28 — 클라/서버 규칙 동치 이중 실행 계약 테스트.

## PAT-U5-04 트랜잭셔널 처리 — 일자별 독립 TX·확정 스냅샷 원자성 `[U5]`

**문제**: 점진 생성은 부분 결과를 노출한다 — 일자별 저장이 하나의 큰 TX면 부분 노출이 불가하고, 실패 시 전체 롤백된다. 확정(plan 동결)과 POI 스냅샷·이벤트 발행이 원자적이지 않으면 반쪽 확정 상태가 생긴다. LLM·솔버 호출을 TX 안에서 하면 외부 지연이 DB 커넥션을 잡는다.

**적용(설계)**: services.md §0.2·§1.3 TX 경계를 그대로 구현한다.

| 경계 | 원자성 단위 | 원칙 |
|---|---|---|
| 일자별 저장 | **일자별 독립 TX**(TX1 day1, TXn dayN) | 부분 노출 허용 — GenerationSession 상태가 진행 정본(§1.3) |
| 전체 완료 | 세션 `GENERATED` 전이 + outbox `ItineraryGenerated` | **전체 완료 TX만 발행**(부분 완료는 이벤트 없음) |
| 확정 | plan 스냅샷 동결 + `M7.snapshotUserConfirmed` + outbox `ItineraryConfirmed` | **단일 TX 원자성**(호출자 TX 참여 예외 (b), D13·D14) |
| 외부 호출 | LLM(4)·솔버(5·7)·TMap은 **TX 밖** | 외부 지연이 커넥션 점유 금지(§0.2) |
| 편집 | `validateEdit` 무상태 / `applyEdit`만 TX | 재검증은 DB 미변경 |

**설계 원칙**: (a) 아웃박스 패턴(상태 변경 TX에 outbox insert, 커밋 후 J10 릴레이 발행, at-least-once, 구독자 멱등, §0.2), (b) 확정=plan 동결+스냅샷+이벤트 원자(반쪽 확정 불가), (c) current 갱신(U6 재계획)은 plan 불변(D14 — U5는 기준선 공급), (d) 외부 호출 TX 밖(커넥션 보호·PAT-PERF-02 벌크헤드 정합).

**U5 적용 지점**: M8 오케스트레이션 TX 경계(TX1·TXn·TX-final·확정 TX), 아웃박스(`ItineraryGenerated`·`ItineraryConfirmed`·`ItineraryChanged` 편집), 일정 데이터 모델(plan 스냅샷/current 분기 D14).

**검증 기준**: services.md §1.3 — 일자별 독립 TX·부분 노출 테스트, 확정 원자성(plan+스냅샷+이벤트 단일 TX) 테스트. PBT (f) — plan/current 직렬화 왕복. 아웃박스 at-least-once·구독자 멱등(eventId) 계약 테스트(CP5). 외부 호출 TX 밖 검증(커넥션 점유 없음).

## PAT-U5-05 LLM rate-limit·기능별 티어 라우팅·비용 계측 `[U5]`

**문제**: LLM 호출은 곧 운영비다. 무제한 호출·상위 모델 남용·후보 폭발은 비용 폭증(unit-of-work.md U5 리스크)이고, 어뷰징은 보안·비용 이중 위협이다. 국외 벤더면 개인정보 국외 이전 고지 의무가 있다(G181).

**적용(설계)**: 호출량을 **구조적으로 억제**하고 계측한다.

- **전 일자 공용 1회 호출**(NFR-U5-COST-01): 생성 세션당 점수 1회, 잔여 일자는 점수 재사용(재호출 없음) — 20일 여행에서 LLM 1회. 1차 비용 방벽.
- **기능별 티어(D11·G194)**: 취향 점수·재질의=경량, 회고·배치 설명=상위. 티어→모델 매핑 remote config(경량이 상위 단가 미소모).
- **사용자별 rate-limit(SECURITY-11)**: U1 PAT-SEC-03 레이트리미터 재사용(LC-3 전역) — 반복 생성·어뷰징 차단(동시 10 G142 정합).
- **후보 프루닝(G142)**: LLM 입력 후보 지역당 5천 상한·closed-set ID만 전송(토큰 비례 억제).
- **비용 계측(A11·A15)**: 호출별 티어·토큰·추정 비용 로그·메트릭 → SI §8.2 LLM 비용 위젯·쿼터 80% 알람·월 비용 예산 알람.
- **국외 이전(G181)**: 전송 필드 목적 최소화(closed-set ID·취향 축만, 타 계정·내부 지표 미포함 — PAT-U5-06·D31), 국외 벤더 시 처리방침 고지(P7).

**U5 적용 지점**: C1 게이트웨이(티어 라우팅·rate-limit·비용 로그·1회 호출 전략), M8(후보 프루닝), LLM 호출 로그 엔티티, SI 대시보드 편입.

**검증 기준**: NFR-U5-COST — 전 일자 1회 호출·잔여 재호출 0 테스트, 티어별 모델 라우팅 테스트, rate-limit 발동 테스트(SECURITY-11), 후보 상한 프루닝 확인. A11·A15 — LLM 쿼터·비용 메트릭 발행 확인(§6.7). G181 — 전송 필드 최소화(타 계정·내부 지표 미포함) 검증.

## PAT-U5-06 closed-set 그라운딩 구조 — 출력 스키마·ID 화이트리스트 교차 검증 `[U5]`

**문제**: LLM은 존재하지 않는 장소를 환각하거나(그라운딩 실패), 프롬프트 인젝션으로 후보 밖 POI를 삽입할 수 있다. 그라운딩 오염은 일정 전체 신뢰를 무너뜨린다(하드 제약 4계열). LLM을 신뢰하는 설계는 근본적으로 취약하다.

**적용(설계)**: **LLM 신뢰 불요 — 구조적 강제**(G115).

- **closed-set 입력**: M7이 발급한 후보 POI **ID 목록만** LLM에 전달 — LLM은 목록 밖 POI를 알 수 없다(component-dependency §2.2 "데이터 결합 구조적으로 낮음").
- **출력 스키마 검증**: LLM 응답은 JSON Schema 검증(poi_id→score) — 형식 위반은 1회 재시도 후 규칙 점수 폴백(PAT-U5-02).
- **ID 화이트리스트 교차 검증**: C1 출구에서 출력 POI ID가 **후보 풀 ⊆ 확인** — closed-set 밖 ID는 구조적으로 불가하나 **방어적으로 드롭·계측**(services.md S1.2 4단계). 프롬프트 인젝션이 그라운딩을 우회할 수 없다(unit-of-work.md U5 리스크 완화).
- **서버 재조회 경계(D31)**: 컨텍스트는 서버가 요청자 권한으로 재조회 — 타 계정·내부 지표 미주입(NFR-U5-SEC-01·PAT-U5-05).

**U5 적용 지점**: C1 출구 검증기(스키마+ID 교차), M7 후보 풀 공급(closed-set ID), M8 배치(출력 POI ⊆ 후보), 계정 무결성(D31 재조회).

**검증 기준**: G115/D37 — POI 그라운딩 하드 제약 100%(출력 POI ⊆ 후보 풀, 머지 차단). 프롬프트 인젝션·후보 밖 ID 주입 시 드롭·계측 테스트(closed-set 구조적 차단). SECURITY-11 — 서버 재조회 경계로 타 계정·내부 지표 미주입 테스트. 계정 무결성 하드 제약(타 계정 미혼입) 테스트.
