# U7 기록·회고 — 기술 스택 결정 (Tech Stack Decisions)

> 2026-07-04 · CONSTRUCTION · U7 NFR Requirements 산출물
> **전제**: 스택 대분류(RN Expo + Spring Boot Kotlin + PostgreSQL, D02)와 서버·클라이언트 라이브러리 정본(U1 [tech-stack-decisions.md](../../u1-foundation/nfr-requirements/tech-stack-decisions.md))은 **기확정**이다. U7은 그 위에 **① 이미지 스토리지 ② CDN ③ EXIF 제거 위치 ④ 썸네일 생성 위치 ⑤ 사진 업로드 파이프라인 ⑥ 오프라인 큐** 6개 지점을 구체화하며, 나머지(프레임워크·인증·DB 접근·마이그레이션·상태 관리·PBT)는 U1 결정을 상속한다.
> **재결정 금지**: 클라우드=AWS(GD-1)·스토리지=**S3**·CDN=**CloudFront**는 SI §5 예약분이 사실상 확정한다(재확정 아닌 활성화). **회고 = U5 C1 재사용**(D11 — 신규 벤더 없음). 본 문서는 U7의 선택지가 있는 지점(EXIF·썸네일·업로드 방식·오프라인 큐)만 대안 비교 후 권고하고, **프라이버시 트레이드오프를 명시**한다.

## 2. 이미지 스토리지 — Amazon S3 (U7-S1)

| 기준 (DAU 1천 G142) | **A. Amazon S3 (권고)** | B. S3 호환 자체 호스팅(MinIO on ECS/EC2) | C. 타 클라우드 오브젝트 스토리지 |
|---|---|---|---|
| SI 정합 | **SI §5.2가 `trippilot-{env}-photos`를 U7 예약** — 활성화만 | SI 이탈(신규 컴퓨트·운영) | 클라우드 이원화(GD-1 위반) |
| 운영 부담 | **관리형** — 버저닝·암호화·수명주기·내구성 11 9s 기본 | 스토리지 노드 운영·백업·확장 전담 | 별도 계정·네트워크 |
| 보안·암호화 | SSE-KMS·퍼블릭 차단·TLS-only 버킷 정책(SI §5.1) | 직접 구성 | 재구성 |
| 규모 정합(G142) | **적합** — 수십 GB/월, S3 게이트웨이 엔드포인트(SI §2.1 무료)로 NAT 요금 우회 | 과설계(운영 표면) | 이원화 오버헤드 |

- **확정: A(Amazon S3).** SI §5.2가 이미 U7 소비자로 예약한 버킷(`trippilot-{env}-photos` — 버저닝·이전 버전 30일 파기)을 **활성화**한다. S3 공통 기준선(퍼블릭 차단·SSE-KMS·TLS-only·버저닝 SI §5.1) 상속. **S3 Gateway 엔드포인트**(SI §2.1)로 사진 업로드·썸네일 경로가 NAT 요금을 우회하고 프라이빗 접근(SECURITY-07).
- B/C 비채택(1줄): 관리형 S3 대비 자체 호스팅·타 클라우드는 이 규모에서 운영·이원화 비용만 추가(G142·GD-1).

## 3. CDN·서빙 — CloudFront + 서명 URL (U7-S2)

| 기준 | **A. CloudFront + 서명 URL + OAC (권고)** | B. 퍼블릭 버킷 URL | C. 앱(ECS) 프록시 서빙 |
|---|---|---|---|
| 접근 제어 | **서명 URL로 소유자만·짧은 만료** — SECURITY-08 IDOR 차단 | 누구나 URL 추측·유출 시 무제한 접근(사생활 사진 노출) | 소유권 검증 가능하나 ECS 대역폭·지연 |
| 하드닝 | OAC로 버킷 직접 접근 차단·디렉터리 리스팅 불가(SECURITY-09) | 퍼블릭(SI §5.1 전면 차단 원칙 위반) | 오리진 은닉 |
| 성능·비용 | **엣지 캐시** — 이미지 로딩 최소화·오리진 부하↓ | 캐시 이점 있으나 통제 상실 | 캐시 없음·ECS CPU/대역폭 소진 |
| SI 정합 | **SI §5.4가 U7 예약**(OAC·서명 URL·TLS1.2+·표준 로깅) | SI §5.1 위반 | SI 이탈 |

- **확정: A(CloudFront + 서명 URL + OAC).** SI §5.4 예약분 활성화 — OAC(버킷 직접 접근 차단), **서명 URL**(소유자·짧은 만료 — 사진은 사적 자산), viewer TLS 1.2+, 표준 로깅(SECURITY-02), 기본 루트 객체 없음(SECURITY-09). 서명 URL 키는 Secrets Manager(infra §5).
- B 비채택(1줄): 사진은 사적 콘텐츠 — 퍼블릭 버킷은 URL 추측·유출로 사생활 노출(SECURITY-08 위반). C 비채택(1줄): ECS 프록시는 대역폭·지연·비용을 오리진에 집중시켜 규모·복원력 열위.

## 4. EXIF GPS 제거 위치 — 클라 전처리 vs 서버 (U7-C1/S3 · 프라이버시 결정)

**프라이버시 트레이드오프가 핵심인 결정.** G168은 원본 GPS의 기기 이탈 최소화를 권고하고, G75는 원본을 기기 보관으로 확정했다.

| 기준 (G185 기본 제거·G168 권고) | **A. 클라 전처리(압축 단계) + 서버 재검증 (권고)** | B. 서버 단독 제거 | C. 미제거(저장) |
|---|---|---|---|
| **원본 GPS 서버 도달** | **미도달** — 압축·EXIF strip이 업로드 전 수행, 위치 정보가 기기를 떠나지 않음(**G168 권고 충족**) | **도달 후 제거** — 서버가 잠시 원본 GPS 보유(로그·중간 버퍼 노출 리스크) | 저장·서빙에 GPS 잔존(프라이버시 침해) |
| 심층 방어 | **서버 재검증·재제거**(클라 우회·잔여 메타 대비) — 2중 | 단일 지점(클라 우회 시 GPS 저장) | 없음 |
| 위치정보법(D34/N2) 정합 | GPS 최소 수집·파기 원칙에 직합 | 수집 후 제거(최소화 약함) | 위반 |
| 구현 부담 | 클라 이미지 라이브러리 EXIF strip + 서버 방어 | 서버만 | 없음(비채택) |

- **확정: A(클라 전처리 기본 + 서버 재검증).** **G168 권고 정합 — 원본 GPS가 기기를 이탈하지 않는 것이 프라이버시 최적**(G75 원본 기기 보관과 결합 시 자연스러운 지점). 클라 압축 단계에서 EXIF GPS(및 불필요 메타) 제거 후 업로드하고, **서버는 저장 전 EXIF를 재검증·재제거**(클라 버전 우회·잔여 위치 태그 대비 — 심층 방어). 사진 촬영 시각 등 비위치 메타는 별도 필드로 보존(US-E8-02 촬영 시각).
- **트레이드오프 명시**: 클라 단독은 우회 리스크(구형 앱·조작), 서버 단독은 원본 GPS의 일시적 서버 도달. **양쪽 결합(A)** 이 "기기 이탈 방지 + 우회 방어"를 동시 충족한다.
- **경계**: 커뮤니티 게시 시 EXIF 처리(G185 '정확 위치 공개' 명시 선택 게시물만 유지)는 **U10 소관**(unit-of-work U7 제외) — U7은 저장 기본 제거만.

## 5. 썸네일 생성 위치 — 클라 vs 서버 Lambda vs 앱 ECS (U7-S4)

| 기준 (G75 서버 썸네일·G142) | **A. 앱(ECS) 동기 생성 (권고)** | B. 서버 Lambda(S3 이벤트) | C. 클라 생성 |
|---|---|---|---|
| G75 정합 | **서버 썸네일 충족** | 서버 충족 | 클라(G75 '서버 썸네일' 이탈) |
| 신규 AWS 리소스 | **0** — 기존 ECS 태스크 흡수 | **Lambda + EventBridge/S3 트리거 + IAM**(SI 미도입 원칙 §1.2·§4) | 0 |
| 규모 정합 | **적합** — 저빈도(첨부 시점)·CPU 소량, 2GB 태스크 여유(SI §1.2) | 이벤트 파이프라인 운영 표면 추가(G142 과잉) | 신뢰 불가(클라 조작·기기 편차) |
| 트리거 | **attach 확정 API**(S3 이벤트 불요) → Lambda 회피 | S3 PutObject 이벤트 필요 | — |
| 실패 처리 | 동기 실패 시 원본 표시 폴백(NFR-U7-MEDIA-03) | 비동기 재시도 | — |

- **확정: A(앱 ECS 동기 생성).** 클라가 압축본을 S3에 직업로드(§6) 후 **attach 확정 API 호출** 시 서버가 S3에서 압축본을 읽어(S3 게이트웨이 엔드포인트) 썸네일을 생성·업로드한다. **S3 이벤트가 아닌 API 트리거이므로 Lambda·EventBridge 불요**(SI §1.2·§4 "브로커·별도 워커 없음" 정합). 라이브러리: 서버 이미지 처리(예: Thumbnailator/imgscalr류 — Code Generation 확정).
- B 비채택(1줄): DAU 1천·저빈도에서 Lambda + 이벤트 파이프라인은 운영 표면만 추가(G142) — 전환 트리거는 월 수만 장 초과(NFR-U7-SC-04). C 비채택(1줄): 클라 생성은 기기 편차·조작으로 서버 정본(G75)에 부적합.

## 6. 사진 업로드 파이프라인 — 서명 URL 직업로드 vs 서버 경유 (U7-C2/S5)

| 기준 | **A. 서명 URL 직업로드(S3 presigned PUT) (권고)** | B. 서버 경유(멀티파트 → ECS → S3) |
|---|---|---|
| ECS 대역폭 | **오프로드** — 이미지 바이트가 ECS를 통과하지 않음(태스크 2개 대역폭 보호) | 전 이미지가 ECS 통과(CPU·대역폭·메모리 부하) |
| 복원력·재시도 | 클라 큐가 S3에 직접 재시도(NFR-U7-RES-01·03) — 서버 무관 복구 | 서버 경유 → 서버 순단 시 업로드 차단 |
| 비용 | S3 PUT 요금만 | + ECS 처리·NAT 아님(내부) |
| 서버 검증 시점 | **presigned 발급 시(정책 조건: content-length·type) + attach 확정 시(EXIF 재검증·크기·타입·매직 바이트)** | 인라인 즉시 검증 |
| 접근·보안 | presigned PUT은 짧은 만료·1객체 스코프·조건부(SECURITY-05) | 서버 전권 |

- **확정: A(서명 URL 직업로드).** 흐름: ① 클라 압축·EXIF 제거(§4) → ② 서버 **presigned PUT URL 발급**(장소당 20장 상한 검사·content-length ≤5MB·content-type 이미지 조건 부여) → ③ 클라가 S3에 직접 PUT(백그라운드 큐·재시도) → ④ 클라 **attach 확정 API** 호출 → ⑤ 서버 EXIF 재검증·매직 바이트 검증·썸네일 생성(§5)·메타 저장. **ECS 대역폭 오프로드 + 클라 재시도 내성**이 규모·복원력에 최적.
- **B 트레이드오프**: 서버 경유는 인라인 검증이 단순하나, 전 이미지가 ECS를 통과해 저규모 태스크 2개의 대역폭·CPU를 소진하고 서버 순단이 업로드를 차단한다. A의 검증 공백(직업로드 중간)은 **presigned 조건(크기·타입) + attach 재검증**으로 보완한다.

## 7. 오프라인 큐 — shared/storage 로컬 영속 큐 (U7-C3)

| 기준 (NFR-U7-OFF-01~06) | **A. 로컬 영속 큐 + 멱등 키 + 지수 백오프 (권고)** | B. 메모리 큐 |
|---|---|---|
| 크래시·전원 유실 | **영속(기기 저장)** — 앱 재시작·강제 종료 생존(NFR-U7-OFF-01) | 유실(High 워크로드 위반) |
| 재시도 멱등 | 클라 생성 ID 멱등 키 → 중복 레코드 방지(NFR-U7-OFF-03) | 재전송 중복 리스크 |
| 충돌 해소 | 레코드 버전·합집합 병합(G74)과 결합(NFR-U7-OFF-04·05) | — |

- **확정: A.** `shared/storage`(U1 스캐폴드 자리)에 **로컬 영속 큐**(방문 체크·사진 참조·메모·수동 체크인 페이로드) — 앱 재시작 생존·멱등 키·지수 백오프 재시도(사진 3회 NFR-U7-RES-01). 저장 엔진(SQLite/AsyncStorage/MMKV류)은 Code Generation 확정(U1 `shared/storage` 자리 채움). 페이로드 직렬화 왕복·임의 재전송 순서 수렴은 PBT 대상(NFR-U7-OFF-06).

## 8. 회고 연산 — U5 C1 재사용 (U7-S6 · 신규 0)

- **재사용 확정**: M13 회고·전체 요약·스타일 분석은 **U5 C1 LLM Gateway**를 상위 티어(D11)로 호출한다(component-dependency §2.13). **신규 벤더·게이트웨이·시크릿·라우팅 없음** — U5가 확정한 단일 벤더·서버 재조회(D31)·rate-limit·비용 계측·출력 스키마 검증·closed-set(해당 시)을 그대로 상속.
- U7이 소유하는 것: 회고 **프롬프트 구성**(방문·사진·메모·changelog·plan/actual 대조 입력)·**폴백**(기본 카드)·**게이트**(10곳)·**재생성 정책**(덮어쓰기 경고 G78)·**비동기 트리거**(DayClosed/TripEnded). 이는 앱 레벨 로직이며 인프라·벤더 표면 증분 0.
