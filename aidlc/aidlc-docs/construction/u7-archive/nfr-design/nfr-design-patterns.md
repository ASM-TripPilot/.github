# 1. U7 신규 패턴

## PAT-U7-01 오프라인 우선 입력 큐·재시도·충돌 해소 — 레코드 단위·합집합 (G74) `[U7]`

**문제**: 산간·해외 로밍에서 네트워크가 끊겨도 방문 체크·사진·메모 입력이 유실되면 안 된다(M12 High). 재접속 시 서버 데이터와 충돌하면 어느 쪽도 소실되면 안 된다.

**적용(설계)**:
- **로컬 영속 큐**(`shared/storage` U7-C3): 입력을 기기 영속 저장에 적재하고 '동기화 대기' 표시(NFR-U7-OFF-01). 앱 재시작·크래시 생존.
- **멱등 동기화**: 각 항목에 클라 생성 ID(멱등 키) 부여 → 재시도·부분 성공에도 중복 레코드 0(NFR-U7-OFF-03).
- **충돌 해소 2계열**(G74):
  - **레코드 단위 버전 비교** → 상충 필드(체류 시각 등)만 항목별 사용자 선택(NFR-U7-OFF-04).
  - **사진·메모 추가 = 합집합 병합**(양쪽 보존, 충돌 아님 — NFR-U7-OFF-05). 사용자 선택 최소화.
- **무손실 불변식**: 어떤 동기화 경로에서도 데이터 소실 0(합집합·양쪽 보존).

```text
[오프라인 입력]--영속 큐(멱등 키)-->[동기화 대기]
        네트워크 복구
             |
   ┌─────────┴──────────┐
   |  서버 충돌 있음?     |
   └─────────┬──────────┘
      아니오 |        예
     [즉시 반영]  ┌────┴─────────────────────┐
                 | 사진·메모 추가 → 합집합 병합 |
                 | 상충 필드 → 레코드 버전 비교  |
                 |            → 항목별 사용자 선택 |
                 └────────────┬─────────────────┘
                     [무손실 병합 완료]→ M8 배지 동기화(E[17])
```

**텍스트 대안**: 오프라인 입력은 멱등 키와 함께 로컬 영속 큐에 '동기화 대기'로 적재된다. 네트워크 복구 시 서버 충돌이 없으면 즉시 반영, 충돌이 있으면 사진·메모는 합집합 병합하고 상충 필드만 레코드 버전 비교로 사용자가 선택한다. 어떤 경로에서도 데이터 무손실이며, 병합 완료 후 M8 변경 배지를 동기화한다.

**U7 적용 지점**: `shared/storage` 로컬 큐, M12 동기화 API(멱등·병합), 충돌 해소 UI, 오프라인 중 당일 회고 보류(복구 후 생성/기본 카드).

**검증 기준**: **PBT 1급**(NFR-U7-OFF-06) — 임의 순서·중복 동기화 시퀀스에 대해 (a) 수렴(동일 결과), (b) 데이터 무손실(합집합·양쪽 보존), (c) 멱등(중복 레코드 0), (d) 직렬화 왕복(큐 페이로드). fast-check(클라)·Kotest(서버 병합). RESILIENCY-01, G74, Δ6/D24.

## PAT-U7-02 사진 업로드 파이프라인 — 클라 압축·EXIF 제거·서명 URL 직업로드 `[U7]`

**문제**: 사진(장소당 20장·최대 5MB)을 저규모 ECS 태스크 2개의 대역폭을 소진하지 않고, 원본 GPS를 서버에 노출하지 않으며, 불안정 네트워크에서도 유실 없이 업로드해야 한다(프로젝트 최초 미디어 파이프라인).

**적용(설계)** — tech-stack §4·§5·§6 확정 흐름:
1. **클라 압축**(5MB/2048px NFR-U7-MEDIA-01) + **EXIF GPS 제거**(§4 A — 원본 GPS 기기 이탈 방지 G168) → 원본은 기기 보관(G75).
2. 서버 **presigned PUT URL 발급**(장소당 20장 검사·content-length ≤5MB·type 이미지 조건 — SECURITY-05).
3. 클라 **S3 직업로드**(백그라운드 큐·지수 백오프 3회 — ECS 대역폭 오프로드 NFR-U7-PERF-01, 재시도 내성 NFR-U7-RES-01·03).
4. 클라 **attach 확정 API** → 서버 **EXIF 재검증·매직 바이트·크기 재확인**(심층 방어) → **썸네일 생성**(앱 ECS 동기 §5, Lambda 미도입) → 메타 저장.
5. **입력 독립**: 메모·방문 체크는 사진 실패와 무관하게 저장(NFR-U7-RES-02).

**U7 적용 지점**: 클라 이미지 압축/EXIF strip, presigned 발급 API, S3 업로드 큐, attach 확정 API·서버 썸네일.

**검증 기준**: SECURITY-05(업로드 검증 2중 — presigned 조건 + attach 재검증), G168(EXIF strip 클라+서버), 업로드 실패 큐·재시도 테이블 주도 테스트(NFR-U7-RES-01), 사진/방문 저장 독립 계약 테스트(NFR-U7-RES-02). RESILIENCY-10.

## PAT-U7-03 CDN 서명 URL 접근 제어 — OAC·소유권·짧은 만료 `[U7]`

**문제**: 사진은 사적 자산이다. URL 추측·유출로 타인이 접근하거나 버킷이 직접 노출되면 사생활 침해다(SECURITY-08·09).

**적용(설계)**:
- **OAC**(SI §5.4): CloudFront만 S3 오리진 접근 — 버킷 퍼블릭 전면 차단(SI §5.1)·직접 접근 불가·디렉터리 리스팅 불가(SECURITY-09).
- **서명 URL**: 사진·썸네일 조회는 **소유권 검증 후** 짧은 만료의 CloudFront 서명 URL 발급(소유자만·시간 제한 — SECURITY-08 IDOR 차단). 서명 개인 키는 Secrets Manager(§0 PAT-SEC-05·infra §5).
- **소유권 스코핑**: 서명 URL 발급 자체가 accountId 스코핑 뒤에 위치(URL 발급 API가 인증·소유권 게이트).

**U7 적용 지점**: 사진 조회 시 서명 URL 발급 API(소유권 검증), CloudFront 배포 설정(OAC·서명·TLS1.2+·로깅), Secrets Manager 서명 키.

**검증 기준**: SECURITY-08(비소유자 서명 URL 발급 거부 테스트)·SECURITY-09(버킷 직접 GET 차단·OAC)·만료 경과 URL 접근 거부. SECURITY-01(TLS-only·SSE-KMS 상속).

## PAT-U7-04 changelog 통합 diff 보관·재구성 — 누적 재생=스냅샷 동등성 (G132) `[U7]`

**문제**: 여행 중 모든 변경(Plan-B·공동편집·어시스턴트)이 통일된 diff 스키마로 보관·열람되고, diff 누적 재생이 실제 스냅샷과 정확히 일치해야 한다(changelog 스키마는 후속 U9~U11 공용 — 초기 실수 시 파급 최대).

**적용(설계)**:
- **통합 diff 스키마**(G57/G132): 항목 단위 diff = 행위자·출처 유형·사유·전/후 값. POI는 내부 ID 참조(스냅샷 아닌 참조 — component-dependency §3.1). **스키마 버전 필드 예약**(후속 유닛 진화 대비 — unit-of-work U7 리스크 완화).
- **plan/current/actual/changelog 구분 저장**(D14/US-E8-04): plan(확정 불변 스냅샷)·current(가변)·actual(실제 방문)·changelog(변경 이력)를 분리. 회고·비교는 plan↔actual 대조(대조 연산 M12 소유).
- **diff 누적 재구성**: current = plan + changelog diff 누적 재생(G132) — 임의 변경 시퀀스에 대해 재생 결과가 current와 일치.
- **소비**: U6 재계획 diff·U8 배지 동기화(E[17])·U10 공개 스냅샷·U11 공동편집 이력의 참조 정본.

**U7 적용 지점**: M12 changelog 스키마(버전 필드)·diff 기록/열람, plan/actual 대조 함수.

**검증 기준**: **PBT 1급**(unit-of-work U7 DoD) — 임의 변경 시퀀스에 대해 **changelog diff 누적 재구성 = current 스냅샷 동등성**(G132 핵심 속성), plan/actual 대조 함수, 직렬화 왕복. changelog 스키마 무결성(전/후 값·행위자 필수) 하드 제약 테스트(D37 계정 무결성 계열).

## PAT-U7-05 회고 폴백 — C1 실패/타임아웃→기본 카드·부분 데이터 명시 `[U7]`

**문제**: LLM 회고 생성이 실패·지연·저품질일 때 빈 화면을 주면 회고 가치가 붕괴한다. 부분 데이터(사진 0장·GPS 미동의)에서도 침묵하면 안 된다(ADR-0011).

**적용(설계)**:
- **C1 계약 상속**(§0): 회고 C1 호출은 U5 서킷·타임아웃(권고 25초 예산 NFR-U7-REF-03) 상속 — U7은 **폴백만 소유**.
- **기본 카드 폴백**(US-E8-06·08): 실패·타임아웃 시 '방문 N곳·이동 Nkm·사진 N장' 기본 카드 제공. 기본 카드 위에 직접 회고 작성 가능(US-E8-07).
- **부분 데이터 명시**(NFR-U7-RES-05): 가용 데이터만으로 생성하고 누락을 명시('사진 없음'·'위치 기록 없음'). 생성 불가 항목(사진 하이라이트·지도 경로)은 제외하되 침묵 금지.
- **10곳 게이트**(스타일 분석 G76): 빈약 입력 차단 — 미달 시 미제공 안내.
- **재생성·덮어쓰기**(G78/C11): 수정본 존재 시 덮어쓰기 경고 후 확인 시에만 교체, 종료 후 자동 갱신 금지(수동 재생성만).
- **비동기 트리거**: DayClosed→당일 회고, TripEnded→전체 요약(component-dependency §40). ReflectionReady 발행까지가 U7(발송 U8).

**U7 적용 지점**: M13 회고 오케스트레이션(폴백·부분 데이터·게이트·재생성), C1 호출 어댑터(U5 재사용), ReflectionReady 발행.

**검증 기준**: RESILIENCY-10(C1 실패→기본 카드 폴백 테스트), US-E8-06(부분 데이터 명시)·07(덮어쓰기 경고)·08(요약 폴백), 회고 폴백율 커스텀 메트릭(§0 PAT-OBS-03).

## PAT-U7-06 GPS 파기·법정 로그 분리 — 옵트인 철회·탈퇴 즉시 파기 (D34/N2) `[U7]`

**문제**: 위치정보법상 GPS 옵트인 철회·탈퇴 시 위치 데이터를 즉시 파기해야 하나, 수집·이용 사실 법정 로그는 6개월+ 보존해야 한다(파기 대상과 보존 대상이 상반).

**적용(설계)**:
- **파기 대상**: GPS 발자취 폴리라인·GPS 귀속(기록 연동). 원시 좌표는 U6에서 이미 파기(G73 — 폴리라인만 보존). 옵트인 철회·탈퇴 트리거 시 **즉시 파기**(D34).
- **보존 대상**: 위치정보 수집·이용 **사실 확인자료(법정 로그)** — **U1 append-only 테이블 재사용**(SI §5.3·NFR-U7-SEC-07). 6개월+ 보존·앱 역할 삭제 불가(2중 강제 유지 — 파기 잡도 `app_migrate` 권한 별도).
- **분리 원칙**: 파기 잡이 위치 데이터를 지우되 법정 로그는 건드리지 않음(권한·경로 분리). 계정 삭제 연쇄(D18)에서도 법정 보존 집합만 잔존(U8 연쇄와 정합).
- **파기 실행**: ShedLock 스케줄러(SI §1.2) 기반 파기 잡 또는 철회 이벤트 즉시 처리(Code Generation 확정).

**U7 적용 지점**: GPS 파기 핸들러(옵트인 철회·탈퇴 구독), 법정 로그 append-only 기록(U1 재사용), 파기/보존 경로 분리.

**검증 기준**: D34(철회·탈퇴 후 GPS 데이터 잔존 0 테스트)·N2(법정 로그 잔존·append-only)·SECURITY-14(앱 역할 법정 로그 삭제 불가 상속), 파기 대상과 보존 대상 분리 계약 테스트. PBT: 삭제 연쇄 후 잔존 = 법정 보존 집합과 정확히 일치(U8 연쇄와 접속).
