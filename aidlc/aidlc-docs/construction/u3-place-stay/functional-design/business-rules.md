## A. 숙소 탐색

### BR-U3-01 여행지 단일 입력 탐색 — 날짜·인원 미수집
- **조건**: 숙소 탐색 요청(`searchStays`) 시.
- **동작**: 여행지(또는 '내 주변')만 입력받아 탐색한다 — **체크인/체크아웃 날짜·인원은 탐색 단계에서 받지 않는다**. 여행지가 비어 있으면 탐색 버튼은 비활성 유지. 일정 생성에 필요한 체크인/아웃 날짜는 등록(BR-U3-10)에서 수집한다.
- **위반 시 처리**: 여행지 없는 탐색 요청은 클라이언트 게이트+서버 `ValidationFailed`로 거부(빈 여행지 조회 금지).
- **근거**: US-E3-01, D09, ADR-0002.

### BR-U3-02 탐색 데이터 소스 — TourAPI+지도, OTA는 딥링크만
- **조건**: 탐색 결과 구성 시.
- **동작**: 결과는 TourAPI+지도 장소 검색(M7 경유)으로 구성한다. **국내 OTA 데이터 크롤링 금지** — 외부 OTA는 숙소명 기반 검색 딥링크로만 연결(부정경쟁 민사 리스크 회피). 소스별 카테고리 코드는 M7이 단일 표준 스키마로 정규화(ADR-0009).
- **위반 시 처리**: OTA 페이지·데이터 크롤링 경로는 구조적으로 존재하지 않음(딥링크 조립만) — 코드 리뷰 차단 항목.
- **근거**: US-E3-01, D09, PRD 04 데이터 모델 註.

### BR-U3-03 대표 가격대 정적 표시 — 정확 가격 일괄 조회·캐싱 금지
- **조건**: 탐색 결과 카드·목록 표시 시.
- **동작**: 카드는 숙소명·위치·**대표 가격대(정적 구간·등급)**를 기본 표시한다(콘텐츠 메타 — 캐싱 가능, INV-CACHE2 예외). **결과 목록 전체에 대한 정확 1박 가격 일괄 라이브 호출을 하지 않으며, 정확 가격 값은 캐싱하지 않는다**(비용·지연·OTA 약관).
- **위반 시 처리**: 목록 렌더 시 정확 가격을 조회·저장하는 것은 결함(약관 위반급) — INV-CACHE2 방어.
- **근거**: US-E3-01, D09, PRD 04 註 ②, G196.

### BR-U3-04 라이브 가격 조회 — '가격 보기' 명시 액션 한정
- **조건**: 사용자가 결과 카드·상세의 '가격 보기'를 탭할 때(`getLivePrice`).
- **동작**: 날짜·인원 바텀시트 입력을 받은 뒤 정확 가격을 **라이브 조회**한다. 입력값은 세션 내 재사용. 조회 결과는 표시만 하고 저장하지 않는다(캐싱 0).
- **위반 시 처리**: 라이브 조회 실패 → `UpstreamUnavailable` → "외부 OTA에서 확인" 딥링크 대체 제시(침묵 실패 금지).
- **근거**: US-E3-01, G33, D09, G196.

### BR-U3-05 '내 주변' 위치 권한 폴백
- **조건**: '내 주변' 탐색 시도 시(위치 just-in-time 발화 1번째 지점 — U1 위치 동의 3층 소비 G182).
- **동작**: 위치 권한 허용이면 단말 좌표로 조회. **권한 거부면 등록 숙소 또는 여행지 중심 좌표로 대체 조회**한다(중단 없음). U1 `effectiveCapabilities`로 서버 위치 서비스 가부를 판정(L1∧L2 필요).
- **위반 시 처리**: 권한 거부만으로 탐색 흐름을 중단하는 것은 결함 — 폴백 좌표로 계속.
- **근거**: US-E3-01 예외, US-E1-04, G182(U1 INV-L1).

## B. 필터·정렬

### BR-U3-06 숙소 유형 필터 전면 제공
- **조건**: 필터 적용 시.
- **동작**: 숙소 유형(호텔·게스트하우스·펜션·모텔·호스텔·아파트·B&B·한옥 등) 필터를 전면 제공한다 — 유형은 가격과 분리된 정적 콘텐츠. 한국형 유형 분류는 **TourAPI cat3를 정본**으로 표준화한다.
- **위반 시 처리**: 소스별 유형 코드 불일치는 M7 표준화로 흡수 — 표준 밖 유형 노출은 결함.
- **근거**: US-E3-02, D09, ADR-0009.

### BR-U3-07 편의시설 필터 — 채움률 검증 항목만 노출
- **조건**: 편의시설(amenities) 필터 구성 시.
- **동작**: TourAPI 필드 **채움률이 검증된 항목만** 노출한다(핵심 3종 주차·조식·와이파이부터 확대). 편의시설 표시는 **가격 비교 화면이 아닌 곳**에 배치(Booking 약관). 미확보 시 핵심 3종 폴백.
- **위반 시 처리**: 채움률 미검증 항목을 필터로 노출(대량 0건 유발)하는 것은 결함.
- **근거**: US-E3-02, PRD 04-2.

### BR-U3-08 가격순 정렬 미제공 — '대표 가격대순' 대체
- **조건**: 정렬 옵션 구성 시.
- **동작**: 정확 가격에 의존하는 **가격순 정렬은 제공하지 않는다**(라이브 전수 호출 전제이므로). 대신 **'대표 가격대순'**(정적 구간 기준)으로 대체한다. 가격대 필터도 대표 가격대(구간) 기준으로 적용.
- **위반 시 처리**: 가격순 정렬 노출은 목록 라이브 전수 호출을 유발하는 결함(BR-U3-03 위반).
- **근거**: US-E3-02, D09, PRD 04-2.

### BR-U3-09 거리 필터·정렬 기준점 — 검색 여행지 중심 좌표 직선거리
- **조건**: 거리 필터·정렬 적용 시.
- **동작**: **검색 여행지 중심 좌표를 기준점**으로 한 **직선거리**로 적용한다. **차량/도보 예상 소요시간은 어떤 화면에도 표시하지 않는다**(거리만) [D25].
- **위반 시 처리**: 소요시간 표기는 D25 위반 결함. 필터 적용 결과 0건이면 원인 필터를 표기하고 완화 제안(BR-U3-10 아님 — 0건 진단).
- **근거**: US-E3-02, G34, D25/Δ1.

## C. 숙소 등록

### BR-U3-10 등록 필수 필드·날짜 순서 검증 (하드 제약)
- **조건**: 숙소 등록(`registerStay`)·수정(`updateStay`) 저장 시.
- **동작**: 숙소(장소)·체크인/체크아웃 날짜·인원을 **필수**로 수집한다(예약번호·금액은 선택). `checkOut > checkIn` ∧ 두 날짜 모두 존재를 검증한다(INV-STAY1).
- **위반 시 처리**: `ValidationFailed` — 누락·역전 항목을 인라인으로 표시하고 저장 차단. **등록 숙소 필수 필드·날짜 순서(체크아웃>체크인) 검증은 D37 하드 제약, 머지 차단 테스트** [U3 DoD].
- **근거**: US-E3-06, D15, PRD 04-6 예외, PBT U3-P6.

### BR-U3-11 좌표 미확정 시 "지도에서 위치 확인" 강제
- **조건**: 등록 시 좌표 확정 여부(coordConfirmed) 판정 시.
- **동작**: 좌표가 확정되지 않은 등록은 `CoordinateUnresolved`를 반환해 **"지도에서 위치 확인" 단계를 강제**한다. coordConfirmed=true가 되기 전에는 거점 자격·일정 생성 기준점 투입을 보류한다(INV-STAY2). 국내 좌표 범위 검증(G120).
- **위반 시 처리**: 좌표 미확정 숙소를 거점으로 연결·일정 생성에 투입하는 것은 결함 — CP1 소비(U4)에서도 동일 차단 [CP1 시나리오 2].
- **근거**: US-E3-06·08 예외, D15, G120.

### BR-U3-12 계정 레벨 풀 — 여행 없이 등록 가능
- **조건**: 등록 저장·귀속 시.
- **동작**: 등록 숙소는 **계정 레벨 풀**에 저장한다 — 여행 없이 등록·유지 가능(D15). 여행 연결(BaseAssignment)은 별개 액션이며, 여행 삭제가 SavedStay를 파기하지 않는다(INV-STAY3).
- **위반 시 처리**: 등록을 특정 여행에 강제 귀속시키는 것은 결함(계정 풀 원칙 위반). 여행 밖 등록 시 연결은 U4 화면에서 명시 선택(G150 대안 C).
- **근거**: US-E3-06·09, D15, G128/G150.

### BR-U3-13 등록 3경로 단일 폼 — 자동 채움
- **조건**: 직접 등록 진입 시(registrationPath).
- **동작**: 세 경로를 단일 등록 폼으로 수렴한다 — (a) 지도/장소 검색 자동 채움(M7), (b) OTA URL 붙여넣기 파싱 프리필(BR-U3-25), (c) 지도 핀 직접 지정. 검색·복귀 핸드오프 시작 시 숙소명·위치·가격대가 자동 채워지고 날짜·인원만 입력하면 완료.
- **위반 시 처리**: 경로별 상이한 폼·중복 정의는 결함(등록 폼은 US-E3-08 정본 — 여행 화면은 진입점만).
- **근거**: US-E3-06·08, G31.

### BR-U3-14 등록 후 [AI 일정 생성하기] 온램프·수정·취소
- **조건**: 등록 완료 직후 및 사후 관리 시.
- **동작**: 등록 완료 시 해당 숙소를 거점으로 한 **[AI 일정 생성하기] 진입점을 즉시 노출**한다. 등록한 숙소의 날짜·인원·예약번호·금액은 나중에 수정 가능, 등록 취소 가능. 여행 연결 숙소의 삭제·기간 변경은 영향 블록 나열 후 **차단형 확인**(G39), 날짜 변경 시 거점 비중첩 재검증(D15).
- **위반 시 처리**: 연결된 숙소를 확인 없이 삭제·변경하는 것은 결함(파괴적 변경 방어 G39).
- **근거**: US-E3-06, US-E3-09, G39, D15.

## D. 거점 연결 준비 (CP1 공급 — 여행 내 비중첩은 U4 소비)

### BR-U3-15 여행 거점 연결 — 일자별 다중 거점
- **조건**: 숙소를 여행 거점으로 연결(`linkToTrip`) 시.
- **동작**: coordConfirmed=true인 SavedStay를 특정 여행의 dateRange 거점으로 연결한다(BaseAssignment). 하나의 여행에 일자별 서로 다른 거점 등록 지원(다도시). 체크인/아웃이 여행 기간을 이탈하면 `PeriodMismatch(proposal)`로 기간 확장을 경고 반환(자동 변경 없음).
- **위반 시 처리**: 좌표 미확정 숙소 연결은 차단(INV-BASE2). 기간 자동 변경은 금지 — 사용자 확인.
- **근거**: US-E3-07, D15, G41.

### BR-U3-16 여행 내 거점 날짜 비중첩 — 판정 공급(검증 실행은 U4)
- **조건**: 같은 여행에 연결된 거점 구간의 겹침 판정 시.
- **동작**: 같은 tripId에 연결된 active 거점끼리 **날짜 비중첩을 판정**한다(INV-BASE1). **계정 풀의 미연결 숙소는 검증 대상 아님**. 겹치면 `ConflictDetected(overlapDates)`로 구간 조정을 요구. **U3은 비중첩 판정 순수 함수와 계약을 공급하고, 여행 컨텍스트 검증·UI 완성은 U4가 소비**(unit-of-work U3 §제외).
- **위반 시 처리**: 겹침을 허용해 거점이 둘인 상태로 일정 생성 진입하는 것은 결함(U4 소비 측에서도 방어) [CP1].
- **근거**: US-E3-07, D15, PBT U3-P5, CP1.

### BR-U3-17 스마트 기본 거점 — 비차단 안내
- **조건**: 거점 타임라인(`getBaseTimeline`) 산출 시.
- **동작**: 다중 숙소는 체크인 순 전환, **공백일은 직전 숙소, 첫날 공백은 여행지 중심 좌표**를 기본 거점으로 채운다(isSmartDefault=true). 스마트 기본 거점은 **비차단 안내**(사후 수정 가능 배지).
- **위반 시 처리**: 스마트 기본 거점으로 사용자 흐름을 차단하는 것은 결함(비차단 원칙 PRD 05-6).
- **근거**: US-E3-07, G41, G50, D15.

## E. 숙소 ID 통합

### BR-U3-18 내부 숙소 ID 정본 + 소스별 외부 ID N:1 매핑 (하드 제약)
- **조건**: 숙소 식별 해석(`resolveStayIdentity`)·등록 시.
- **동작**: **내부 canonical 숙소 ID를 정본**으로 하고 소스별 외부 ID(OTA·지도)를 N:1로 매핑한다(INV-SID1·2). `(source, externalId)`는 정확히 1개 내부 ID로 결정적 해석(라운드트립 보장).
- **위반 시 처리**: 하나의 외부 ID가 두 내부 ID에 매핑되는 것은 하드 결함 — INV-SID2 저장 계층 거부, PBT U3-P2.
- **근거**: US-E3-05, D17, PBT U3-P2, U3 DoD.

### BR-U3-19 좌표+이름 유사도 자동 매칭 — 결정성·대칭성·보수 임계
- **조건**: 신규 소스 ID의 canonical 귀속 판정 시.
- **동작**: 좌표(50m 근접)+명칭 유사도로 자동 매칭한다 — **결정적**(동일 입력 동일 결과)·**대칭적**(A↔B). 오매칭(동명·근접 병합 오류) 위험에 대비해 **매칭 임계값은 보수 설정**하고 운영자 보정 여지를 둔다(operatorOverride).
- **위반 시 처리**: 매칭이 비결정적이거나 비대칭이면 결함(PBT U3-P4). 임계 미달은 신규 canonical 생성(과병합보다 과분리를 택함 — 보수적).
- **근거**: US-E3-05, D17, G133/G148, PBT U3-P4.

### BR-U3-20 동일 숙소 복수 OTA 묶기
- **조건**: `getOtaLinkOptions` — OTA 선택 목록 구성 시.
- **동작**: **내부 숙소 ID 기준으로** 동일 숙소에 연결된 OTA를 묶어 OTA명 목록을 반환한다 — 사용자가 어느 링크로 나갈지 선택.
- **위반 시 처리**: 외부 ID 기준 묶기(동일 숙소 분산 표시)는 결함(내부 ID 정본 원칙).
- **근거**: US-E3-05, D17.

## F. OTA 딥링크·수수료 고지

### BR-U3-21 숙소명 검색 딥링크만 — 자체 예약·결제 미제공
- **조건**: OTA 예약 이동(`buildDeeplink`) 시.
- **동작**: **숙소명 기반 OTA 검색 딥링크**만 생성한다(파트너별 URL 템플릿 조립, 실호출 없음). 앱은 자체 예약·객실 선택·예약 확정·인앱 결제를 **제공하지 않으며** 모든 결제 흐름을 외부 OTA로 위임한다.
- **위반 시 처리**: 앱 내 예약·결제 경로 도입은 범위 위반. 딥링크 조립 외 OTA API 실호출은 결함.
- **근거**: US-E3-05, D09, ADR-0003·0012.

### BR-U3-22 제휴 수수료 고지 — 외부 이동 직전 표시
- **조건**: 외부 OTA 이동 직전.
- **동작**: **"예약·결제는 외부 OTA에서 진행되며 TripPilot은 제휴(어필리에이트) 수수료를 받을 수 있다"**는 제휴 링크 안내·수수료 고지를 표시한다(추가 비용 없음 문안 포함). '다시 보지 않기'(setDisclosureHidden) 설정 연동 — 숨김 설정 시 showDisclosure=false.
- **위반 시 처리**: 고지 없는 외부 이동은 광고·제휴 표시 규제 위반(컴플라이언스 결함).
- **근거**: US-E3-05, PRD 04-5, PRD 12-12.

### BR-U3-23 리뷰·평점 미표시 — 외부 OTA 위임
- **조건**: 숙소 상세(`getStayDetail`) 구성 시.
- **동작**: 상세는 위치(핀)·사진·대표 가격대·편의시설·체크인/아웃 **시각** 등 정적 콘텐츠만 구성한다. **리뷰·평점은 앱 내부에 표시하지 않고**(원문·점수 모두) "리뷰·평점 보기 → 외부 OTA" 딥링크로 위임한다. 데이터 누락 필드는 빈칸이 아닌 **"미확인"** 명시값.
- **위반 시 처리**: 리뷰·평점 필드를 앱 내부 화면에 표시하는 것은 결함(INV-MTX4). 빈칸 방치는 결함(미확인 표기 강제).
- **근거**: US-E3-03, PRD 04-3, G199, INV-MTX4.

### BR-U3-24 딥링크 실패 우회 경로
- **조건**: OTA 링크가 깨졌거나(404·만료) 응답하지 않을 때.
- **동작**: "지금 이 OTA로 연결할 수 없어요"를 표시하고, 가능하면 **같은 숙소의 다른 OTA 링크** 또는 **숙소 장소 검색**으로 우회 경로를 제시한다.
- **위반 시 처리**: 링크 실패 시 침묵·막다른 흐름은 결함(ADR-0011 침묵 실패 금지).
- **근거**: US-E3-05 예외, D09, ADR-0011.

## G. URL 파싱 (화이트리스트)

### BR-U3-25 OTA 링크 붙여넣기 — 화이트리스트 패턴 파싱만 (fetch 없음)
- **조건**: OTA 예약 링크 붙여넣기(`parseOtaLink`) 시.
- **동작**: **지원 도메인 화이트리스트 내 URL 패턴 파싱만으로**(페이지 fetch 없음) 숙소명·위치를 추출해 등록 폼 프리필을 만든다. 화이트리스트 밖 URL·패턴 불일치는 전부 거부한다.
- **위반 시 처리**: 화이트리스트 밖 URL 파싱 시도·페이지 fetch는 결함(SSRF·약관 리스크 방어) — 전부 `LinkParseFailed`. **화이트리스트 밖 URL 전부 거부·파싱 실패 시 폴백 경로는 PBT U3-P7 대상** [U3 DoD].
- **근거**: US-E3-08, G31, SECURITY-05, PBT U3-P7.

### BR-U3-26 파싱 실패 시 수동 핀 지정 폴백
- **조건**: URL 추출 실패·장소 미검색 시.
- **동작**: "위치를 찾지 못했어요. 지도에서 직접 지정해 주세요"로 안내하고 **수동 핀 지정 흐름(PIN_DROP)**으로 넘긴다. 지도/장소 검색 API 무응답도 동일 폴백(침묵 실패 금지).
- **위반 시 처리**: 파싱 실패로 등록 흐름이 막히는 것은 결함(항상 핀 지정 폴백 존재).
- **근거**: US-E3-08 예외, G31, ADR-0011.

## H. 복귀 핸드오프 카드

### BR-U3-27 복귀 핸드오프 카드 — 24h·1회 노출
- **조건**: OTA 딥링크 이탈 후 앱 포그라운드 복귀 시(`getReturnHandoffCard`).
- **동작**: 딥링크 이탈 후 **24시간 내 첫 복귀에 최근 이탈 숙소 1건에 대해 1회** "방금 본 [숙소명], 예약하셨나요? → 거점으로 등록하기" 카드를 노출한다(handoffShownAt 기록). 카드는 빠른 등록(OTA_RETURN 경로, BR-U3-13)으로 연결.
- **위반 시 처리**: 24h 경과분·복수 숙소 동시 노출은 결함. 노출은 정확히 1회(INV-CLICK2, PBT U3-P8).
- **근거**: US-E3-05, G32.

### BR-U3-28 카드 무시 시 재노출 억제 — 비강제
- **조건**: 사용자가 핸드오프 카드를 무시(디스미스)할 때(`dismissHandoffCard`).
- **동작**: handoffDismissedAt를 기록해 **재노출을 억제**한다. 사용자는 카드를 무시하고 계속 탐색할 수 있다(강제 아님) — 이후 동일 이탈 건에 `null` 반환.
- **위반 시 처리**: 무시 후 재노출은 결함. 등록 강제는 결함(비강제 원칙).
- **근거**: US-E3-05, G32.

## I. 위시리스트

### BR-U3-29 위시 저장 — 계정 귀속·기기 전환 유지
- **조건**: 숙소 저장(`addToWishlist`) 시.
- **동작**: 저장은 **로그인 사용자 계정에 귀속**되어 기기를 바꿔도 유지된다(로그인 필수 전제 D22). `(accountId, stayRef)` 유니크(재저장 멱등). 메모는 SECURITY-05 검증.
- **위반 시 처리**: 비로그인 저장·기기 로컬 저장은 결함(계정 귀속 원칙). 소유권 검증(SECURITY-08).

### BR-U3-30 위시 카드 표시·가격 변동 고지
- **조건**: 위시 목록(`listWishlist`) 조회 시.
- **동작**: 각 항목을 숙소명·대표 사진·가격대·위치 카드로 표시하고, **"외부 OTA 기준, 변동 가능"**을 병기한다(INV-WISH2). 원본 소실 항목은 저장 당시 캐시로 표시+"최신 정보 확인 불가" 플래그(staleFlag).
- **위반 시 처리**: 저장 시점 가격을 확정 가격으로 표시하는 것은 결함. 빈 카드 표시는 결함.
- **근거**: US-E3-04 예외, G103.

### BR-U3-31 위시·저장 POI·등록 숙소 별도 목록
- **조건**: 목록 관리 전반.
- **동작**: 숙소 위시리스트와 저장 POI(SavedPlace)는 **별도 목록 2개**로 관리한다(G129 원칙). 위시·등록 통합 목록(`listMyStays`)은 출처 라벨('외부 OTA 예약 등록'/'앱 내 저장')로 구분하고, 저장 항목엔 '등록하기' 액션을 둔다.
- **위반 시 처리**: 위시와 저장 POI의 목록 혼합·상호 오염은 결함.
- **근거**: US-E3-04, US-E3-09, G103, G129.

## J. POI 정본·캐싱 정책

### BR-U3-32 canonical POI ID 무결성 (하드 제약)
- **조건**: POI 수집·정규화·식별 해석(`resolvePoiIdentity`) 시.
- **동작**: 내부 canonical POI ID를 정본으로 하고 소스별 place_id를 N:1 매핑한다. **동일 실세계 장소(좌표 50m 근접+명칭 유사도)는 정확히 1개 canonical**(INV-POI1). 한국어명 정본+영문 alias(G148), 한국어 필드 우선(G123).
- **위반 시 처리**: 동일 장소가 복수 canonical로 분열되면 U5 POI 그라운딩 하드 제약의 전제가 무너짐 — **D37 하드 제약, 머지 차단 테스트** [U3 DoD, PBT U3-P1].
- **근거**: US-E3-01·03, D13, G133/G148, PBT U3-P1.

### BR-U3-33 소스별 캐싱 정책 게이트 (D13)
- **조건**: POI·숙소 필드의 저장·캐싱 판정 시.
- **동작**: [domain-entities.md](./domain-entities.md) §10 소스별 캐싱 매트릭스를 정본으로 판정한다(`cachingPolicy(source, field)` 총함수) — **지도 API(카카오·네이버) 유래 비식별자 필드(명칭·좌표·영업시간·카테고리)는 영구 저장 0**(TTL 캐시만, 만료 후 재조회 강제). TourAPI(공공)·사용자 확정 스냅샷만 영구 저장 허용.
- **위반 시 처리**: 지도 API 유래 필드의 영구 저장은 약관 위반급 결함 — **금지 소스 영구저장 0 불변식은 PBT U3-P3** [ADR-0017, G193].
- **근거**: US-E3-01·02, D13, G70/G105/G124/G169/G193, ADR-0017, PBT U3-P3.

### BR-U3-34 사용자 확정 데이터 영구 스냅샷 (D13)
- **조건**: 사용자 확정 데이터(등록 숙소·저장 POI·방문 체크·필수 방문지) 확정 시(`snapshotUserConfirmed`).
- **동작**: 확정 시점 사본(이름·좌표·카테고리·영업시간·출처)을 **'사용자 입력 데이터'로 영구 저장**한다(PoiSnapshot, INV-SNAP1·3). 지도 API 실시간 캐싱 금지의 정책적 예외 축(P2 법무 확인 전제). 원본 갱신은 스냅샷에 역류하지 않는다(시점 고정).
- **위반 시 처리**: 스냅샷 사후 변경·원본 역류는 결함(대조 무결성 훼손). 탐색·추천 풀을 스냅샷화하는 것은 결함(INV-SNAP3).
- **근거**: US-E3-06(등록 스냅샷), D13, G70, G124, P2.

### BR-U3-35 TTL 캐시 만료 강제
- **조건**: StaySearchCache·POI 탐색 풀 조회 시.
- **동작**: 모든 캐시 행은 ttlExpiresAt를 가지며, 만료분은 무효로 취급해 **재조회를 강제**한다(INV-CACHE1). 재조회 실패 시 stale 데이터에 '갱신 실패' 표기(허위 최신성 금지). 정적 콘텐츠 캐시 일 1회 갱신(G196).
- **위반 시 처리**: 만료 캐시를 최신으로 표시하는 것은 결함 — **만료 후 재조회 강제는 PBT U3-P3** [U3 DoD].
- **근거**: US-E3-01·03·11, D13, G196, ADR-0017, PBT U3-P3.

### BR-U3-36 탐색 부분 실패·전체 실패 처리
- **조건**: 탐색 소스(TourAPI·지도) 응답 시.
- **동작**: **일부 소스 실패**면 가용 결과로 목록을 구성하고 "일부 정보가 빠졌을 수 있어요"(degraded)를 표기한다(침묵 실패 금지). **전체 실패**면 "지금 숙소 정보를 불러올 수 없어요"+재시도, 원하면 숙소 직접 등록(US-E3-08)으로 우회. **필터 0건**은 원인 필터 표기+완화 제안, **데이터 부족 지역**은 "직접 등록하면 일정을 만들어 드려요" 안내+등록 경로 연결.
- **위반 시 처리**: 부분 실패 시 전체 실패로 처리하거나 침묵하는 것은 결함. 1차 소스(카카오) 실패 → 2차(네이버) 폴백(D08).
- **근거**: US-E3-10, US-E3-11, D09, D08, ADR-0011.

## K. 소실 POI 처리

### BR-U3-37 소실 POI '확인 불가' 배지·시드 투입 제외
- **조건**: 정본 동기화 배치에서 POI가 소스에서 확인 불가일 때(dataStatus=`LOST`).
- **동작**: '확인 불가' 배지를 유지하고, **필수 방문지 시드 투입에서 제외 안내**한다(G8). 이미 저장된 PoiSnapshot(사용자 확정 사본)은 마지막 스냅샷으로 잔존 표시(INV-SNAP2 — 허위 최신성 금지·배지 병기).
- **위반 시 처리**: 소실 POI를 후보 풀·필수 방문지 시드에 투입하는 것은 결함(U5 그라운딩 오염). 스냅샷 파기는 결함(사용자 데이터 보존).
- **근거**: US-E3(횡단), G8, PBT U3-P9, CP1 시나리오 3.

### BR-U3-38 소실 처리 멱등성
- **조건**: 정본 동기화 배치 재실행 시.
- **동작**: 소실 판정·시드 제외 신호는 **멱등**하다 — 동일 소실 상태에 배치를 재실행해도 상태·스냅샷·제외 신호가 중복 변경되지 않는다.
- **위반 시 처리**: 배치 재실행이 스냅샷을 중복 생성·상태를 진동시키는 것은 결함 — PBT U3-P9(멱등).
- **근거**: G8, PBT U3-P9.

## L. 포스트백 자동등록 이연

### BR-U3-39 포스트백 1탭 자동 등록 후속 이연 — 수동 경로만
- **조건**: 예약 확인→자동 등록 경로 설계 시.
- **동작**: **포스트백 기반 1탭 자동 등록은 1차 제외**(후속 이연) — 1차는 **복귀 핸드오프 카드(BR-U3-27)+수동 빠른 등록만** 제공한다. PostbackRecord는 **집계용 스키마 여지만** 확보하고 수신 로직은 구현하지 않는다(G29/G108).
- **위반 시 처리**: 1차에서 포스트백 자동 등록을 활성화하는 것은 범위 위반. 수신 엔드포인트 실동작은 결함(스키마 여지만).
- **근거**: US-E3-06, D09, G29/G108.
